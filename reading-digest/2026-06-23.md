# Reading digest — 2026-06-23

Found 10 paper(s) in the last 48h window.

## 1. Self-Compacting Language Model Agents
- Authors: Tianjian Li, Jingyu Zhang, William Jurayj et al.
- Summary: The paper hands agents a compaction tool plus a short rubric that tells the model itself when to summarize its accumulated trace and when to wait, instead of triggering compaction at fixed token thresholds. Across six math and agentic-search benchmarks and seven models, this learner-driven compaction matches or beats fixed-interval summarization at 30–70% lower per-question token cost and gains up to 18.1 points over a no-summary baseline, showing that "when to compact" can be supplied by scaffolding rather than training.
- Theme: Memory for LLMs and agents
- Relevance: Treats context as a managed memory resource and shows a training-free recipe for agentic memory governance — a direct hit on theme (1).
- Link: https://arxiv.org/abs/2606.23525

## 2. Kamera: Unified Position-Invariant Multimodal KV Cache for Training-Free Reuse
- Authors: Bole Ma, Jan Eitzinger, Harald Koestler
- Summary: Kamera makes the KV cache position-invariant by re-rotating RoPE to any target position at reuse time and applying a patch that restores cross-chunk binding, so cached prefixes from one conversation can be spliced into another without retraining. The authors report ~1.65x inference speedup while keeping outputs intact, effectively turning the KV cache into a portable long-term working-memory store for multimodal LLM serving.
- Theme: Memory for LLMs and agents
- Relevance: Reframes the KV cache as a reusable memory artifact for agent/state management — the second clear memory paper in the window.
- Link: https://arxiv.org/abs/2606.23581

## 3. MAS-PromptBench: When Does Prompt Optimization Improve Multi-Agent LLM Systems?
- Authors: Juyang Bai, Laixi Shi
- Summary: This benchmark systematically sweeps prompt-optimization methods across diverse multi-agent configurations (different tasks, workflows, and team sizes) to isolate when joint prompt search actually helps coordination versus when it is wasted compute. The study reports both the gains achievable and structural failure modes, giving the multi-agent community a clearer picture of how prompt engineering interacts with role and topology design.
- Theme: Multi-agent frameworks & orchestration
- Relevance: Targets the orchestration knob (prompts) of multi-agent systems and gives a principled empirical map — directly on theme (2).
- Link: https://arxiv.org/abs/2606.23664

## 4. PlanBench-XL: Evaluating Long-Horizon Planning of LLM Tool-Use Agents in Large-Scale Tool Ecosystems
- Authors: Jiayu Liu, Qihan Lin, Cheng Qian et al.
- Summary: PlanBench-XL is an interactive benchmark of 327 retail tasks spanning 1,665 tools where agents must iteratively retrieve which tool to use, chain calls to surface intermediate evidence, and recover when tools are blocked, fail, or distract. Across ten leading LLMs the best model (GPT-5.4) drops from 51.90% on clean settings to 11.36% under severe blocking, exposing how brittle current planners are when failures lack explicit error signals.
- Theme: Multi-agent frameworks & orchestration
- Relevance: Stresses tool discovery, planning, and adaptation in large tool ecosystems — the orchestration substrate most agent frameworks rely on.
- Link: https://arxiv.org/abs/2606.22388

## 5. EnterpriseClawBench: Benchmarking Agents from Real Workplace Sessions
- Authors: Jincheng Zhong, Weizhi Wang, Che Jiang et al.
- Summary: The authors mine 852 reproducible tasks from a large archive of real enterprise agent sessions, each task paired with recovered fixtures, role classes, hard rules, and semantic rubrics so harnesses and models can be compared on file-handling, tool-use, and artifact-delivery work. Even the strongest configuration (Codex + GPT-5.5) reaches only 0.663, leading them to argue that agent evaluation must report harness–model combinations, cost, runtime, and skill-transfer rather than a single score.
- Theme: Multi-agent frameworks & orchestration
- Relevance: A realistic, sessions-grounded testbed for agent harnesses — useful for thinking about how to evaluate agent architectures end-to-end.
- Link: https://arxiv.org/abs/2606.23654

## 6. Causal Discovery in the Era of Agents
- Authors: Yujia Zheng, Vishal Verma, Mantej Gill
- Summary: This work casts causal discovery as a multi-agent workflow: specialized agents coordinate data analysis, preprocessing, method recommendation, expert-knowledge incorporation, formal discovery, and interpretation, while a final stage keeps causal claims grounded in data and verified algorithms. The result is an orchestration pattern for "human-in-the-loop science" where agents handle scaffolding and a formal core enforces correctness.
- Theme: Multi-agent frameworks & orchestration
- Relevance: A concrete role/tool-assignment pattern that pairs agentic flexibility with a formal verifier — a useful template beyond the causal-discovery domain.
- Link: https://arxiv.org/abs/2606.23608

## 7. Randomized YaRN Improves Length Generalization for Long-Context Reasoning
- Authors: Manas Mehta, Fangcong Yin, Greg Durrett
- Summary: The authors combine YaRN-style positional extrapolation with a randomized positional encoding that exposes the model to out-of-distribution positions during short-context training, so it generalizes to much longer inputs at inference. They report consistent gains on long-context reasoning from 16K up to 128K tokens, offering a low-cost training-side path to longer working memory.
- Theme: Core LLM advances (long-context reasoning; memory-adjacent)
- Relevance: A simple training recipe for length generalization — directly improves the substrate that any long-term context or memory system runs on.
- Link: https://arxiv.org/abs/2606.23687

## 8. SPIRAL: Learning to Search and Aggregate
- Authors: Jubayer Ibn Hamid, Ifdita Hasan Orney, Michael Y. Li
- Summary: SPIRAL trains models to interleave sequential reasoning inside a single trace with independently sampled parallel traces and a learned aggregation step, instead of choosing one of those scaling modes upfront. The paper reports better scaling efficiency than GRPO on reasoning benchmarks, suggesting that learning when to branch and how to fuse results is itself a learnable skill.
- Theme: Core LLM advances (reasoning/planning relevant to agents)
- Relevance: A general reasoning-scaling primitive that maps cleanly onto multi-agent search/aggregation patterns.
- Link: https://arxiv.org/abs/2606.23595

## 9. CLI-Universe: Towards Verifiable Task Synthesis Engine for Terminal Agents
- Authors: Zhanbo Hua, Yifan Yao, Weihao Xie et al.
- Summary: CLI-Universe samples candidate terminal-agent tasks across a capability taxonomy, grounds each candidate in real technical materials, and then instantiates them in Docker with rubric-gated test construction and fail-to-pass checks, discarding ~2/3 of candidates as ambiguous or trivial. Fine-tuning Qwen3-32B on the resulting 6K trajectories reaches 33.4% on Terminal-Bench 2.0, a new SOTA for open-data models at ≤32B parameters.
- Theme: Multi-agent frameworks & orchestration (agent training data pipeline)
- Relevance: Shows how verifiable, executable supervision unlocks much smaller agent backbones — useful for anyone building agent training pipelines.
- Link: https://arxiv.org/abs/2606.22883

## 10. SQLConductor: Search-to-Policy Learning for Step-wise Text-to-SQL Orchestration
- Authors: Yizhang Zhu, Zhangyang Peng, Boyan Li
- Summary: SQLConductor uses Monte Carlo Tree Search to explore workflows over frozen specialist modules (schema linking, SQL generation, refinement) and then distills the search behaviour into a step-wise policy that picks the next module on its own. The learned conductor reaches 73.2% execution accuracy on BIRD-Dev, demonstrating that "orchestrator-as-policy" is a tractable way to coordinate heterogeneous tools.
- Theme: Multi-agent frameworks & orchestration
- Relevance: Concrete instance of the search → policy distillation pattern for multi-tool agents, transferable beyond text-to-SQL.
- Link: https://arxiv.org/abs/2606.23537

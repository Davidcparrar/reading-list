# Reading digest — 2026-07-22

Found 10 paper(s) in the last 48h window.

1. **Supra Cognitive Modes**
   - Authors: Joshua Tobkin, David Yang
   - Summary: Proposes an agent-memory routing architecture that dispatches each query to one of several specialized retrieval "modes" — fused lexical + dense lookup, graph-based or iterative multi-hop resolution, and stratified long-form synthesis — rather than relying on a single monolithic RAG stack. The mode router lets the agent match retrieval strategy to query shape (fact, relational, or long-context). Reports task accuracy between 61.5% and 86.0% across three memory-heavy benchmarks, though baseline comparisons and latency data are still incomplete.
   - Theme: Memory for LLMs and agents
   - Relevance note: Directly targets long-term/RAG memory as a routed capability rather than a fixed pipeline — the single memory-native contribution in-window.
   - Link: https://arxiv.org/abs/2607.19096

2. **EduPanel: A Three-Agent LLM Judge for Teaching Videos — Reliability, Complementarity, and Human Trust Calibration**
   - Authors: Jia-Kai Dong, Yi-Cheng Lin, Hung-yi Lee
   - Summary: Structures LLM-as-judge evaluation of teaching videos as a panel of three specialized rubric-grounded agents rather than a single scorer, with each agent conditioned on a different learner perspective. The panel design targets the reliability, complementarity, and human-trust calibration failure modes typical of single-judge setups. Reports agreement comparable to the median human expert and drops scoring MAE from 0.87 to 0.73 when the panel exchanges feedback.
   - Theme: Multi-agent frameworks & orchestration
   - Relevance note: Clean, empirically evaluated multi-agent judging pattern that generalizes beyond video and quantifies the marginal value of role specialization.
   - Link: https://arxiv.org/abs/2607.18529

3. **FinSAgent: Corpus-Aligned Multi-Agent RAG for SEC Filing QA**
   - Authors: Jijun Chi, Zhenghan Tai, Hanwei Wu, et al.
   - Summary: Builds a multi-agent RAG system whose role-specialized agents are anchored to the mandated 10-K item structure, then performs database-aware query decomposition and a feature-gated reranking step before answer synthesis. The design turns the SEC corpus schema itself into an orchestration prior for agent roles. Improves retrieval faithfulness and answer accuracy over strong RAG and single-agent baselines on regulatory QA.
   - Theme: Multi-agent frameworks & orchestration
   - Relevance note: Concrete example of schema-driven role assignment — a pattern likely to generalize to any structured corpus.
   - Link: https://arxiv.org/abs/2607.18102

4. **AgentDebugX: An Open-Source Toolkit for Failure Observability, Attribution, and Recovery in LLM Agents**
   - Authors: Kunlun Zhu, Xuyan Ye, Zhiguang Han, et al.
   - Summary: Frames LLM agent debugging as an explicit Detect-Attribute-Recover-Rerun loop, with a DeepDebug component that does multi-turn root-cause diagnosis over the trace. Provides open-source instrumentation for the whole loop rather than a single evaluator model. Lifts GAIA end-to-end accuracy from 55.8% to 63.6% by repairing 13 of 73 previously failed tasks and reports the strongest strict attribution accuracy against baselines.
   - Theme: Agentic-system evaluation (core LLM / orchestration)
   - Relevance note: Turns agent evaluation from pass/fail into observability + repair — reusable infrastructure for any agent stack.
   - Link: https://arxiv.org/abs/2607.18754

5. **AI Tour Meeting: Multi-Agent Group Travel Planning via Persona Discussion**
   - Authors: Daisuke Kikuta
   - Summary: Assigns distinct personas to LLM agents that then negotiate a shared travel plan through natural-language discussion, resolving competing constraints and preferences without a central planner. Communication is unstructured dialogue rather than a fixed protocol, and the paper studies whether persona diversity actually improves consensus quality. Demonstrates the pattern on group-planning tasks with multi-party preference conflicts.
   - Theme: Multi-agent frameworks & orchestration
   - Relevance note: Small but clean study of persona-conditioned dialogue as a coordination primitive.
   - Link: https://arxiv.org/abs/2607.18806

6. **ETAS: An Effect-Typed Language for Agent Systems**
   - Authors: Huiri Tan, Yikun Wang, Puyang Zhang, et al.
   - Summary: Introduces a programming language whose type system separates deterministic computation from agentic non-determinism, tracking which effects an agent may "escape" into the outside world and which action traces persist. The static semantics give a principled place to hang authorization checks and audit logs across multi-agent workflows. Positions the language as a foundation for tool-use safety rather than yet another agent framework.
   - Theme: Multi-agent frameworks & orchestration
   - Relevance note: Rare typed-language approach to agent orchestration — worth a look if you care about tool-call auditing.
   - Link: https://arxiv.org/abs/2607.17780

7. **ISO: An RLVR-Native Optimization Stack**
   - Authors: Hanqing Zhu, Wenyan Cong, Zhizhou Sha, et al.
   - Summary: Analyzes reinforcement learning with verifiable rewards through the singular structure of the base model's weights, then proposes Isospectral Optimization, which optimizes the frames while preserving the base spectra. This constrained update rule keeps the base model's capabilities intact while still shaping reasoning behavior. Reports better accuracy on reasoning and coding tasks with substantially fewer training steps than standard RLVR baselines.
   - Theme: Core LLM advances (RL fine-tuning)
   - Relevance note: Interesting spectral prior for RLVR that could translate into a lighter-weight PEFT-style fine-tune.
   - Link: https://arxiv.org/abs/2607.19331

8. **Stale but Stable: Staleness-Adaptive Trust Regions for Asynchronous Reinforcement Learning**
   - Authors: Junyao Yang, Yucheng Shi, Zongxia Li, et al.
   - Summary: Diagnoses instability in asynchronous LLM RL as a mismatch driven by high-staleness policy updates, then uses a detached log-ratio as a proxy for that staleness and kernel-scales the trust region accordingly. The resulting SAT-GSPO variant stabilizes async training without collapsing to synchronous throughput. Reports best-observed AIME24 scores while maintaining async wall-clock advantages.
   - Theme: Core LLM advances (reasoning fine-tuning)
   - Relevance note: Practical fix for a well-known async-RL failure mode as reasoning-oriented RL scales up.
   - Link: https://arxiv.org/abs/2607.18722

9. **H²SD: Hybrid Hindsight Self-Distillation**
   - Authors: Qiye Cai, Yichuan Ma, Linyang Li, et al.
   - Summary: Introduces a self-distillation scheme that treats the teacher signal differently depending on whether the sampled trajectory succeeded or failed — modulating update magnitudes on wins and injecting explicit corrective direction on losses. This asymmetric treatment avoids the "one signal fits all" behavior of standard RLVR. Consistently outperforms RLVR and related self-improvement baselines on reasoning benchmarks.
   - Theme: Core LLM advances (fine-tuning)
   - Relevance note: Nice conceptual move — success and failure trajectories carry different information, and it shows.
   - Link: https://arxiv.org/abs/2607.18955

10. **Two-Level Meta-Rubrics for Evaluating Open-Ended Generation: GAMUT, a Benchmark for Factual Completeness**
    - Authors: Xilun Chen, Zhaleh Feizollahi, Ross Goodwin, et al.
    - Summary: Proposes a two-level meta-rubric framework that mechanically compiles structured rubrics into machine-gradable checklists, then applies it to a 1,813-question benchmark grounded in wearable imagery across ten domains. The design targets factual completeness of long-form answers, which point-wise LLM judges tend to underweight. Provides the checklist compiler as reusable evaluation infrastructure.
    - Theme: Core LLM advances (agentic-system evaluation)
    - Relevance note: Rubric-compilation approach is generally useful whenever you need auditable open-ended eval.
    - Link: https://arxiv.org/abs/2607.19322

## 1. FlashMemory-DeepSeek-V4: Lightning Index Ultra-Long Context via Lookahead Sparse Attention

- **Authors:** Yan Wang, Qifan Zhang, Jiachen Yu, et al.
- **Summary:** Introduces a Lookahead Sparse Attention (LSA) inference paradigm in which a
  separately trained "Neural Memory Indexer" (a standard dual-encoder, trained without
  loading the backbone into GPU memory) predicts which historical KV chunks the model
  will actually need, evicting the rest. On the DeepSeek-V4 architecture this compresses
  the physical KV-cache footprint to ~13.5% of full context on LongBench-v2 / LongMemEval
  / RULER while slightly improving downstream accuracy, and suppresses >90% of KV
  overhead at 500K-token context lengths.
- **Theme:** Memory for LLMs and agents — explicit working-memory / KV-cache management
  for ultra-long context.
- **Relevance:** Bullseye for theme (1): proactive, learned eviction of historical
  context with strong empirical numbers on long-context memory benchmarks, exactly the
  long-term/working-memory direction the digest tracks.
- **Link:** https://arxiv.org/abs/2606.09079

## 2. End-to-End Context Compression at Scale

- **Authors:** Ang Li, Sean McLeish, Haozhe Chen, et al.
- **Summary:** Revisits encoder–decoder context compression and closes the gap to
  KV-cache compression by performing an architecture search and continually
  pre-training a 0.6B-encoder / 4B-decoder family on 350B tokens per checkpoint at 1:4,
  1:8, and 1:16 ratios. The resulting Latent Context Language Models (LCLMs) push the
  Pareto frontier across general task accuracy, compression speed, and peak memory,
  and act as efficient backbones for long-horizon agents that "skim" compressed context
  and expand relevant segments on demand.
- **Theme:** Memory for LLMs and agents — retrieval-augmented / compressed long-term
  memory, with an explicit agentic deployment.
- **Relevance:** Both an architectural memory contribution and a usable substrate for
  agentic long-horizon memory — pairs naturally with the FlashMemory paper above.
- **Link:** https://arxiv.org/abs/2606.09659

## 3. Echo-Memory: A Controlled Study of Memory in Action World Models

- **Authors:** Wayne King, Zeyue Xue, Yuxuan Bian, et al.
- **Summary:** Holds the backbone, optimizer, action interface, sampler, and evaluation
  pipeline fixed and varies *only* the memory mechanism — raw context, compression-based
  memory, spatial summaries with different read-out paths, and state-space recurrence —
  in an action-conditioned video world model. A three-branch protocol (replay quality,
  in-domain loop revisit, open-domain return) separates capacity, compression, read-out,
  and recurrence; raw context is a strong capacity baseline, aggressive compression
  loses salient evidence on return, and block-wise state-space recurrence wins on
  open-domain return.
- **Theme:** Memory for LLMs and agents — though the substrate is a video world model,
  the paper is explicitly a controlled memory study (state vs. context vs. compression).
- **Relevance:** One of the few controlled, ceteris-paribus memory ablations of the
  week; transferable lessons for LLM-agent memory architectures.
- **Link:** https://arxiv.org/abs/2606.09803

## 4. Latent Spatial Memory for Video World Models (Mirage)

- **Authors:** Weijie Wang, Haoyu Zhao, Yifan Yang, et al.
- **Summary:** Replaces the standard explicit RGB-space point-cloud memory in video
  world models with a persistent 3D cache stored *directly in diffusion latent space*,
  lifted via depth-guided back-projection and queried by latent-space warping. Avoiding
  the round trip through pixels yields up to 10.57× faster end-to-end generation and
  55× less memory than explicit 3D baselines while reaching SOTA on WorldScore and
  strong RealEstate10K reconstruction.
- **Theme:** Memory for LLMs and agents — persistent latent-space spatial memory; a
  state-management technique adjacent to LLM-side memory work.
- **Relevance:** Demonstrates that storing state in the *model's own latent space*
  rather than in an external reconstructed format is a real efficiency lever; useful
  parallel for LLM/agent memory designers.
- **Link:** https://arxiv.org/abs/2606.09828

## 5. OmniGameArena: A Unified UE5 Benchmark for VLM Game Agents with Improvement Dynamics

- **Authors:** Mingxian Lin, Shengju Qian, Yuqi Liu, et al.
- **Summary:** Builds twelve new Unreal Engine 5 games (7 Solo, 3 PvP, 2 Coop) with a
  unified text-action interface so commercial VLMs, open-weight VLMs, and specialized
  game policies can be compared on equal footing. Adds an "Improvement Dynamics Curve"
  harness in which a tool-using reflector LLM autonomously refines a bounded skill
  prompt over rounds, surfacing not just first-attempt scores but how each agent
  improves and whether its learned skill transfers to held-out variants.
- **Theme:** Multi-agent frameworks & orchestration — PvP/Coop multi-agent gameplay
  plus an explicit reflector-agent orchestration loop.
- **Relevance:** Rare combination of true multi-agent settings (PvP/Coop) and a
  measurable self-improvement protocol; directly probes agent-architecture quality
  rather than only cold-start ability.
- **Link:** https://arxiv.org/abs/2606.09826

## 6. SpatialWorld: Benchmarking Interactive Spatial Reasoning of Multimodal Agents

- **Authors:** Hongcheng Gao, Hailong Qu, Jingyi Tang, et al.
- **Summary:** Unifies eight heterogeneous simulators under a single text-action,
  vision-only protocol and ships 760 human-annotated tasks (household, travel, social
  collaboration) with reference trajectories and terminal-state verifiers. Evaluating
  15 frontier agents shows even GPT-5 only reaches 17.4% task success and exposes a
  consistent mismatch between success and execution efficiency.
- **Theme:** Multi-agent frameworks & orchestration — really an *agent-system*
  evaluation harness rather than multi-agent per se, but the unified action interface
  and simulator-agnostic protocol are directly relevant to orchestration research.
- **Relevance:** Useful new evaluation surface for tool-using / planning agents under
  partial observability; the sharp gap between best models and the ceiling is a clean
  research signal.
- **Link:** https://arxiv.org/abs/2606.09669

## 7. AHA-WAM: Asynchronous Horizon-Adaptive World-Action Modeling with Observation-Guided Context Routing

- **Authors:** Jisong Cai, Long Ling, Shiwei Chu, et al.
- **Summary:** A dual-DiT robot world-action model that decouples a low-frequency
  "world planner" DiT — which maintains a rolling KV memory over past observations and
  exposes layer-wise latent context encoding long-horizon scene evolution — from a
  high-frequency action DiT that queries that context via joint attention. With
  horizon-adaptive offset training and Observation-Guided Video-Context Routing, it
  reaches 92.8% RoboTwin success, 78.3% across 4 real-world tasks, and 24 Hz
  closed-loop control without any robot-data pretraining.
- **Theme:** Multi-agent frameworks & orchestration — specifically the
  planner/executor split with a shared memory channel, which is a recurring pattern in
  agent orchestration.
- **Relevance:** Concrete demonstration that asynchronous, memory-mediated
  planner/executor decomposition pays off; ideas transfer to LLM-agent orchestration.
- **Link:** https://arxiv.org/abs/2606.09811

## 8. Reasoning Arena: Trace Tournaments When Verifiable Rewards Fall Short

- **Authors:** Han Zhou, Adam X. Yang, Laurence Aitchison, et al.
- **Summary:** Tackles the "all-same-reward" failure mode of RLVR by routing
  zero-advantage prompt groups to a judge that runs head-to-head trace tournaments,
  with each new trace compared only against a small dynamically updated anchor pool;
  a Bradley–Terry model is fit on the resulting incomplete comparison graph to
  recover scalar rewards. Across math and coding benchmarks this improves the RLVR
  baseline by 7.6% on average and accelerates training by 27–41% while saving ~50%
  of generation compute.
- **Theme:** Core LLM advances — reasoning / RL post-training directly relevant to
  agentic systems that depend on RLVR-trained backbones.
- **Relevance:** Clean, plug-in fix for a real pain point in current reasoning RL; the
  efficiency wins make it a likely default for follow-up work.
- **Link:** https://arxiv.org/abs/2606.09380

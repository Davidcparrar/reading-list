# Reading digest — 2026-07-16

Found 10 paper(s) in the last 48h window.

## 1. Experience Memory Graph: One-Shot Error Correction for Agents
- **Authors:** Wenjun Wang, Yuchen Fang, Fengrui Liu, et al.
- **Summary:** Reframes agent failure recovery as a graph-matching problem over past trajectories: successful and failed runs are compressed into an experience graph whose common subgraphs act as reusable repair patches. At inference the agent matches the current failing sub-trajectory against this graph and applies the retrieved correction in a single shot, avoiding the multi-round reflection loops used by prior methods while still improving success on the benchmarks tested.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Structured episodic memory used for cross-episode error correction — exactly the retrieval-augmented memory angle the digest tracks.
- **Link:** https://arxiv.org/abs/2607.13884

## 2. A Self-Evolving Agent for Longitudinal Personal Health Management (HealthClaw)
- **Authors:** Haoran Li, Jiebi Deng, Tong Jin, et al.
- **Summary:** Introduces an agent architecture that decouples immutable safety rules from a longitudinal user-history memory, keeping the prompt short while still conditioning on months of accumulated events. In a 900-probe evaluation the memory-backed agent lifts accuracy from ~0.2% (current-query-only prompting) to ~45.7% while cutting prompt context by ~72%, showing that most of the health-management skill lives in the memory, not the model.
- **Theme:** Memory for LLMs and agents
- **Relevance:** A concrete long-term / episodic memory design with a real evaluation of how much retrieval buys vs. raw prompting.
- **Link:** https://arxiv.org/abs/2607.13940

## 3. KnowAct-GUIClaw: Personal GUI Assistant with Self-Evolving Memory and Skill
- **Authors:** Yunxin Li, Jinchao Li, Shibo Su, et al.
- **Summary:** A cross-platform GUI agent (Android, iOS, HarmonyOS, Windows) that pairs a self-evolving memory store with a persistent skill library the agent grows from its own successful executions. Built on open-weight Kimi-2.6 backbones it reaches 64.1% on MobileWorld, arguing that skill accretion — not larger prompts — is what closes the gap to closed-source GUI agents.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Combines working-memory management with a persistent skill/tool library, one of the digest's core memory sub-themes.
- **Link:** https://arxiv.org/abs/2607.12625

## 4. SPyCE: Skill-Policy Co-evolution for Multimodal Agents
- **Authors:** Ru Zhang, Weijie Qiu
- **Summary:** Distills raw multimodal agent trajectories into a hierarchy of reusable skills, then co-evolves those skills with the policy during RL so each improved policy produces cleaner trajectories which in turn yield better skills. The result is a closed loop where the skill library and the policy sharpen each other, without needing a separate distillation stage between RL runs.
- **Theme:** Multi-agent frameworks and orchestration
- **Relevance:** Direct contribution to role/tool/skill assignment inside agent architectures — the orchestration side of the theme.
- **Link:** https://arxiv.org/abs/2607.13854

## 5. Navigating the Mirage: A Dual-Path Agentic Framework for Robust Misleading Chart Question Answering (ChartCynics)
- **Authors:** Yanjie Zhang, Yafei Li, Rui Sheng, et al.
- **Summary:** Proposes a two-agent pipeline for chart QA where one path interrogates the chart visually and a second path reconstructs the underlying numbers via OCR, then a coordinator reconciles the two before answering. Splitting the labor this way lifts robustness on deliberately misleading charts by ~29 absolute points over single-agent baselines, showing the value of complementary perception modes as an orchestration primitive.
- **Theme:** Multi-agent frameworks and orchestration
- **Relevance:** Concrete role-specialization + coordination protocol between two agents with different perceptual biases.
- **Link:** https://arxiv.org/abs/2603.28583

## 6. CAVA: Canonical Action Verification and Attestation for Runtime Governance of Agentic AI Systems
- **Authors:** Alexandra E. Michael, Franziska Roesner (and collaborators)
- **Summary:** Adds a runtime-semantics layer beneath heterogeneous agents (browsers, SDKs, API gateways) that rewrites every emitted action into a canonical, attestable form so downstream policy engines can reason about behavior uniformly. This makes cross-agent governance, permission checks, and audit trails tractable in orchestration setups where the constituent agents were never designed to interoperate.
- **Theme:** Multi-agent frameworks and orchestration
- **Relevance:** A communication/attestation protocol layer for multi-agent systems, addressing the coordination-plane gap.
- **Link:** https://arxiv.org/abs/2607.13716

## 7. Do Agent Optimizers Compound? A Continual-Learning Evaluation on Terminal-Bench 2.0
- **Authors:** Wenxiao Wang, Priyatham Kattakinda, Soheil Feizi
- **Summary:** Tests whether prompt/agent optimizers (GEPA-style) actually stack when applied sequentially over a stream of tasks, or whether later optimization erodes earlier gains. Their RELAI-VCL variant, which explicitly controls regression across task boundaries, hits 76.4% lifelong performance versus 66.0% for GEPA — evidence that compounding is real but only under an explicit anti-regression constraint.
- **Theme:** Core LLM advances (agentic-system evaluation)
- **Relevance:** Rare longitudinal evaluation of agent-optimization pipelines, directly on the digest's agentic-eval axis.
- **Link:** https://arxiv.org/abs/2607.14004

## 8. AgentCompass: A Unified Evaluation Infrastructure for LLM-Based Agents
- **Authors:** Zichen Ding, Jiaye Ge, Shufan Jiang, et al.
- **Summary:** A harness that decomposes agent evaluation into three orthogonal components (environment, protocol, judge) and runs 20+ existing benchmarks over five capability axes through a single fault-tolerant async runtime. The contribution is more infrastructural than scientific, but it lets practitioners compare agents across benchmarks without re-implementing plumbing per suite.
- **Theme:** Core LLM advances (agentic-system evaluation)
- **Relevance:** Reusable evaluation infrastructure for the whole agent ecosystem — high leverage for the eval theme.
- **Link:** https://arxiv.org/abs/2607.13705

## 9. Deep Interaction: An Efficient Human-AI Interaction Method for Large Reasoning Models
- **Authors:** Hefeng Zhou, Jinxuan Zhang, Jiong Lou, et al.
- **Summary:** Instead of re-prompting from scratch when a reasoning model goes off-track, this method lets a human (or supervisor) edit the intermediate reasoning trace in place and resume decoding from the edit point. The reported gains — >25% higher correction success and ~40% fewer generated tokens versus re-prompting — suggest treating the reasoning chain as an editable artifact is a meaningful lever for reasoning-time control.
- **Theme:** Core LLM advances (reasoning)
- **Relevance:** Direct-editing-of-thoughts is a fresh reasoning-control primitive that is likely to influence downstream agent designs.
- **Link:** https://arxiv.org/abs/2607.14049

## 10. DeltaMerge-LowRes: Composing Language and Task Deltas for Low-Resource Adaptation
- **Authors:** Son Ha Xuan, Xuan-Bach Le, Phat T. Tran-Truong
- **Summary:** Learns two independent LoRA-style deltas — one that encodes a target low-resource language, one that encodes a target task — and merges them at inference via a cross-axis TIES rule instead of retraining on the joint distribution. Reported gains of +4 to +7 chrF on summarization and a ~36% reduction in classification ECE suggest the factorization is preserving both axes better than a single joint fine-tune.
- **Theme:** Core LLM advances (PEFT / fine-tuning)
- **Relevance:** Concrete new PEFT composition scheme with a measured calibration benefit — squarely on the fine-tuning theme.
- **Link:** https://arxiv.org/abs/2607.13967

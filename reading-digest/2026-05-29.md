## 1. Rethinking Memory as Continuously Evolving Connectivity (FluxMem)
**Authors:** Jizhan Fang, Buqiang Xu, Zhixian Wang, et al. (Zhejiang University, Alibaba Group, MemTensor, Tongji University)

**Summary:** FluxMem targets a structural weakness in memory-augmented LLM agents: most systems fix the memory representation and retrieval pipeline up front, which is brittle once an agent's environment, feedback, and task mix keep changing what is worth remembering and how items should relate. Instead of a static store, the authors model memory as a heterogeneous graph whose *connectivity* is learned and revised online, evolving through three phases — initial connection formation, feedback-driven refinement, and long-term consolidation. The contribution is a framework where the graph topology itself (not just its contents) adapts continuously to the agent's experience; concrete benchmark numbers were not retrievable through the search-only channel available in this run.

**Theme:** 1 — Memory for LLMs and agents (long-term/episodic memory, retrieval, state management).

**Relevance:** Treats memory connectivity as a dynamic, feedback-driven object rather than a fixed index — a genuine method/architecture contribution in the top-priority memory theme. Its ID (`2605.28773`, ~May 28) postdates everything in the previous digest, so it is a new in-window paper rather than a repeat.

**Link:** https://arxiv.org/abs/2605.28773

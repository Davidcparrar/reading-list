1. **Shared Selective Persistent Memory for Agentic LLM Systems**
   - Authors: Sanjana Pedada, Aditya Dhavala, Neelraj Patil
   - Summary: The paper proposes a persistent-memory layer for code-writing agentic systems that keeps four *reusable* slices of prior sessions — task specs, data schemas, tool configurations, output constraints — while explicitly discarding session-specific reasoning traces, and shares that memory across users via role-based access control. A companion "zero-token data refresh" mechanism decouples the generated program from the runtime data it operates over, so a stored plan can be re-executed against fresh inputs without re-injecting the data into the prompt. On three enterprise scenarios they report 96% task completion vs. 79% memory-less and 71% with naive full-history replay, with ~14× less task time and ~97× lower per-invocation token cost than raw data injection.
   - Theme: (3) memory for LLMs and agents (long-term / shared persistent memory).
   - Relevance note: The clearest new "what to actually persist between agent sessions" contribution in the window — separates reusable knowledge from throw-away reasoning, which is exactly the operational memory question most agent stacks avoid.
   - Link: https://arxiv.org/abs/2607.09493

2. **SAGEAgent: A Self-Evolving Agent for Cost-Aware Modality Acquisition in Multimodal Survival Prediction**
   - Authors: Chongyu Qu, Can Cui, Zhengyi Lu, et al.
   - Summary: SAGEAgent formulates multimodal clinical prediction as a sequential decision problem — should the agent request the next diagnostic modality (imaging, genomics, etc.) or commit — and equips an LLM agent with two complementary memory stores: an *episodic* memory that retrieves clinically-similar past patients, and a *semantic* memory that accumulates reusable decision heuristics learned from experience. Combined with tools that translate numerical predictions into text the LLM can reason over, this lets the agent trade predictive accuracy against acquisition cost per patient. On a combined glioma cohort (TCGA-LGG, TCGA-GBM, BraTS) the agent matches full-workup accuracy while cutting average acquisition burden by 55%.
   - Theme: (3) memory for LLMs and agents (episodic + semantic memory for decision-making).
   - Relevance note: A concrete, evaluable instance of the episodic+semantic memory split for agents — the pattern is directly transportable to any cost-aware sequential-decision agent, not just clinical ones.
   - Link: https://arxiv.org/abs/2607.09521

3. **Self-Guided Test-Time Training for Long-Context LLMs**
   - Authors: Xinyu Zhu, Zhe Xu, Xiaohan Wei, et al.
   - Summary: S-TTT tackles the observation that extending the context window does not by itself buy accurate long-context use — models still lose track of the actually-relevant spans. The recipe has the model itself first mark evidence spans in the long input, then applies parameter adaptation (a small LM-loss update) *only* on those spans rather than on random passages. Evaluated on LongBench-v2 and LongBench-Pro with Qwen3-4B and Llama-3.1-8B, they report up to 15% relative accuracy improvement over no-TTT baselines, with the self-guidance being the crucial ingredient over random-span TTT.
   - Theme: (1) core LLM advances + (3) memory / context management (test-time adaptation as an alternative to KV-cache tricks).
   - Relevance note: TTT is quietly becoming a serious lever for long-context; making the selection self-guided (no external supervision) is the piece that makes it deployable inside an agent loop.
   - Link: https://arxiv.org/abs/2607.09415

4. **Mosaic: Runtime-Efficient Multi-Agent Embodied Planning**
   - Authors: Kunjal Panchal, Saayan Mitra, Sunav Choudhary, et al.
   - Summary: Mosaic argues that latency in LLM-based multi-agent embodied planning is dominated by *failed* actions, which stem from bad state tracking under partial observability and from redundant/conflicting inter-agent actions. It combines an "agent-centric semantic memory" that stores objects in each agent's local relative frame (so geometric transformations across agents are cheap) with an Integer Linear Program that allocates the next joint action set under physical-feasibility and coordination constraints. On AI2-THOR and search-and-rescue benchmarks it lands 27-32% faster execution, 30-33% fewer LLM calls, 25-31% fewer steps, and 4-10 pt higher success than prior multi-agent LLM planners.
   - Theme: (2) multi-agent frameworks & orchestration (planning + coordination).
   - Relevance note: One of the few multi-agent papers in the window that treats runtime cost as a first-class metric, and its agent-centric semantic memory is a nice cross-cut with track (3).
   - Link: https://arxiv.org/abs/2607.09603

5. **Agora: Enhancing LLM Agent Reasoning Via Auction-Based Task Allocation**
   - Authors: Kaiji Zhou, Ales Leonardis, Yue Feng
   - Summary: Agora replaces the usual coarse "match a task to a tool by description" routing step with an incentive-compatible auction over expert models and tools: reasoning steps are treated as tradeable items, and candidate solvers bid based on a rectified competence estimate rather than raw self-reported confidence. This is intended to counter the standard failure mode where the most *overconfident* candidate wins routing, not the most *capable*. Across five benchmarks Agora beats single-model, routing, and cascade baselines with a comparable candidate pool, and exposes a single auction parameter that dials the cost-quality trade-off.
   - Theme: (2) multi-agent frameworks & orchestration (role/tool assignment, communication protocol).
   - Relevance note: The auction framing is a clean, testable mechanism for the "who does this subtask" problem — an alternative to the debate/critic patterns that dominate current multi-agent lit.
   - Link: https://arxiv.org/abs/2607.09600

6. **Long-Horizon-Terminal-Bench: Testing the Limits of Agents on Long-Horizon Terminal Tasks with Dense Reward-Based Grading**
   - Authors: Zongxia Li, Zhongzhi Li, Yucheng Shi, et al.
   - Summary: A benchmark of 46 long-horizon terminal tasks (experiment reproduction, software engineering, and seven more categories) where each task decomposes into finely-graded subtasks so evaluation can hand out dense intermediate rewards and partial credit — not just pass/fail at the end. The tasks are genuinely long: hundreds of episodes, minutes-to-hours of wall clock each. Across 15 frontier models the strongest reaches only 15.2% pass rate at a high reward threshold and 10.9% at perfect reward, with the average across all models at 4.3% / 1.7%; the paper also catalogs recurring failure modes to guide future work.
   - Theme: (1) core LLM advances (agentic-system evaluation).
   - Relevance note: The dense-reward + long-horizon combination is what current agent evals lack; the low absolute scores make it a genuine headroom benchmark rather than a saturating one.
   - Link: https://arxiv.org/abs/2607.08964

7. **Failure as a Process: An Anatomy of CLI Coding Agent Trajectories**
   - Authors: Xiangxin Zhao, Han Li, Shuaiting Li, et al.
   - Summary: The authors collect 3,843 CLI-coding-agent trajectories from seven frontier models running under three scaffolds (OpenHands, MiniSWE, Terminus2) on Terminal-Bench, filter to 1,794 clean trajectories with 63k+ execution steps, and manually annotate them to study failure as a *temporal process* rather than a final label. Fourteen findings emerge, of which the most consequential are: most failures are epistemic (bad beliefs about the environment, not bad actions), they usually appear in the very first steps of a trajectory, and they typically go undetected until recovery is no longer possible. The implication for scaffolds is that reliability gains have to come from early detection and intervention, not from grading the final output.
   - Theme: (1) core LLM advances (agentic-system evaluation, but from the trajectory-analysis angle).
   - Relevance note: Complements #6 well — where LHT-Bench asks "how far can agents go", this asks "when and how do they die", and gives an actionable early-detection thesis.
   - Link: https://arxiv.org/abs/2607.09510

8. **Towards Mechanistically Understanding Why Memorized Knowledge Fails to Generalize in Large Language Model Finetuning**
   - Authors: Lu Dai, Ziyang Rao, Yili Wang, et al.
   - Summary: The paper formalizes an under-studied phenomenon in LLM finetuning — the "Knowing-Using Gap": models memorize new facts quickly during finetuning but fail to use them in downstream reasoning, with both an accuracy gap and a temporal delay before the fact becomes usable. Using a *self-patching* intervention that traces how the newly-learned representation propagates through the model, they support a "knowledge-circuit misalignment" hypothesis: the memorized content lives internally but is not routed to layers where computation could use it. A simple heuristic derived from the diagnosis recovers 58-75% of the oracle gap-to-generalization across several domains.
   - Theme: (1) core LLM advances (fine-tuning mechanistics).
   - Relevance note: One of the more concrete mechanistic-interp results in the window, and it identifies a lever (routing to computation-effective layers) that a practitioner could actually target during finetuning.
   - Link: https://arxiv.org/abs/2607.08393

9. **TrustX Agent Risk Classification Framework (ARC): Risk-Tiering Internally Created Agentic AI Systems**
   - Authors: Hannah M. Liu, Rhea Saxena, Shiv Asthana
   - Summary: ARC is a governance-side framework for internal-to-an-org agentic systems: it defines seven agent types, scores each on a twelve-dimension rubric that spans autonomy, blast radius, data sensitivity, and tool authority, and folds in an autonomy classification model plus existing AI-risk frameworks. The output is a three-tier risk classification with mapped control recommendations, and the paper includes a specialized extension for coding assistants and an interactive tool. This is a survey/framework paper rather than a method paper — its value is in giving teams a common vocabulary and rubric for tiering agent deployments.
   - Theme: (2) multi-agent frameworks & orchestration (agent architecture / governance).
   - Relevance note: Softest fit of the nine — included because agent-risk tiering is becoming a hard requirement for anyone deploying agents in an org, and the 12-dimension rubric is a useful starting checklist even if the framework isn't adopted wholesale.
   - Link: https://arxiv.org/abs/2607.09586

# Reading digest — 2026-06-16

Found 10 paper(s) in the last 48h window.

## 1. User as Code: Executable Memory for Personalized Agents
**Authors:** Bojie Li

Argues that long-term user memory for agents should be stored as typed Python objects in an append-only log that is periodically compiled into executable code, rather than as retrieval-time text fragments. The paper reports 99% accuracy on aggregate historical questions versus 6–43% for conventional retrieval-based memory, framing memory as something agents run rather than something they read.

**Theme:** Memory for LLMs and agents (long-term/episodic memory)
**Relevance:** A genuinely novel memory paradigm with a large empirical gap over RAG baselines — top-tier fit for the memory theme.
**Link:** https://arxiv.org/abs/2606.16707

## 2. TokenPilot: Cache-Efficient Context Management for LLM Agents
**Authors:** Buqiang Xu, Zirui Xue, Dianmou Chen, et al.

Proposes a dual-granularity context manager for long-horizon agent sessions: an Ingestion-Aware Compaction harness that stabilises prompt prefixes for prompt-cache reuse, and a Lifecycle-Aware Eviction policy that tracks residual utility of context segments and drops them when they stop paying their way. Reported 61–87% cost reductions across benchmarks while preserving task quality.

**Theme:** Memory for LLMs and agents (context/state management)
**Relevance:** A method paper on how agents should manage their working context in production — directly on the memory theme and immediately practical.
**Link:** https://arxiv.org/abs/2606.17016

## 3. HiMPO: Hindsight-Informed Memory Policy Optimization for Less-Entangled Credit in Long-Horizon Agents
**Authors:** Jiangze Yan, Yi Shen, Wenjing Zhang, et al.

Tackles the credit-assignment problem for memory operations in long-horizon agents by estimating the local utility of each memory update from the task-relevant information recoverable before vs. after the write. The hindsight signal is folded into a policy-optimisation loop so the agent learns when memory writes actually help, instead of entangling memory rewards with overall task reward.

**Theme:** Memory for LLMs and agents (learned memory policy)
**Relevance:** Memory + RL credit assignment is a specific bottleneck in agentic training; this is a concrete proposal aimed at it.
**Link:** https://arxiv.org/abs/2606.16285

## 4. KVEraser: Learning to Steer KV Cache for Efficient Localized Context Erasing
**Authors:** Mufei Li, Shikun Liu, Dongqi Fu, et al.

Introduces a method for selectively removing specific spans from a model's KV cache after the fact, using learned steering states rather than recomputing from scratch. Reports matching full-recomputation quality at 3–4× lower latency, which makes context-level forgetting cheap enough to use in long-running agents.

**Theme:** Memory for LLMs and agents (state/context management)
**Relevance:** Localized forgetting in the KV cache is a missing primitive for durable agent memory — niche but load-bearing.
**Link:** https://arxiv.org/abs/2606.17034

## 5. Control-Plane Placement Shapes Forgetting: An Architectural Study of Agent Memory Across Thirteen System Configurations
**Authors:** Dongxu Yang

An empirical comparison of thirteen agent-memory architectures on 385 adversarial cases, with the finding that forgetting failures dominate recall failures in production deployments. The paper argues that mutation-time hooks — placing memory-edit control on the same plane as the action — recover intent-aware deletion (78–85%) and lift nearly every failure category at once (91.7–93.2%).

**Theme:** Memory for LLMs and agents (architecture)
**Relevance:** A rare side-by-side architectural study of agent memory; the "forgetting > recall as the production failure mode" framing is itself worth tracking.
**Link:** https://arxiv.org/abs/2606.15903

## 6. SciOrch: Learning to Orchestrate Expert LLMs for Solving Frontier Multimodal Scientific Reasoning Tasks
**Authors:** Jingru Guo, Xiangyuan Xue, Lian Zhang, et al.

Trains a lightweight 8B model whose job is to orchestrate calls to frontier LLMs through APIs for multimodal scientific reasoning, sidestepping expensive multi-agent RL. The training signal comes from MCTS-rolled-out orchestration trajectories that the small model distils, so orchestration policy is learned rather than prompt-engineered.

**Theme:** Multi-agent frameworks & orchestration
**Relevance:** A clean method paper on learning the orchestrator role itself — exactly the kind of orchestration architecture worth tracking.
**Link:** https://arxiv.org/abs/2606.15872

## 7. Tensor-Coord: Algebraic Decomposition of Joint Plan Tensors for Conflict-Free Multi-Agent LLM Planning
**Authors:** Mudit Rastogi

Represents the joint plans of multiple LLM agents as tensors and applies multilinear decomposition to surface coordination conflicts — spatial collisions, resource contention, temporal deadlocks — which the LLM then replans around iteratively. The contribution is a structured conflict-detection layer between LLM planners rather than a new agent role.

**Theme:** Multi-agent frameworks & orchestration
**Relevance:** Coordination/conflict-resolution is the under-built half of most multi-agent stacks; the algebraic formulation is a fresh angle.
**Link:** https://arxiv.org/abs/2606.16478

## 8. Misinformation Propagation in Benign Multi-Agent Systems
**Authors:** Jonas Becker, Jan Philip Wahle, Terry Ruas, Bela Gipp

Studies how injected-but-non-adversarial misinformation spreads when LLM agents debate over reasoning and knowledge tasks. The empirical finding is that errors do persist through debate but multi-agent setups are more robust than single-agent ones, with robustness depending on group composition and information-aggregation protocol.

**Theme:** Multi-agent frameworks & orchestration (communication/aggregation protocols)
**Relevance:** Useful as evidence on when adding agents actually helps and how the aggregation protocol matters — a recurring open question for multi-agent design.
**Link:** https://arxiv.org/abs/2606.16710

## 9. VibeThinker-3B: Exploring the Frontier of Verifiable Reasoning in Small Language Models
**Authors:** Sen Xu, Shixi Liu, Wei Wang, et al.

A 3B-parameter model trained with specialised post-training to reach 94.3 on AIME26 and 80.2 Pass@1 on LiveCodeBench, matching or beating much larger flagship models on those reasoning benchmarks. The paper's value is the post-training recipe for compact verifiable reasoners, not the size milestone itself.

**Theme:** Core LLM advances (reasoning)
**Relevance:** Compact strong reasoners are the underlying engine for cheap agentic systems — directly relevant to the agent-systems theme.
**Link:** https://arxiv.org/abs/2606.16140

## 10. Context-Aware RL for Agentic and Multimodal LLMs (ContextRL)
**Authors:** Peiyang Xu, Bangzheng Li, Sijia Liu, et al.

Frames long-horizon agentic reasoning as an explicit context-selection problem and trains the agent under that objective via RL, so the model learns which prior context to keep in scope for the next decision. Reports 1.8–2.2% gains on coding and visual agent benchmarks over context-agnostic RL.

**Theme:** Core LLM advances (RL for agents; touches memory)
**Relevance:** Sits at the intersection of memory and agent training — the right kind of cross-theme paper to keep an eye on.
**Link:** https://arxiv.org/abs/2606.17053

# Reading digest — 2026-07-09

Found 10 paper(s) in the last 48h window.

## 1. Co-LMLM: Continuous-Query Limited Memory Language Models
- **arXiv:** https://arxiv.org/abs/2607.07707
- **Authors:** Yair Feldman, Linxi Zhao, Nathan Godey, et al.
- **Summary:** Trains a language model whose parameters store only reasoning and language patterns while factual knowledge is offloaded to an external key–value store queried continuously during pretraining. Because the model must issue a retrieval query at every step, it learns to compose retrieved snippets rather than memorize them, and it matches or beats baselines trained on 40× more raw tokens while keeping the retrieved evidence human-readable.
- **Theme:** Memory for LLMs and agents
- **Relevance:** A concrete architecture for externalized long-term / semantic memory that is trained end-to-end with the model — one of the cleanest recent proposals for making LLM memory both scalable and inspectable.

## 2. Dual Latent Memory in Vision-Language-Action Models for Robotic Manipulation
- **arXiv:** https://arxiv.org/abs/2607.07608
- **Authors:** Hongyu Qu, Jianzhe Gao, Xiaobin Hu, et al.
- **Summary:** Extends VLA policies with two complementary latent memory tracks — a "curator/condenser" episodic buffer summarizing recent frames and a "seeker/weaver" retrieval pathway that pulls in older task-relevant tokens on demand. The dual layout lets a single policy plan across long-horizon, temporally dependent manipulation tasks without blowing up its attention context.
- **Theme:** Memory for LLMs and agents
- **Relevance:** A grounded case study in episodic + retrieval-based memory for an embodied agent, with clear design language (curator/seeker/condenser/weaver) that transfers back to text-only agent memory work.

## 3. A hierarchical memory architecture overcomes context limits in long-horizon multi-agent computational modeling
- **arXiv:** https://arxiv.org/abs/2607.07666
- **Authors:** Shivendra G. Tewari, Holly Kimko
- **Summary:** Introduces a three-layer hierarchical memory (working / episodic / long-term) that lets an ensemble of LLM agents run continuous scientific-modeling workflows without exceeding the underlying model's context. In practice injected context stays around 301 tokens (max ~4k) while the agents preserve state across long runs.
- **Theme:** Memory for LLMs and agents (with a multi-agent framing)
- **Relevance:** Sits cleanly at the memory ↔ multi-agent intersection: shows a working recipe for bounded, autonomously operated memory shared across cooperating agents.

## 4. From Noisy Traces to Root Causes: Structural Trajectory Analysis and Causal Extraction for Agent Optimization
- **arXiv:** https://arxiv.org/abs/2607.07702
- **Authors:** Ying Chang, Jiahang Xu, Xuan Feng, et al.
- **Summary:** Proposes STRACE, an offline pipeline that clusters agent trajectories, strips out redundant or already-successful sub-paths, and runs a causal-attribution step to point at the single tool call or reasoning span that actually caused a failure. Using those extracted causes as targeted feedback yields ~1.4× success-rate gains on formal-verification agent benchmarks.
- **Theme:** Multi-agent frameworks & orchestration (agent evaluation/optimization)
- **Relevance:** A rare paper that treats agent debugging as a first-class research problem with a reproducible method — directly useful for anyone building or auditing tool-using LLM agents.

## 5. Institutional Red-Teaming: Deployment Rules, Not Just Models, Causally Shape Multi-Agent AI Safety
- **arXiv:** https://arxiv.org/abs/2607.07695
- **Authors:** Yujiao Chen
- **Summary:** Runs controlled red-team simulations across populations of LLM agents while varying the *deployment rules* (consequence structures, escalation policies) rather than the models themselves. Changing rules alone moves collective failure rates by 22–58 percentage points, arguing that multi-agent safety must be evaluated at the institutional level, not per-model.
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** Shifts the multi-agent safety conversation from prompt-level jailbreaks to system design; a useful mental model for anyone deploying agent swarms.

## 6. Agent Delivery Engineering Predictive Reliability Framework
- **arXiv:** https://arxiv.org/abs/2607.07689
- **Authors:** Dexing Liu
- **Summary:** Proposes ADE-PRF, a monitoring layer that tracks per-agent latency, tool-error and reasoning-quality signals over time to forecast when an LLM multi-agent system is drifting toward failure even while surface-level metrics still look green. The paper reports 76.8% directional accuracy on predicting quality regressions before user-visible symptoms appear.
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** Practical instrumentation angle on multi-agent systems — the "false-prosperity" framing is a useful diagnostic for real deployments.

## 7. SkillCenter: A Large-Scale Source-Grounded Skill Library for Autonomous AI Agents
- **arXiv:** https://arxiv.org/abs/2607.07676
- **Authors:** Tianming Sha, Yue Zhao, Licheng Sun, Yushun Dong
- **Summary:** Builds an open library of ~217k reusable, executable skills covering 24 domains, where every skill retains a pointer back to the exact source passage that justifies it. Agents can retrieve, compose, and cite skills as first-class objects instead of re-deriving them from scratch each turn.
- **Theme:** Multi-agent frameworks & orchestration (tool/role assignment for agents)
- **Relevance:** A concrete substrate for skill-based agent architectures and a nice complement to memory-for-agents work — skills as long-term, verifiable procedural memory.

## 8. Agon: Competitive Cross-Model RL with Implicit Rival Grading of Reasoning
- **arXiv:** https://arxiv.org/abs/2607.07690
- **Authors:** Vladislav Beliaev
- **Summary:** Trains reasoning LLMs by having them grade each other's solutions instead of relying on a fixed verifier or teacher, turning peer disagreement into a reward signal. On the hardest DeepMath problems this roughly doubles GRPO's pass@1, about 8× the gain a plain Mixture-of-Agents ensemble delivers.
- **Theme:** Core LLM advances (reasoning + RL fine-tuning)
- **Relevance:** A clean, verifier-free RL setup for reasoning — practically relevant for anyone trying to squeeze more out of RL post-training without curating labels.

## 9. RL Post-Training Builds Compositional Reasoning Strategies
- **arXiv:** https://arxiv.org/abs/2607.07646
- **Authors:** Azwar Abdulsalam, Nishil Patel, Andrew Saxe
- **Summary:** Analyzes what RL post-training actually changes inside a reasoning model and finds a phased pattern: the model first consolidates a small set of reusable primitive skills, then discovers sequential and parallel compositions of those primitives. This gives a mechanistic story for the qualitative jumps seen in RL-trained reasoners.
- **Theme:** Core LLM advances (reasoning)
- **Relevance:** Rare interpretability lens on RL for reasoning — helps calibrate expectations about what fine-tuning can and cannot invent.

## 10. Max Out GRPO Signal: Adaptive Trace Prefix Control for Hard Reasoning Problems
- **arXiv:** https://arxiv.org/abs/2607.07674
- **Authors:** Vladislav Beliaev
- **Summary:** Attacks the "reward-collapse" problem in GRPO on hard problems by seeding rollouts with an adaptively chosen prefix of a reference solution, then shrinking that prefix as the model improves. The curriculum roughly doubles GRPO accuracy on held-out reasoning tasks and yields up to 2.1× improvement on the hardest math splits.
- **Theme:** Core LLM advances (reasoning fine-tuning / PEFT-adjacent)
- **Relevance:** A simple, drop-in modification to GRPO that addresses a well-known failure mode — likely to influence future reasoning RL recipes.

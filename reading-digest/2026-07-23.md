# Reading digest — 2026-07-23

Found 10 paper(s) in the last 48h window.

## 1. AutoIndex: Learning Representation Programs for Retrieval
- **Authors:** Sam O'Nuallain, Nithya Rajkumar, Ramya Narayanasamy, et al.
- **Summary:** AutoIndex reframes indexing as a search over executable programs that transform documents (slicing, enriching, reorganizing) before they are embedded, rather than tuning a single monolithic embedder. A validation-guided controller composes these transformation programs and selects the pipeline that maximises downstream retrieval quality on held-out queries. The result is a data-shape-adaptive retrieval memory that outperforms fixed pipelines when documents vary in structure.
- **Theme:** Memory for LLMs and agents (retrieval-augmented memory)
- **Relevance:** Directly targets the retrieval / indexing layer that sits between agents and their external memory — the "how do we structure long-term knowledge" question.
- **Link:** https://arxiv.org/abs/2607.18603

## 2. Scaling Laws for Hypernetwork-Based Knowledge Injection in Large Language Models
- **Authors:** Nischay Dhankhar, Dos Baha, Abulhair Saparov
- **Summary:** The authors study hypernetworks as a mechanism for injecting factual knowledge into LLM weights at train time and introduce MegaWikiQA to probe how the approach scales. They report predictable power-law behaviour in accuracy versus model/hypernetwork size and show stronger out-of-distribution generalisation than LoRA-style parameter-efficient fine-tuning on the same knowledge slices. This positions hypernetwork-based injection as a competitive PEFT-adjacent route to parametric memory.
- **Theme:** Memory for LLMs and agents (parametric / long-term memory)
- **Relevance:** Sits at the memory ↔ PEFT boundary — treats weights themselves as a memory store and gives a scaling recipe, which is directly on-priority.
- **Link:** https://arxiv.org/abs/2607.19604

## 3. Beyond Relevance-Centric Retrieval: Rubric-Oriented Document Set Selection and Ranking
- **Authors:** Kailin Jiang, Lei Liu, Jian Xi, et al.
- **Summary:** The paper argues that retrieval for downstream LLM tasks needs to pick a coherent *set* of documents against explicit rubrics rather than just top-k relevance. It contributes SetwiseEvalKit (~28K rubrics) and Rubric4Setwise, a selection-and-ranking method that jointly optimises coverage, redundancy, and rubric satisfaction, reporting SOTA on the new benchmark. The framing is useful for agent memory pipelines that need to hand LLMs a curated context bundle, not raw hits.
- **Theme:** Memory for LLMs and agents (retrieval / context management)
- **Relevance:** Rethinks the retrieval step that feeds agent context, which matters for any RAG or long-horizon memory stack.
- **Link:** https://arxiv.org/abs/2607.19747

## 4. AgentDebugX: An Open-Source Toolkit for Failure Observability, Attribution, and Recovery in LLM Agents
- **Authors:** Kunlun Zhu, Xuyan Ye, Zhiguang Han, et al.
- **Summary:** AgentDebugX packages agent failure handling as an explicit detect → attribute → recover → rerun loop, with multi-turn root-cause analysis and opt-in cross-run error sharing between agents. It exposes trace instrumentation and attribution primitives so heterogeneous agent stacks can be diagnosed uniformly. The toolkit is aimed at making agentic-system evaluation and post-mortems reproducible rather than ad hoc.
- **Theme:** Multi-agent frameworks & orchestration (also agentic-system evaluation)
- **Relevance:** Practical infrastructure for orchestrated agents — attribution and recovery are the parts most frameworks ignore.
- **Link:** https://arxiv.org/abs/2607.18754

## 5. DocOps: A Verifiable Benchmark for Autonomous Agents in Complex Document Operations
- **Authors:** Jiazhen Jiang, Boxi Cao, Lingyong Yan, et al.
- **Summary:** DocOps is a deterministically verifiable evaluation harness for agents that edit and transform structured documents, with checkable ground-truth outcomes rather than LLM-judge scores. The authors run current agents through it and identify three recurring failure modes, including collapse of long-term state tracking across many tool calls. The benchmark gives an objective testbed for agent orchestration and memory in a domain where correctness is machine-checkable.
- **Theme:** Multi-agent frameworks & orchestration (agentic-system evaluation)
- **Relevance:** Verifiable evaluation for tool-using agents is exactly the missing rung between demos and dependable orchestration.
- **Link:** https://arxiv.org/abs/2607.19865

## 6. SLPO: Scaling Latent Reasoning via a Surrogate Policy
- **Authors:** Runyang You, Zhiyuan Liu, Yongqi Li, Wenjie Li
- **Summary:** SLPO enables outcome-reward RL for latent-reasoning models by introducing a surrogate policy density over hidden reasoning trajectories together with a correctness-supervised stopping criterion. This lets the trainer allocate compute per-example with variable reasoning horizons and yields Pass@k improvements over fixed-length latent baselines. The method makes RL feasible on models that reason in continuous latents rather than emitted chain-of-thought tokens.
- **Theme:** Core LLM advances (reasoning + RL fine-tuning)
- **Relevance:** Latent reasoning + outcome-reward RL is one of the more speculative but influential threads in the reasoning space right now.
- **Link:** https://arxiv.org/abs/2607.19691

## 7. ISO: An RLVR-Native Optimization Stack
- **Authors:** Hanqing Zhu, Wenyan Cong, Zhizhou Sha, et al.
- **Summary:** ISO is a fixed-spectrum optimiser designed specifically for RL-with-verifiable-rewards training of LLMs, exploiting spectral inheritance to keep updates stable across large batches. A notable knock-on effect is that model merging works without post-merge fine-tuning data, and convergence accelerates over standard AdamW-based RLVR pipelines. The paper argues that RLVR deserves an optimiser tuned to its gradient geometry rather than reusing pretraining defaults.
- **Theme:** Core LLM advances (fine-tuning / RL for reasoning)
- **Relevance:** Systems-y RLVR contribution with concrete efficiency numbers — the kind of paper that changes recipes if it holds up.
- **Link:** https://arxiv.org/abs/2607.19331

## 8. H^2SD: Hybrid Hindsight Self-Distillation
- **Authors:** Qiye Cai, Yichuan Ma, Linyang Li, et al.
- **Summary:** H^2SD applies distinct self-distillation teachers to successful versus failed rollouts inside an RLVR loop, converting sparse trajectory-level rewards into denser token-level supervision. The hybrid signal stabilises optimisation and reduces the reward-sparsity pathologies typical of long-horizon reasoning tasks. Reported gains come from letting failed trajectories still teach the policy something without collapsing behaviour.
- **Theme:** Core LLM advances (RL fine-tuning for reasoning)
- **Relevance:** Turning failed rollouts into useful gradient signal is a live problem for anyone doing RLVR on reasoning traces.
- **Link:** https://arxiv.org/abs/2607.18955

## 9. Stale but Stable: Staleness-Adaptive Trust Regions for Stabilizing Asynchronous Reinforcement Learning
- **Authors:** Junyao Yang, Yucheng Shi, Zongxia Li, et al.
- **Summary:** The paper diagnoses instability in asynchronous LLM RL as a function of training-versus-inference policy drift ("staleness") and proposes trust regions whose size adapts to a measured staleness signal. This lets asynchronous pipelines keep their throughput advantage without the divergent updates that usually force synchronous training. Empirically the method preserves gains from decoupled actor–learner setups on reasoning workloads.
- **Theme:** Core LLM advances (RL fine-tuning infrastructure)
- **Relevance:** Async RL infra matters as reasoning-model RL runs get longer; a practical stabilisation trick.
- **Link:** https://arxiv.org/abs/2607.18722

## 10. Two-Level Meta-Rubrics for Evaluating Open-Ended Generation: GAMUT, a Benchmark for Factual Completeness
- **Authors:** Xilun Chen, Zhaleh Feizollahi, Ross Goodwin, et al.
- **Summary:** GAMUT introduces a two-level rubric structure — global evaluation criteria refined into per-question sub-rubrics — for scoring factual completeness of long-form multimodal answers. The accompanying 1,813-question benchmark is designed so that "did the answer cover the required facts" is checkable rather than vibes-based. The framework is a step toward reusable judge scaffolds for open-ended agentic outputs.
- **Theme:** Core LLM advances (agentic-system / LLM evaluation)
- **Relevance:** Evaluation is the bottleneck on both reasoning and agent research; meta-rubric approaches are exactly what agent benchmarks need.
- **Link:** https://arxiv.org/abs/2607.19322

1. **StreamMemBench: Streaming Evaluation of Agent Memory for Future-Oriented Assistance**
   - Authors: Guanming Liu, et al.
   - Summary: Introduces a streaming benchmark that probes whether an agent can convert past interactions and stored facts into proactive, future-oriented help, rather than just answer one-shot recall queries. The benchmark feeds a continuous event stream and measures whether the agent later surfaces the right memory at the right moment. Findings expose a gap between current memory modules and the predictive use of memory that real assistants need.
   - Theme: Memory for LLMs and agents
   - Relevance: Direct, evaluation-focused contribution on agent long-term/working memory — a category the digest explicitly prioritises and one with surprisingly few rigorous benchmarks.
   - Link: https://arxiv.org/abs/2606.14571

2. **GitOfThoughts: Version-Controlled Reasoning and Agent Memory You Can Replay, Diff, and Merge**
   - Authors: Pavan C Shekar, Abhishek H S, Aswanth Krishnan
   - Summary: Proposes treating an agent's reasoning trace and memory as a versioned artifact, so chains of thought become replayable, diffable, and mergeable across runs and agents. The paper tests several substrate formats and reports that memory only helps above a "copyability" similarity threshold — i.e. it reliably accelerates answer retrieval but does not transfer methods to truly novel problems. The result reframes when persistent agent memory is worth the engineering cost.
   - Theme: Memory for LLMs and agents
   - Relevance: A fresh, mechanism-level take on episodic / procedural memory with a clear empirical limit — useful counter-evidence to "more memory is always better" hype.
   - Link: https://arxiv.org/abs/2606.14470

3. **MUSE: Agentic 3D Scene Authoring via Memory-Grounded Incremental Requirement Satisfaction**
   - Authors: Ruijie Xu, et al.
   - Summary: A multi-agent 3D-scene authoring system that splits work across specialised roles (planner, layout, asset, critic) and grounds them in three explicit memory stores — Working Memory, Scene Memory, and Skill Memory — so the team can satisfy edits incrementally without re-deriving prior decisions. The memory split is the core contribution: it lets agents reuse skills and respect prior layout commitments under iterative user requests. Results show better consistency across editing turns than monolithic baselines.
   - Theme: Memory for LLMs and agents (also multi-agent orchestration)
   - Relevance: Concrete instantiation of differentiated memory types inside a working multi-agent system — touches both top themes at once.
   - Link: https://arxiv.org/abs/2606.14168

4. **AgentSpec: Understanding Embodied Agent Scaffolds Through Controlled Composition**
   - Authors: Jixuan Chen, Jianzhi Shen, Haoqiang Kang, et al.
   - Summary: Defines a modular schema for agent scaffolds (perception, memory, reasoning, reflection, action, optional learning) with standardised interfaces, then runs controlled swap-in/swap-out studies to ask which module choices actually matter. The headline finding is that scaffold-level compatibility — how modules interact — dominates the strength of any single module in isolation. This argues against benchmarking individual components in a vacuum.
   - Theme: Multi-agent frameworks and orchestration
   - Relevance: A rare ablation-grade study of agent architectures themselves; the kind of "what actually drives agent performance" evidence the field needs.
   - Link: https://arxiv.org/abs/2606.14674

5. **tap: A File-Based Protocol for Heterogeneous LLM Agent Collaboration**
   - Authors: Minseo Kim
   - Summary: Proposes a minimal, transport-agnostic protocol where heterogeneous agents (e.g. Claude and Codex) cooperate on a shared codebase by exchanging files in a conventional directory, instead of sharing a runtime or message bus. Reported experiments show cross-vendor agent pairs catching defects in 69.8% of code reviews, demonstrating useful collaboration without bespoke integration. The simplicity is the point: it's a "lowest common denominator" alternative to bespoke A2A/MCP wiring.
   - Theme: Multi-agent frameworks and orchestration
   - Relevance: Practical, deployable take on the agent-to-agent communication problem; complements the heavier orchestration frameworks below.
   - Link: https://arxiv.org/abs/2606.14445

6. **HarnessX: A Composable, Adaptive, and Evolvable Agent Harness Foundry**
   - Authors: Tingyang Chen, Shuo Lu, Kang Zhao, et al.
   - Summary: A "foundry" for agent harnesses that treats prompts, tools, memory, and control flow as composable primitives and then evolves them using execution traces from real runs. Across five benchmarks the authors report an average +14.5% performance gain over fixed harnesses. The contribution is showing that the harness — not just the underlying LLM — can be programmatically improved from trace data.
   - Theme: Multi-agent frameworks and orchestration
   - Relevance: Methods-focused agent-framework paper with concrete numbers; aligns with the "evolvable orchestration" direction the digest tracks.
   - Link: https://arxiv.org/abs/2606.14249

7. **Towards Direct Latent-Space Synthesis for Parallel Branches**
   - Authors: Shikun Liu, et al.
   - Summary: A coordination mechanism for parallel-worker / synthesizer agent topologies in which the synthesizer consumes the workers' KV caches directly in latent space, instead of re-ingesting their text outputs. Reported speedups range from 2.5× to 11× across math, science, and code tasks with no quality loss, by removing the redundant re-prompting step. This is an architectural answer to a known cost driver in multi-branch agent systems.
   - Theme: Multi-agent frameworks and orchestration (efficiency)
   - Relevance: Targets the practical bottleneck — token re-ingestion — in parallel agent ensembles; the kind of systems-level insight that scales orchestration.
   - Link: https://arxiv.org/abs/2606.14672

8. **From Chatbot to Digital Colleague: The Paradigm Shift Toward Persistent Autonomous AI**
   - Authors: Yongheng Zhang, Ziang Liu, Jiaxuan Zhu, et al.
   - Summary: A position paper arguing that the next platform shift is from request-response chatbots to persistent agents with their own workspaces, skills, verification loops, and governance layers. It frames advanced reasoning, durable state, and self-improvement as co-requirements rather than independent research threads, and sketches the architectural commitments each implies. Useful as a synthesis read across the memory and multi-agent themes.
   - Theme: Multi-agent frameworks and orchestration (with memory implications)
   - Relevance: Surveys cut against the "method paper only" rule, but this one explicitly ties together persistence, memory, and multi-agent governance — directly on-theme.
   - Link: https://arxiv.org/abs/2606.14502

9. **SIMMER: Benchmarking Latent Failures in LLM Planning**
   - Authors: (multiple authors)
   - Summary: Constructs a benchmark for "latent" planning failures — cases where the model produces a plausible-looking plan that silently fails to satisfy stated constraints — and characterises how often current LLMs exhibit them. The benchmark separates outcome correctness from plan validity, which most prior LLM-planning evaluations conflate. Results indicate that surface-level plan quality substantially overstates true planning competence.
   - Theme: Core LLM advances (planning evaluation)
   - Relevance: Evaluation-side contribution for agentic-system planning — feeds directly into how we should be measuring orchestration upgrades.
   - Link: https://arxiv.org/abs/2606.14574

10. **CacheRL: Multi-Turn Tool-Calling Agents**
    - Authors: (multiple authors)
    - Summary: Introduces an RL fine-tuning recipe for multi-turn tool-calling agents that explicitly rewards efficient reuse of prior tool outputs (a "cache"), rather than treating each turn as independent. The training signal pushes the agent to remember intermediate results within an episode and avoid redundant tool calls. Reported gains target both accuracy and latency on multi-turn tool benchmarks.
    - Theme: Core LLM advances (agentic fine-tuning)
    - Relevance: Direct fine-tuning method for tool-use agents that operationalises within-episode working memory — bridges core-LLM and memory themes.
    - Link: https://arxiv.org/abs/2606.14179

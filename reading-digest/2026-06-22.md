## 1. AtomMem: Building Simple and Effective Memory System for LLM Agents via Atomic Facts
- **Authors:** Yanyu Yao, Shangze Li, Zhi Zheng, et al.
- **Summary:** AtomMem proposes a long-term memory layer for multi-session LLM agents that decomposes a user's interaction history into "atomic facts" extracted by a dedicated Fact Executor, then organises them into hierarchical event structures plus per-attribute temporal profiles, and serves them at retrieval time through an associative memory graph. The design targets the two recurring failure modes of agent memory — bloated, low-value representations and unstable updates — by making the storage unit small, typed, and time-stamped. The authors report SOTA results on the LoCoMo long-conversation benchmark across the standard reasoning task suite.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Dedicated, end-to-end memory architecture for LLM agents with explicit episodic + profile structure and a clean benchmark win — a near-perfect fit for the top theme.
- **Link:** https://arxiv.org/abs/2606.19847

## 2. Execution-State Capsules: Graph-Bound Execution-State Checkpoint and Restore
- **Authors:** Liang Su
- **Summary:** Argues that the KV cache is only one positional fragment of an LLM agent's actual execution state, and introduces "execution-state capsules" — graph-bound snapshots of the entire restorable boundary (KV, recurrent state, convolution state, MTP state, metadata) over contiguous static buffers. A FlashRT runtime makes capsule snapshot/restore byte-exact and sub-millisecond on RTX 5090, Jetson Thor, and DGX Spark; TTFT speedup over cold prefill grows from 3.9× at 2k tokens to 27× at 16k tokens, and a KV-only ablation breaks correctness — showing the recurrent state was load-bearing. The framing repositions reuse from token-addressed KV fragments to graph-bound execution boundaries, which is the right primitive for branching/resetting interactive agents.
- **Theme:** Memory for LLMs and agents (state/context management)
- **Relevance:** Pushes a real systems-level memory primitive for agentic LLM serving that goes beyond KV-cache thinking — directly relevant to anyone building branching/checkpoint-aware agent stacks.
- **Link:** https://arxiv.org/abs/2606.20537

## 3. UltraQuant: 4-bit KV Caching for Context-Heavy Agents
- **Authors:** Inesh Chakrabarti, David Limpus, Aditi Ghai Rana, et al.
- **Summary:** Frames 4-bit KV-cache compression specifically for multi-round agentic workloads — long shared prefixes reused across many short turns — and lays out the design choices needed to keep quality (asymmetric K/V, Walsh-Hadamard rotation, QJL removal, block-scale variants). Pairs this with serving-side work on AMD GPUs: optimised decode-attention kernels and an FP4 approximation path using FP8 queries, FP4 KV, UE8M0 group scales, and scaled-MFMA on CDNA4. Reports 3.47× P50 TTFT reduction in cache-pressured late rounds and 1.63× higher output throughput over an FP8 KV baseline.
- **Theme:** Memory for LLMs and agents (KV/context memory at serving time)
- **Relevance:** The KV cache *is* the working memory of a deployed agent — a quantisation recipe explicitly tuned for multi-turn agent traffic is directly on-theme.
- **Link:** https://arxiv.org/abs/2606.20474

## 4. Marginal Advantage Accumulation for Memory-Driven Agent Self-Evolution
- **Authors:** Mingyu Yang, Keye Zheng, Congchao Cheng, et al.
- **Summary:** Targets batch-style trace distillation for self-evolving agents: the same memory operation can receive contradictory feedback across batches, and existing methods can't separate stably effective operations from accidental hits. MAA formalises the missing structural properties (alignability and comparability), builds differential signals that survive across batches, accumulates signed per-operation evidence via EMA, and merges semantically identical operations for cross-batch traceability. As a post-processing layer it wins 14/16 settings across 4 benchmarks and 4 models, matching or beating online alternatives while cutting optimisation-phase tokens ~75%.
- **Theme:** Memory for agents (operation-level credit assignment for memory writes)
- **Relevance:** Treats the agent's *memory operations* as first-class learnable targets — a concrete recipe for the "memory-driven self-evolution" thread that several recent agent papers gesture at without operationalising.
- **Link:** https://arxiv.org/abs/2606.20475

## 5. SIGMA: Skill-Incidence Graphs for Compositional Multi-Agent Design
- **Authors:** Kun Zeng, Yu Huo, Siyu Zhang, et al.
- **Summary:** Argues that graph-based multi-agent designers have been over-investing in communication-topology optimisation while leaving the agent nodes themselves closed-set, which blocks generalisation to unseen capability combinations. SIGMA instead treats each agent as a task-conditioned bundle of reusable skills: it predicts a skill–agent incidence matrix, composes agent node embeddings from selected skills, and decodes the communication topology over those constructed agents, with skill-specific mailboxes routing messages to the assigned capabilities at runtime. Across six reasoning/coding benchmarks and three base LLMs, SIGMA beats the strongest non-compositional topology baseline (CARD) by ~2 points on average and shows the smallest performance drop on unseen skill libraries.
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** Adds a new design axis — compositional node construction — to the multi-agent stack, with an honest comparison against communication-topology baselines. Strong fit for the orchestration theme.
- **Link:** https://arxiv.org/abs/2606.19758

## 6. LedgerAgent: Structured State for Policy-Adherent Tool-Calling Agents
- **Authors:** Md Nayem Uddin, Amir Saeidi, Eduardo Blanco, Chitta Baral
- **Summary:** Diagnoses two failure modes in standard tool-calling agents: implicit state reconstruction from prompt every turn, and syntactically valid but policy-violating tool calls. LedgerAgent maintains task state in a separate explicit ledger, renders it back into the prompt, and uses it to pre-check state-dependent policy constraints before any environment-changing tool call executes — blocking violations rather than detecting them after the fact. Across four customer-service domains and a mix of open- and closed-weight models, it lifts average pass^k over prompt-based tool-calling, with the largest gains under the stricter multi-trial consistency metrics.
- **Theme:** Multi-agent frameworks & orchestration (agent state + policy adherence)
- **Relevance:** Practical inference-time pattern for tool-using agents — directly addresses the "agent state management" gap and is easy to layer on top of existing harnesses.
- **Link:** https://arxiv.org/abs/2606.20529

## 7. Beyond Global Replanning: Hierarchical Recovery for Cross-Device Agent Systems (H-RePlan)
- **Authors:** Shu Yao, Yuhua Luo, Qian Long, et al.
- **Summary:** Existing multi-device agent systems can decompose and assign subtasks but recover from runtime failures coarsely (retry, reassign, or replan globally), without distinguishing repairable local failures from those that truly need cross-device replanning. H-RePlan gives each device a set of interchangeable execution strategies and separates device-local strategy recovery from orchestrator-level global replanning via a compact cross-layer failure abstraction over a unified API/CLI/GUI interface. They release HeraBench, a fault-injected cross-device (Linux + Android) benchmark, and report higher completion, instruction adherence, and perfect-pass rates over single-strategy and coarse multi-device baselines while reducing tokens for reliable end-to-end success.
- **Theme:** Multi-agent frameworks & orchestration (cross-device coordination + recovery)
- **Relevance:** Concrete two-tier orchestration pattern with a fault-injection benchmark — exactly the kind of orchestration-protocol work the digest tracks.
- **Link:** https://arxiv.org/abs/2606.20487

## 8. Heterogeneous LLM Debate Under Adversarial Peers: Honest Gains, Replacement Costs, and Resilience
- **Authors:** Prashanti Nilayam, Kiran Kumar Ramanna, Prashil Tumbade, Sankalp Nayak
- **Summary:** Measures heterogeneity in LLM debate not by accuracy but by *revision behaviour* — how often honest agents change their answers, and whether that change is corrective or harmful — across matched homogeneous, honest-mixed, adversarial-mixed, and contaminated panels over four model families and three reasoning benchmarks. An honest cross-family peer sharply lowers harmful revisions (e.g. Llama-3.1-70B defenders on MATH-hard: 89% → 35%), an adversarial peer roughly reverses it, and crucially, when an adversary is *already* present, an added honest heterogeneous peer cuts the rate at which initially-correct answers flip from 31% to 6%. The framing reframes heterogeneity as both an attack surface and, conditionally, a defence.
- **Theme:** Multi-agent frameworks & orchestration (communication protocols + safety)
- **Relevance:** Rigorous behavioural analysis of how heterogeneity shapes multi-agent communication outcomes — useful empirical grounding for anyone designing debate-style agent panels.
- **Link:** https://arxiv.org/abs/2606.19826

## 9. Beyond Entropy: Learning from Token-Level Distributional Deviations for LLM Reasoning
- **Authors:** Xuanzhi Feng, Zhengyang Li, Zeyu Liu, et al.
- **Summary:** Targets the classic RLVR instability dichotomy — entropy collapse vs entropy explosion — by shifting the optimisation signal from scalar entropy to the distributional shape of token logits. The Independent Combinatorial Tokens (ICT) framework uses Jensen–Shannon divergence between token logits to identify "branching" tokens and restricts updates to those, with a Shannon + second-order Rényi entropy analysis showing this jointly regulates uncertainty and concentration. On Qwen2.5 (0.5B/1.5B/7B), updating only the top 10% of tokens yields +4.58% average pass@4 over GRPO, 20-Entropy, and STAPO across seven math/commonsense/Olympiad benchmarks (max +14.9%).
- **Theme:** Core LLM advances (reasoning / RL post-training stability)
- **Relevance:** Concrete fix for a known RLVR failure mode and a cheap, drop-in selector for the agent-RL stack many memory/multi-agent papers in this digest implicitly rely on.
- **Link:** https://arxiv.org/abs/2606.19771

## 10. ORAgentBench: Can LLM Agents Solve Challenging Operations Research Tasks End to End?
- **Authors:** Jiajun Li, Mingshu Cai, Yixuan Li, et al.
- **Summary:** Introduces a 107-task, human-reviewed benchmark for end-to-end OR agentic work where each task ships a natural-language brief, multi-file data, configuration artefacts, and a required submission schema, and is graded by hidden validators on schema validity, hard-constraint feasibility, and normalised objective quality. Across fourteen frontier agent-model configurations, the best agent passes only 35.51% of all tasks and 20.59% of hard tasks; failure analysis attributes most errors to missed operational rules, brittle formulations, weak feasibility construction, and insufficient solution improvement. OR-specific procedural skills lift hard-task feasibility but don't reliably improve solution quality or pass rate.
- **Theme:** Core LLM advances (agentic-system evaluation)
- **Relevance:** Adds a domain-grounded, execution-graded harness to the agentic-eval stack — useful counterpart to the more code/UI-centric benchmarks currently dominating the agentic-eval conversation.
- **Link:** https://arxiv.org/abs/2606.19787

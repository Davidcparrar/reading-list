# Reading digest — 2026-06-12

Found 10 paper(s) in the last 48h window.

## 1. EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments
**Authors:** Jundong Xu, Qingchuan Li, Jiaying Wu, et al.

The paper introduces EvoArena, a benchmark targeting LLM agents that must operate as their world changes underneath them, paired with EvoMem — a memory paradigm built around progressive, structured updates rather than a flat append-only log. The combination is used to study how agents' memory representations should evolve to remain robust as facts shift, tasks drift, and prior context becomes stale.

**Theme:** Memory for LLMs and agents
**Relevance:** Directly proposes a memory paradigm and an evaluation harness for memory-evolution in agents — a near-perfect fit for the top theme.
**Link:** https://arxiv.org/abs/2606.13681

## 2. Toward Generalist Autonomous Research via Hypothesis-Tree Refinement (Arbor)
**Authors:** Jiajie Jin, Yuyang Hu, Kai Qiu, et al.

Arbor frames autonomous research as the construction and refinement of a persistent hypothesis tree where nodes carry evidence and intermediate insights, so an agent can revisit, branch, and prune prior reasoning across long timeframes. The persistent knowledge-tree structure is the system's long-term memory and the substrate that coordinates iterative experimentation.

**Theme:** Memory for LLMs and agents (also touches multi-agent orchestration)
**Relevance:** Persistent, structured long-term memory is the load-bearing idea — strong fit for memory-for-agents.
**Link:** https://arxiv.org/abs/2606.11926

## 3. MiniMax Sparse Attention
**Authors:** Xunhao Lai, Weiqi Xu, Yufeng Yang, et al.

Proposes a blockwise sparse attention scheme that the authors claim cuts per-token attention compute by ~28× at 1M-token contexts while keeping quality competitive. The mechanism is positioned as enabling practical million-token working memory inside the model, rather than offloading to external retrieval.

**Theme:** Memory for LLMs and agents (context/state management)
**Relevance:** Long-context working memory is the second pillar of the memory theme; the efficiency numbers make this method-paper worth tracking.
**Link:** https://arxiv.org/abs/2606.13392

## 4. On Subquadratic Architectures: From Applications to Principles
**Authors:** Anamaria-Roberta Hartl, Levente Zólyomi, David Stap, et al.

A controlled comparison of xLSTM, Mamba-2, and Gated DeltaNet on hard sequence tasks, attributing xLSTM's edge to stronger state-tracking and accumulation behavior in its recurrent state. The framing is principles-first: which architectural choices give a recurrent state the capacity to act as durable working memory.

**Theme:** Memory for LLMs and agents (model-internal state as memory)
**Relevance:** Directly analyzes how recurrent/state-space models implement working memory — a useful complement to the attention-side memory papers above.
**Link:** https://arxiv.org/abs/2606.12364

## 5. EurekAgent: Agent Environment Engineering is All You Need For Autonomous Scientific Discovery
**Authors:** Amy Xin, Jiening Siow, Junjie Wang, et al.

EurekAgent argues that the bottleneck for autonomous science isn't smarter agents but better engineered environments, and proposes a structure of explicit permissions, persistent artifacts, budgets, and human oversight hooks that the agents coordinate over. The system is presented as a multi-agent orchestration pattern where the environment, not the prompt, is what shapes behavior.

**Theme:** Multi-agent frameworks & orchestration
**Relevance:** A concrete orchestration design (artifacts/permissions/budgets) — exactly the kind of multi-agent framework architecture the digest cares about.
**Link:** https://arxiv.org/abs/2606.13662

## 6. InterleaveThinker: Reinforcing Agentic Interleaved Generation
**Authors:** Dian Zheng, Harry Lee, Manyuan Zhang, et al.

A multi-agent pipeline pairing a planner agent with a critic agent under reinforcement learning to produce interleaved text-and-image outputs across different image generators. The contribution is the planner/critic coordination protocol and its RL training loop, more than the specific generation task it lands on.

**Theme:** Multi-agent frameworks & orchestration
**Relevance:** Planner/critic role assignment with RL is a clean instance of agent communication protocols — relevant to theme (2) even though the application is generative.
**Link:** https://arxiv.org/abs/2606.13679

## 7. Agentic Environment Engineering for Large Language Models: A Survey
**Authors:** Jiachun Li, Zhuoran Jin, Tianyi Men, et al.

A survey covering the lifecycle of agentic environments — modeling, synthesis, evaluation, and agent-environment co-evolution — and how today's multi-agent stacks treat the environment as a first-class design surface. Useful as a map of where current orchestration work converges and where it forks.

**Theme:** Multi-agent frameworks & orchestration
**Relevance:** Survey rather than method, but the taxonomy and references give a fast on-ramp to the orchestration landscape.
**Link:** https://arxiv.org/abs/2606.12191

## 8. MaxProof: Scaling Mathematical Proof with Generative-Verifier RL and Population-Level Test-Time Scaling
**Authors:** Jiacheng Chen, Xinyu Zhang, Shunkai Zhang, et al.

MaxProof couples a generator with a learned verifier under RL and adds a population-level test-time scaling strategy — running and aggregating across many proof attempts — to push competition-math performance to a reported gold-medal threshold. The recipe is generator/verifier RL plus structured search at inference rather than a new base model.

**Theme:** Core LLM advances (reasoning)
**Relevance:** Reasoning-focused RL with explicit verifier and test-time scaling — strong fit for the "core LLM advances relevant to agents" bucket.
**Link:** https://arxiv.org/abs/2606.13473

## 9. Redesign Mixture-of-Experts Routers with Manifold Power Iteration
**Authors:** Songhao Wu, Ang Lv, Ruobing Xie, et al.

Proposes aligning each MoE router row with the principal singular directions of the corresponding expert's weight matrix via Manifold Power Iteration, so routing decisions are matched to where each expert actually has capacity. The result is presented as a cleaner geometric foundation for routing and better downstream MoE quality.

**Theme:** Core LLM advances (architecture)
**Relevance:** MoE routing is increasingly load-bearing for frontier-scale agentic models; a principled router-redesign is worth tracking.
**Link:** https://arxiv.org/abs/2606.12397

## 10. FORT-Searcher: Synthesizing Shortcut-Resistant Search Tasks for Training Deep Search Agents
**Authors:** Jia Deng, Yimeng Chen, Xiaoqing Xiang, et al.

Builds a training-data pipeline that explicitly identifies four common shortcut risks deep-search agents exploit (e.g., guessable answers from titles), then synthesizes tasks that close each shortcut. The contribution is methodological — a recipe for training data that forces actual multi-hop search behavior.

**Theme:** Core LLM advances (agentic-system training/evaluation)
**Relevance:** Shortcut-resistant training data is a recurring blocker in agentic-system evaluation; the diagnosis + recipe makes this useful beyond the specific system.
**Link:** https://arxiv.org/abs/2606.12087

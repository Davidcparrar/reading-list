1. **Are We Ready For An Agent-Native Memory System?**
   - Authors: Wei Zhou, Xuanhe Zhou, Shaokun Han, et al.
   - Summary: The authors run a systematic data-management style audit of 12 representative agent memory systems (long-term, episodic, working-memory, retrieval-augmented variants) across a battery of workloads and datasets, asking whether today's stacks can serve as a true "agent-native" memory layer. The study frames memory as a first-class data system problem (storage, indexing, retrieval, freshness, cost) rather than an RAG add-on, and reports where each system breaks under realistic agent workloads. The takeaway is a gap analysis pointing to where an agent-native memory system would need to differ from current RAG/vector-store designs.
   - Theme: (1) memory for LLMs and agents.
   - Relevance note: Directly evaluates the memory landscape for agents — exactly the top-priority track.
   - Link: https://arxiv.org/abs/2606.24775

2. **Qwen-Image-Agent: Bridging the Context Gap in Real-World Image Generation**
   - Authors: Zekai Zhang, Jiahao Li, Jie Zhang, et al.
   - Summary: The paper names the "Context Gap" between an underspecified user prompt and the richer context a T2I model actually needs, then proposes a unified agentic stack (Qwen-Image-Agent) that combines Context-Aware Planning with Context Grounding over plan/reason/search/memory/feedback modules to progressively assemble that context. A companion benchmark (IA-Bench) is introduced to score each of the four agent capabilities — Plan, Reason, Search, Memory — separately. Reported gains over strong baselines on IA-Bench, Mindbench, and WISE-Verified.
   - Theme: (2) multi-agent frameworks & orchestration, with a strong (1) memory component.
   - Relevance note: One of the few new agentic frameworks this window that builds memory in as a first-class module rather than as RAG glue.
   - Link: https://arxiv.org/abs/2606.26907

3. **Confidence-Aware Tool Orchestration for Robust Video Understanding (Robust-TO)**
   - Authors: Yangfan He, Yujin Choi, Jaehong Yoon
   - Summary: Robust-TO is an agent framework that routes a question to heterogeneous visual perception tools under a unified evidence interface, where each tool returns a typed result plus a calibrated reliability score; a three-tier (high/medium/low) synthesis stage and a confidence-cost GRPO reward weight those signals during reasoning. The motivating "Blind Trust Problem" is that frontier video LLMs lose 15–30 pp accuracy under realistic perturbations without realising their inputs were degraded. On two benchmarks the system beats the strongest open-source baseline by 10.6 pp clean and 5.8 pp under corruption, and exceeds Gemini-2.5-Pro on clean.
   - Theme: (2) multi-agent frameworks & orchestration.
   - Relevance note: A concrete tool-orchestration pattern (typed evidence + calibrated confidence) that generalises beyond the video setting.
   - Link: https://arxiv.org/abs/2606.26904

4. **Autodata: An agentic data scientist to create high quality synthetic data**
   - Authors: Ilia Kulikov, Chenxi Whitehouse, Tianhao Wu, et al.
   - Summary: Autodata casts a Meta-led team's "agentic data scientist" as the operator that designs, generates, and quality-controls synthetic training data through meta-optimisation across multiple task domains, rather than relying on a fixed prompt-engineered pipeline. The agent iterates on data recipes against downstream training signals, with the goal of producing fine-tuning corpora at higher quality than hand-crafted prompts. Reported across several task domains.
   - Theme: (2) multi-agent frameworks & orchestration (agent-driven workflow), with relevance to (3) for fine-tuning data.
   - Relevance note: Concrete agentic system that closes the loop between data generation and downstream training; potentially reusable as a recipe.
   - Link: https://arxiv.org/abs/2606.25996

5. **OPID: On-Policy Skill Distillation for Agentic Reinforcement Learning**
   - Authors: Shuo Yang, Jinyang Wu, Zhengxi Lu, et al.
   - Summary: OPID extracts hierarchical "skills" directly from completed on-policy trajectories — episode-level skills for global workflows and step-level skills for local critical decisions — and routes them via a critical-first mechanism back into the interaction history; the resulting log-prob shift becomes a token-level self-distillation advantage combined with outcome reward. This avoids the cost and distribution-mismatch of external skill memories or retrieved privileged context used by prior skill-conditioned methods. Gains over outcome-only RL and prior skill-distillation baselines are reported on ALFWorld, WebShop, and search QA.
   - Theme: (2) multi-agent / agentic frameworks; touches (1) memory via hierarchical skill stores.
   - Relevance note: A clean, code-released agentic-RL recipe whose "hindsight skills" idea also speaks to the agent-memory track.
   - Link: https://arxiv.org/abs/2606.26790

6. **The Verification Horizon: No Silver Bullet for Coding Agent Rewards**
   - Authors: Binghai Wang, Chenlong Zhang, Dayiheng Liu, et al.
   - Summary: The paper argues that for modern coding agents the binding constraint has flipped from generation to verification, and characterises verification signal quality along three axes — scalability, faithfulness, robustness — that cannot all be maximised at once. It studies four concrete reward constructions (test verifier, rubric verifier, user-as-verifier, automated agent verifier) across task types and policy capability levels, showing how targeted verifier design reduces reward hacking and lifts task completion. The headline claim is that no fixed reward survives a stronger policy and verification must co-evolve with the generator.
   - Theme: (3) core LLM / agentic-system evaluation.
   - Relevance note: Useful framing piece for anyone building RL pipelines for coding agents.
   - Link: https://arxiv.org/abs/2606.26300

7. **Why Multi-Step Tool-Use Reinforcement Learning Collapses and How Supervisory Signals Fix It**
   - Authors: Yupu Hao, Zhuoran Jin, Huanxuan Liao, Kang Liu, Jun Zhao
   - Summary: The authors trace catastrophic collapse in tool-use RL to unexpected probability spikes on specific control tokens that wreck the structured invocation format while leaving the underlying capability intact, then sweep a matrix of supervisory signals (off-policy SFT, hint-based guidance, erroneous-example supervision, etc.) under synchronous and interleaved schedules. Interleaving SFT with RL recovers stability but degrades format/content OOD behaviour, exposing a trade-off, and the authors analyse learning-rate sensitivity across settings. Code is released as a "Tool-RL-Box".
   - Theme: (3) core LLM advances for agentic systems / (2) tool-using agents.
   - Relevance note: Diagnostic-style paper that's actionable if you're training a tool-using agent and seeing format breakdown.
   - Link: https://arxiv.org/abs/2606.26027

8. **iLLaDA: Improved Large Language Diffusion Models**
   - Authors: Shen Nie, Qiyang Min, Shaoxuan Xu, et al.
   - Summary: iLLaDA is an 8B masked-diffusion language model with fully bidirectional attention, presenting a refined recipe for scaling diffusion LMs against autoregressive baselines. Improvements are reported across general, mathematical, and code benchmarks. The result is meant as a serious data point on whether diffusion can be a viable LLM backbone at the 8B scale.
   - Theme: (3) core LLM advances.
   - Relevance note: One of the cleaner non-AR LLM efforts to land this window; useful prior on whether to bet on diffusion LMs for agent backbones.
   - Link: https://arxiv.org/abs/2606.25331

9. **JetSpec: Breaking the Scaling Ceiling of Speculative Decoding with Parallel Tree Drafting**
   - Authors: Lanxiang Hu, Zhaoxiang Feng, Yulun Wu, et al.
   - Summary: JetSpec resolves the "causality-efficiency dilemma" in head-based speculative decoding by training a causal parallel draft head over fused hidden states of a frozen target model, producing tree-shaped candidates whose scores stay aligned with the target's autoregressive factorisation. This lets larger draft budgets translate into longer accepted prefixes instead of plateauing. On Qwen3 dense and MoE models the authors report up to 9.64× on MATH-500 and 4.58× on open-ended chat, plus vLLM-integrated latency gains under realistic serving loads.
   - Theme: (3) core LLM advances (inference efficiency).
   - Relevance note: Concrete, code-released speculative-decoding upgrade with vLLM integration — relevant for anyone serving long-running agent loops.
   - Link: https://arxiv.org/abs/2606.18394

10. **In-Context World Modeling for Robotic Control (ICWM)**
    - Authors: Siyin Wang, Junhao Shi, Senyu Fei, et al.
    - Summary: ICWM treats system identification as in-context adaptation: a VLA policy reads a short history of self-generated, task-agnostic interactions, implicitly infers the world dynamics of the current setup, and then performs the actual task without parameter updates. This separates "what task to do" (the classic ICL signal) from "how this system behaves" (the new in-context signal). Reported gains over standard VLA baselines on novel camera viewpoints in simulation and on real robots.
    - Theme: (3) core LLM advances (in-context state management), with adjacency to (1) memory/context.
    - Relevance note: The cleanest "context as memory" idea this window — uses the context window itself as the place where a policy stores and uses environment state.
    - Link: https://arxiv.org/abs/2606.26025

## 1. SearchOS-V1: Towards Robust Open-Domain Information-Seeking Agent Collaboration

- **Authors:** Yuyao Zhang, Junjie Gao, Zhengxian Wu, et al.
- **Summary:** Proposes a multi-agent framework for open-domain information seeking that externalizes shared search progress into typed, persistent memory structures — a Frontier Task queue, an Evidence Graph, a Coverage Map, and a Failure Memory. Instead of coordinating via loose message pools, sibling agents read and write these structured stores as they research, which the authors argue is what keeps collaboration coherent across long horizons and repeated failed subqueries.
- **Theme:** Memory + multi-agent orchestration.
- **Relevance:** Rare direct fit for both top-priority themes at once: the whole design is *shared external memory as the coordination substrate* for a multi-agent search team.
- **Link:** https://arxiv.org/abs/2607.15257

## 2. KnowAct-GUIClaw: Know Deeply, Act Perfectly, Personal GUI Assistant with Self-Evolving Memory and Skill

- **Authors:** Yunxin Li, Jinchao Li, Shibo Su, et al.
- **Summary:** A personal GUI-agent architecture built around a self-evolving memory of past user interactions plus a growing skill library, so a phone assistant improves with usage rather than restarting each task cold. Reported gains on cross-platform mobile benchmarks come from surfacing prior traces and reusable skills at execution time, letting the agent recognize when a new task is a variant of something it already knows how to do.
- **Theme:** Memory (also touches agent architecture).
- **Relevance:** Concrete instance of long-term episodic memory + skill memory for an on-device agent — directly the "memory for agents" theme, with a working benchmark story.
- **Link:** https://arxiv.org/abs/2607.12625

## 3. RoboTTT: Context Scaling for Robot Policies

- **Authors:** Yunfan Jiang, Yevgen Chebotar, Ruijie Zheng, et al.
- **Summary:** Uses test-time training with fast weights to push a robot visuomotor policy's usable context to 8K timesteps, effectively giving it a working-memory-like handle on long demonstrations rather than relying only on short-horizon token windows. The result they report is one-shot in-context imitation and roughly an 87% jump on multi-stage tasks over shorter-context baselines.
- **Theme:** Memory / context & state management (for embodied policies).
- **Relevance:** Test-time-trained fast weights are one of the more interesting working-memory analogues showing up in policies right now; relevant to state-management questions for LLM agents too.
- **Link:** https://arxiv.org/abs/2607.15275

## 4. LongStraw: Long-Context RL Beyond 2M Tokens under a Fixed GPU Budget

- **Authors:** Changhai Zhou, Kieran Liu, Yuhua Zhou, et al.
- **Summary:** An architecture-aware execution stack that makes RL post-training feasible past 2M tokens on a fixed GPU budget, mainly by computing gradients selectively along the rollout and reworking activation/KV memory management for the long-context path. Positioned less as a new algorithm and more as the plumbing needed to keep reasoning-oriented RL from OOM-ing when rollouts get very long.
- **Theme:** Memory-adjacent (long-context state/gradient management) + core LLM (RL post-training).
- **Relevance:** Infrastructure for making very-long-context RL tractable is upstream of every long-horizon agentic-RL claim; useful reference for scaling questions.
- **Link:** https://arxiv.org/abs/2607.14952

## 5. SEED: Self-Evolving On-Policy Distillation for Agentic Reinforcement Learning

- **Authors:** Jinyang Wu, Shuo Yang, Zhengxi Lu, et al.
- **Summary:** Converts completed agent trajectories into "hindsight skills" and then distills their behavioral effects back into the policy on-policy, so the agent iteratively teaches itself rather than relying on hand-curated demonstrations. The target regime is long-horizon interactive tasks with tool use and environment feedback, where reward is sparse and imitation data is thin — exactly where standard SFT/RL loops tend to plateau.
- **Theme:** Core LLM (agentic RL, self-improvement, on-policy distillation).
- **Relevance:** Sits at the intersection of fine-tuning/PEFT-style updates and agent self-improvement, which is where a lot of current agentic-RL work is converging.
- **Link:** https://arxiv.org/abs/2607.14777

## 6. AgentCompass: A Unified Evaluation Infrastructure for Agent Capabilities

- **Authors:** Zichen Ding, Jiaye Ge, Shufan Jiang, et al.
- **Summary:** Open-source evaluation infrastructure that separates benchmarks, harnesses, and environments as pluggable pieces, so the same agent implementation can be run against 20+ existing tasks without rewriting per-benchmark glue. The stated contribution is the standardized substrate itself, aimed at making cross-agent comparisons less lossy and less bespoke.
- **Theme:** Core LLM (agentic-system evaluation).
- **Relevance:** Directly the "agentic-system evaluation" facet of the core-LLM theme; worth knowing about even just as tooling.
- **Link:** https://arxiv.org/abs/2607.13705

## 7. Demystifying On-Policy Distillation: Roles, Pathologies, and Regulations

- **Authors:** Rui Wang, Hongru Wang, Yi Chen, et al.
- **Summary:** An empirical study of what on-policy distillation is actually doing when it works, framing it as an exploration catalyst rather than pure supervision, and diagnosing failure modes such as student–teacher capacity mismatch and length exploitation. From those diagnoses the paper proposes signal-shaping regulations designed to keep distillation gains from collapsing on longer chains.
- **Theme:** Core LLM (fine-tuning / distillation for reasoning).
- **Relevance:** Complements the SEED paper above — one proposes an agentic-RL system, this one interrogates the training signal that both rely on. Useful pairing.
- **Link:** https://arxiv.org/abs/2607.13399

## 8. Partition, Prompt, Aggregate: Statistical Self-Consistency in Language Models

- **Authors:** Patrik Wolf, Thomas Kleine Buening, Andreas Krause, et al.
- **Summary:** Shows that LLMs systematically violate statistical self-consistency at the aggregate level: fine-grained subpopulation estimates elicited from the same model line up with reference data far better than the direct population-level estimate — the paper's "macro fallacy." The practical takeaway is that partition-then-prompt-then-aggregate can substantially outperform asking for the whole distribution in one shot.
- **Theme:** Core LLM (reasoning / prompting consistency).
- **Relevance:** Clean, methodologically-crisp reasoning-behavior finding with a concrete prompting prescription — the kind of result that composes into agent pipelines.
- **Link:** https://arxiv.org/abs/2607.15277

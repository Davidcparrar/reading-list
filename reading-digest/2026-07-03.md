## 1. AgenticSTS: A Bounded-Memory Testbed for Long-Horizon LLM Agents
- **Authors:** Xiangchen Cheng, Yunwei Jiang, Jianwen Sun, et al.
- **Summary:** Proposes a testbed where long-horizon LLM agents operate under an explicit bounded-memory contract: every step reassembles a prompt from typed retrieval over a stored memory store rather than appending raw transcripts, so prompt length stays constant across arbitrarily long runs. The setup exposes memory as a first-class, ablatable layer, letting the authors compare retrieval schemes and episodic vs. semantic stores under identical decision policies. Reported ablations isolate which memory design choices actually move task success on long trajectories.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Directly the top-priority theme — a benchmark that pulls memory management out of the model and makes it independently measurable.
- **Link:** https://arxiv.org/abs/2607.02255

## 2. MemSyco-Bench: Benchmarking Sycophancy in Agent Memory
- **Authors:** Zhishang Xiang, Zerui Chen, Yunbo Tang, et al.
- **Summary:** Introduces a benchmark specifically probing "memory sycophancy" — cases where an agent takes stored memory at face value and drifts from ground truth. Five tasks stress-test whether the agent can reject bad memory as evidence, respect scope limits, resolve conflicts between memory entries, track updates to superseded facts, and still use valid memory when it applies. The evaluation exposes failure modes that generic memory benchmarks miss because they only check recall, not appropriate distrust.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Complements AgenticSTS with a robustness/safety angle on memory — measures when memory is a liability, not just a resource.
- **Link:** https://arxiv.org/abs/2607.01071

## 3. BioInsight: Multi-Agent Orchestration for Interactive Biomedical Knowledge Discovery
- **Authors:** Jieyi Wang, Bingxuan Li, Nanyi Jiang, et al.
- **Summary:** A multi-agent system for biomedical discovery that replaces static, monolithic research reports with an interactive orchestration whose agents exchange typed intermediate artifacts — ranked pathway lists, evidence packets, citation-grounded analyses — rather than free-form text. The typed-artifact interface lets each agent's contribution be inspected, cited, and refined by downstream agents, and lets the user redirect the pipeline mid-run. Case studies in biomedical Q&A show that the typed hand-off reduces information loss compared to text-only agent chains.
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** A concrete architecture for structured inter-agent communication — orchestration protocols and typed artifacts are exactly the multi-agent design surface. (Note: `published_at` shows 2026-06-18 but the paper is on HF's 2026-07-02 daily list, suggesting a recent revision.)
- **Link:** https://arxiv.org/abs/2606.20997

## 4. AutoTrainess: Teaching Language Models to Improve Language Models Autonomously
- **Authors:** Zhaojian Yu, Penghao Yin, Shuzheng Gao, et al.
- **Summary:** Turns model post-training itself into an agentic loop: an LM agent drives planning, data preparation, training, evaluation, and logging through a structured agent-computer interface rather than raw shell commands. The interface exposes each training operation as a typed action the agent can compose, which the authors show is essential — the same underlying model plateaus when limited to CLI-only calls. Evaluated on the PostTrainBench suite, the interface-mediated agent beats the CLI baseline on end-to-end model-improvement runs.
- **Theme:** Multi-agent frameworks & orchestration (agent-as-operator over training tooling)
- **Relevance:** Interesting orchestration case: the agent's tools *are* the training pipeline. Relevant to how role/tool assignment shapes agent capability. (Note: `published_at` 2026-06-29, appears on 2026-07-02 daily list.)
- **Link:** https://arxiv.org/abs/2606.31551

## 5. AgenticDataBench: A Comprehensive Benchmark for Data Agents
- **Authors:** Zhaoyan Sun, Shan Zhong, Daizhou Wen, et al.
- **Summary:** A broad benchmark for LLM agents doing end-to-end data-science work, covering 15 domains plus 5 real B2B workflows and grounded in a skill taxonomy mined from Stack Overflow. The Stack-Overflow-derived skill list lets the authors quantify skill coverage — measuring which capabilities a benchmark actually exercises rather than only which tasks it contains — and to attach fine-grained per-step annotations. Baseline agent runs reveal specific skill gaps rather than only aggregate accuracy numbers.
- **Theme:** Multi-agent frameworks & orchestration (agent evaluation)
- **Relevance:** Skill-based coverage measurement is a useful angle for evaluating agent frameworks; gives evaluators a diagnostic instead of a leaderboard number.
- **Link:** https://arxiv.org/abs/2607.01647

## 6. SkillCoach: Self-Evolving Rubrics for Evaluating and Enhancing Agentic Skill-Use
- **Authors:** Jiayin Zhu, Kelong Mao, Yudong Guo, et al.
- **Summary:** Proposes rubrics that evolve during evaluation across four axes — skill selection, skill following, skill composition, and skill reflection — instead of scoring only final task success. The evolving rubric supplies a step-level process signal that flags failures which outcome-only metrics hide (e.g., an agent that succeeds on the surface but chose the wrong tool). The authors also plug the rubric back in as a training signal and report gains over outcome-only reward filtering.
- **Theme:** Multi-agent frameworks & orchestration (skill/tool-use evaluation)
- **Relevance:** Rubric-as-process-supervision is a reusable pattern for agentic evaluation and training data curation.
- **Link:** https://arxiv.org/abs/2607.01874

## 7. PACE: A Proxy for Agentic Capability Evaluation
- **Authors:** Yueqi Song, Lintang Sutawika, Jiarui Liu, et al.
- **Summary:** Builds cheap proxy benchmarks for agentic capability by selecting a small subset of instances from cheap non-agentic evaluations whose aggregate score correlates with expensive agentic-benchmark scores. Two selection strategies — target-relevance-local and globally-informative — pick complementary instances, yielding PACE-Bench, which the authors report tracks costly agentic benchmarks with far less compute per evaluation. The framing treats "which cheap eval predicts my agentic eval?" as a subset-selection problem.
- **Theme:** Multi-agent frameworks & orchestration (agent evaluation)
- **Relevance:** Cheap agent evals matter for iteration; the subset-selection framing is a general and reusable idea.
- **Link:** https://arxiv.org/abs/2607.02032

## 8. The State-Prediction Separation Hypothesis
- **Authors:** Giovanni Monea, Nathan Godey, Kianté Brantley, Yoav Artzi
- **Summary:** Argues that a single transformer stream is asked to do two conflicting jobs — predict the next token and maintain long-lived state — and proposes a variant with two parallel computation streams, one for each job. The state stream carries information forward without paying the prediction bottleneck, and vice versa; the authors report improved data and compute efficiency and 2–3 point downstream gains at matched budget. The paper is a clean architectural pitch tied to a specific working hypothesis about what transformers do badly.
- **Theme:** Core LLM advances (architecture, working state)
- **Relevance:** Working memory / state maintenance is close to Theme 1's concerns; an architectural take complements the memory-as-external-store papers above.
- **Link:** https://arxiv.org/abs/2607.01218

## 9. Denser ≠ Better: Limits of On-Policy Self-Distillation for Continual Post-Training
- **Authors:** Meng Wang, Haohan Zhao, Wenzhuo Liu, et al.
- **Summary:** Studies Self-Distillation Policy Optimization (SDPO) in continual post-training and finds that denser on-policy distillation actually accelerates forgetting and induces representation collapse rather than helping. The analysis attributes this to self-reinforcing drift — dense signals amplify artifacts in the student that then feed back through the on-policy loop. The takeaway is that on-policy data alone is insufficient for continual updates, and denser is not automatically better.
- **Theme:** Core LLM advances (fine-tuning / continual learning)
- **Relevance:** A negative result that changes how one should design continual post-training pipelines; useful signal even without a positive fix.
- **Link:** https://arxiv.org/abs/2607.01763

## 10. Program-as-Weights: A Programming Paradigm for Fuzzy Functions
- **Authors:** Wentao Zhang, Liliana Hotsko, Woojeong Kim, et al.
- **Summary:** Compiles natural-language specifications into parameter-efficient adapters over a frozen small "interpreter" model, treating the resulting weights as an executable program. A 4B compiler trained on a fuzzy-function benchmark emits adapters that let a 0.6B Qwen3 interpreter match the accuracy of a 32B baseline at roughly 1/50th the memory, running locally at ~30 tokens/s on a MacBook M3. Positions PEFT as the substrate for a compiler-style workflow rather than as a fine-tuning trick.
- **Theme:** Core LLM advances (PEFT / adaptation)
- **Relevance:** A concrete, empirically supported take on making adaptation the primary interface to LLMs — relevant to agent stacks that need per-task specialization at low cost.
- **Link:** https://arxiv.org/abs/2607.02512

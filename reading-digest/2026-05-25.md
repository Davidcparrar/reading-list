## 1. DeferMem: Query-Time Evidence Distillation via Reinforcement Learning for Long-Term Memory QA
**Authors:** Jianing Yin, Tan Tang

**Summary:** DeferMem rethinks long-term memory QA by deferring the hard work to query time: it keeps a cheap, high-recall segment-link store for retrieval and then runs a learned "evidence distiller" that turns noisy retrieved candidates into faithful, self-contained, query-conditioned evidence. The distiller is trained with a custom RL algorithm, DistillPO, that frames distillation as message selection plus evidence rewriting and optimizes it with a gated, span-attributed reward. On LoCoMo and LongMemEval-S it reports the highest QA accuracy with the fastest runtime and zero commercial-API token cost for memory operations.

**Theme:** Memory for LLMs/agents (long-term, retrieval-augmented memory).

**Relevance:** A novel framework plus RL method aimed squarely at the central long-term-memory-QA problem, with strong head-to-head results — the most on-theme paper in the window.

**Link:** https://arxiv.org/abs/2605.22411

## 2. MemAudit: Post-hoc Auditing of Poisoned Agent Memory via Causal Attribution and Structural Anomaly Detection
**Authors:** Zhewen Tan, Yilun Yao, Huiyan Jin et al.

**Summary:** MemAudit targets memory poisoning in agents that persist past interactions: after harmful behavior is observed, it asks which stored memories were responsible. It blends a counterfactual "memory influence" score that measures each memory's causal contribution to a harmful output with a memory-consistency graph that flags structurally anomalous entries, operating purely post-hoc rather than via online prompt filtering. Against the MINJA query-only injection attack it cuts QA attack success from 70% to 0% and reasoning-agent (RAP) attack success from 83.3% to 0%.

**Theme:** Memory for agents (security / governance of long-term memory).

**Relevance:** Tackles a fast-growing risk in agent memory systems with a concrete causal-plus-structural method and dramatic empirical gains.

**Link:** https://arxiv.org/abs/2605.23723

## 3. What Training Data Teaches RL Memory Agents: An Empirical Study of Curriculum Effects in Memory-Augmented QA
**Authors:** Xinjie He, Zhiyuan Lin, Su Liu et al.

**Summary:** Holding architecture, RL algorithm, and hyperparameters fixed, this study varies only the training curriculum (in-domain LoCoMo, mixed, or out-of-domain LongMemEval) for memory-augmented multi-session QA agents. It finds curriculum composition acts as a specialization lever rather than a uniform performance scaler — mixed data gives the best overall F1, while a narrow out-of-domain set selectively transfers temporal reasoning — and that per-question-type effects are systematically masked by single-number benchmarks. It also reports practical GRPO-on-one-GPU lessons: filter format-specific noise when mixing memory banks, and use continuous rather than binary-exact-match rewards at small group sizes.

**Theme:** Memory for agents (training memory-augmented agents).

**Relevance:** Rigorous, controlled evidence on how training data shapes RL memory agents — directly informs how memory systems should be trained and benchmarked.

**Link:** https://arxiv.org/abs/2605.23067

## 4. The Efficiency Frontier: A Unified Framework for Cost-Performance Optimization in LLM Context Management
**Authors:** Binqi Shen, Lier Jin, Hanyu Cai et al.

**Summary:** This paper casts context-strategy selection (retrieval vs. memory compression vs. full context) as a deployment-aware optimization that jointly accounts for task performance, token cost, and amortized preprocessing reuse, instead of comparing methods in isolation. On 5,000 HotpotQA instances it identifies distinct "operational regimes" and transition boundaries where each strategy becomes preferable. Deployment-aware selection cuts effective token usage by ~25% at comparable F1 (~0.78), and amortized memory compression saves over 50% tokens versus full-context prompting in higher-performance settings.

**Theme:** Memory for LLMs (context / state management).

**Relevance:** A decision-oriented framework for the core context-management trade-off, grounded in concrete cost-performance numbers.

**Link:** https://arxiv.org/abs/2605.23071

## 5. ArborKV: Structure-Aware KV Cache Management for Scaling Tree-based LLM Reasoning
**Authors:** Yeqiu Chen, Ziyan Liu, Zhenxin Huang et al.

**Summary:** In Tree-of-Thoughts-style reasoning, retaining KV states for a frontier of partial trajectories becomes a memory bottleneck that caps search depth and width. ArborKV observes that near-term decoding depends mainly on the active branch and its ancestors while inactive subtrees have low short-term reuse yet must stay recoverable for backtracking, and accordingly pairs a lightweight value estimator with a tree-aware eviction policy plus token-extractive eviction and lazy rehydration. It achieves up to ~4x peak KV-memory reduction while preserving near-full-retention accuracy, enabling larger search configurations under a fixed hardware budget.

**Theme:** Memory for LLMs (working memory / KV-cache management).

**Relevance:** Novel structure-aware memory management that unlocks deeper reasoning search — core to working-memory/state handling.

**Link:** https://arxiv.org/abs/2605.22106

## 6. Parallel Context Compaction for Long-Horizon LLM Agent Serving
**Authors:** Musa Cim, Burak Topcu, Chita Das et al.

**Summary:** Long-horizon agents accumulate histories that outgrow the context window, and the usual LLM-summarization "compaction" is lossy, blocks inference for tens of seconds, and yields unpredictable summary volume across runs. This work introduces parallel compaction, giving operators fine-grained, predictable control over summary size and per-block prompt engineering, and characterizes it against the sequential baseline across four 8B–120B backbones (dense and MoE, reasoning and non-reasoning) on HotpotQA and LoCoMo. At matched compaction decode volume it reduces end-to-end wall time and improves compaction throughput.

**Theme:** Memory for agents (context / state management).

**Relevance:** A practical, systematically evaluated method for the recurring problem of bounding and controlling long-horizon agent context.

**Link:** https://arxiv.org/abs/2605.23296

## 7. LCGuard: Latent Communication Guard for Safe KV Sharing in Multi-Agent Systems
**Authors:** Sadia Asif, Mohammad Mohammadi Amiri, Momin Abbas et al.

**Summary:** When multi-agent LLM systems share transformer KV caches as latent "working memory" for efficiency, those caches can silently carry sensitive per-agent inputs and reasoning states across agents. LCGuard learns representation-level transformations applied before a cache is transmitted, formalizing leakage operationally as an adversarial decoder's ability to reconstruct sensitive inputs, and trains adversarially to preserve task-relevant semantics while minimizing reconstructable content. Across model families and multi-agent benchmarks it lowers reconstruction-based leakage and attack success rates while keeping task performance competitive with standard KV-sharing baselines.

**Theme:** Memory (KV / latent working memory) bridging into multi-agent communication.

**Relevance:** Connects memory representations to safe inter-agent communication, sitting at the intersection of the top two priority themes.

**Link:** https://arxiv.org/abs/2605.22786

## 8. The Log is the Agent: Event-Sourced Reactive Graphs for Auditable, Forkable Agentic Systems
**Authors:** Yohei Nakajima

**Summary:** ActiveGraph inverts the usual agent-framework stack: an append-only event log is the source of truth, the working graph is a deterministic projection of that log, and behaviors react to graph changes and emit new events, so components coordinate only through the shared graph rather than instructing one another. This design yields deterministic replay of any run, cheap forking that branches at any event without re-executing the shared prefix, and end-to-end lineage from a high-level goal down to each individual model call — properties retrieval-and-summarization memory does not provide. The paper presents the architecture, a determinism contract that makes replay sound, and a worked diligence example reconstructable from the log alone.

**Theme:** Multi-agent frameworks & orchestration (agent architecture, coordination, state).

**Relevance:** A distinctive coordination/state substrate for agentic systems (from the BabyAGI author) that reframes agent memory as an event log.

**Link:** https://arxiv.org/abs/2605.21997

## 9. Foundation Protocol: A Coordination Layer for Agentic Society
**Authors:** Bang Liu, Yongfeng Gu, Jiayi Zhang et al.

**Summary:** Arguing that as agents scale the bottleneck shifts from raw model capability to coordination, this paper proposes the Foundation Protocol, a graph-first coordination layer for an emerging human-AI society. It unifies heterogeneous entities (agents, tools, resources, humans, institutions, organizations), supports native multi-party organization and event-based collaboration, and adds economic primitives (metering, receipts, settlement) while treating policy, provenance, and audit as first-class concerns. It is designed to wrap and bridge existing protocols rather than replace them, enabling incremental adoption while keeping coordination composable and accountable.

**Theme:** Multi-agent frameworks & orchestration (communication / coordination protocol).

**Relevance:** A broad coordination-and-governance protocol for inter-agent collaboration — central to multi-agent orchestration, though more infrastructural/positional than empirical.

**Link:** https://arxiv.org/abs/2605.23218

## 10. When Planning Fails Despite Correct Execution: On Epistemic Calibration for LLM-Based Multi-Agent Systems
**Authors:** Zehao Wang, Shilong Jin, Zhao Cao et al.

**Summary:** This paper names "epistemic miscalibration in planning" — multi-agent failures where plans are self-consistent and execute correctly yet fail because agents misjudge what they actually know about feasibility, a problem that is latent during planning and shifts as new information arrives. Its Epistemic Planning Calibration Agentic Workflow (EPC-AW) selects plans whose feasibility assessments stay stable across agents (information-consistency-based selection) and refines agents' epistemic state over time by leveraging past discrepancies, rather than directly verifying feasibility. The approach improves system-level success by an average of 9.75%.

**Theme:** Multi-agent frameworks & orchestration (planning / coordination).

**Relevance:** Targets a subtle, underexplored multi-agent planning failure mode with a concrete workflow and a measured success gain.

**Link:** https://arxiv.org/abs/2605.23414

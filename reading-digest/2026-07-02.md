1. **MemSyco-Bench: Benchmarking Sycophancy in Agent Memory**
   - Authors: Zhishang Xiang, Zerui Chen, Yunbo Tang, et al.
   - Summary: The paper stakes out a specific failure mode of agent long-term memory — retrieved past interactions bias the agent toward the *user's* prior beliefs at the expense of factual accuracy — and builds a benchmark to measure it across five task categories: memory rejection, scope respect, evidence-conflict resolution, update tracking, and personalization use. The setup pits a memory-augmented agent against situations where the correct answer contradicts what the user previously said or believed, so a well-behaved system has to actively *ignore or override* the retrieved memory rather than defer to it. The take-away is a concrete, adversarial evaluation harness that most current memory stacks likely fail on, since they were tuned for recall rather than principled override.
   - Theme: (1) memory for LLMs and agents.
   - Relevance note: The most on-target new memory paper in the window — sycophancy from stored history is exactly the failure mode that will bite production agent-memory systems, and this is the first benchmark I've seen framed around it.
   - Link: https://arxiv.org/abs/2607.01071

2. **The State-Prediction Separation Hypothesis**
   - Authors: Giovanni Lozano, Nikolaos Rapathakis, Yoav Artzi, Kris Brantley
   - Summary: The claim is that Transformers overload one forward computation stream to do two jobs — predict the next token *and* stash useful state for future tokens — and that disentangling those two roles into separate streams pretrains more compute- and data-efficiently. Experiments across scales report a 2–3 percentage-point downstream improvement over standard Transformers at matched compute, with ablations meant to rule out mundane confounders and gradient-behavior analyses that argue the two designs are fundamentally different, not just tuned differently. Positioned as an architectural rethink of what "hidden state" means in autoregressive language modeling.
   - Theme: (1) memory for LLMs and agents — treats internal hidden state as an implicit working-memory mechanism and gives it its own computation path.
   - Relevance note: If it holds up at scale, it's the kind of result that quietly changes how backbone memory (state) is engineered inside dense LMs — memory-mechanism papers at this altitude are rare.
   - Link: https://arxiv.org/abs/2607.01218

3. **DiscoPER: Autonomous Scientific Discovery via Iterative Meta-Reflection**
   - Authors: Bingchen Zhao, Sara Beery, Oisin Mac Aodha
   - Summary: DiscoPER is an LLM-driven autonomous research agent that dynamically writes and runs code to interrogate datasets *without* a predefined research question, gating every proposed discovery through a statistical test. Its distinguishing move is a second-order "meta-reflection" mechanism that periodically treats its own accumulated confirmed discoveries as empirical data, extracting structural patterns, confounds, and epistemic gaps from them and steering the next round of hypothesis search into under-explored regions. On iNatDisco — a new multimodal ecological benchmark with pattern-level ground truth mined from peer-reviewed papers — it recovers 8 of 9 known patterns with a 72.7% hypothesis-support rate, beating both classical causal-discovery methods and standard LLM-guided baselines.
   - Theme: (2) multi-agent frameworks & orchestration (single-agent architecture with an explicit self-reflection loop over persistent discovery history).
   - Relevance note: The meta-reflection-over-accumulated-outputs pattern is directly transferable to any long-horizon agent that needs to avoid redundant work and steer its own trajectory — think research assistants, ops agents, code agents that maintain a discovery log.
   - Link: https://arxiv.org/abs/2607.01131

4. **ASPIRE: Agentic Skills Discovery for Robotics**
   - Authors: Runyu Lu, Yubo Wu, Ethan Kou, et al.
   - Summary: ASPIRE is an open-ended, continual-learning agent that writes and repeatedly rewrites its own robot-control programs (code-as-policy) while distilling every validated fix into a persistent, reusable skill library. Three components carry the design: a closed-loop execution engine that surfaces fine-grained multimodal traces (perception overlays, grasp candidates, collision feedback) so the agent can diagnose and repair its own failures; the skill library itself; and an evolutionary search that generates diverse task sequences and control programs and debugs them against real and simulated executions. The framework is meant to compound experience across tasks, embodiments, and sim-vs-real settings rather than restart from scratch each time.
   - Theme: (2) multi-agent frameworks & orchestration (agentic skill acquisition + persistent skill memory).
   - Relevance note: The persistent, cross-task skill library is essentially long-term *procedural* memory for an agent — a memory-shape adjacent to Theme 1 that most LLM-agent stacks still don't have. Also from the NVIDIA/Berkeley/CMU cluster whose agent recipes have a good track record.
   - Link: https://arxiv.org/abs/2607.00272

5. **AutoTrainess: Teaching Language Models to Improve Language Models Autonomously**
   - Authors: Zhaojian Yu, Penghao Yin, Shuzheng Gao, et al.
   - Summary: AutoTrainess argues that plain CLI access is a bad interface for an LLM tasked with running its own training loop, and instead exposes a structured "agent–computer interface" over planning, data preparation, training, evaluation, and logging so the model reasons over higher-level operations rather than raw commands. On a new PostTrainBench benchmark, the paper reports 26.94 average with the structured interface versus 23.21 CLI-only for a strong coding-model backbone, with generalization across model families claimed as evidence the gains come from the interface abstraction rather than one backbone. The larger bet is that agent-computer interface design, not just prompting, is the lever for autonomous self-improvement pipelines.
   - Theme: (2) multi-agent frameworks & orchestration (agent architecture / tool-and-interface design).
   - Relevance note: The "interface design as agent design" framing is worth watching alongside recent tool-use RL work; also directly relevant to anyone building agents that run their own experiments.
   - Link: https://arxiv.org/abs/2606.31551

6. **Spire: Personalization as Inverse Planning — Learning Latent Design Intents for Agentic Slide Generation via Structural Denoising**
   - Authors: Zihan Dong, Linjun Zhang, Haoyu Wang, et al.
   - Summary: Spire reformulates page-level slide personalization as an inverse-planning problem in which the design intent is a latent to be inferred without assuming any particular renderer (PowerPoint, Beamer, etc.), then converts that intractable objective into a tractable surrogate by deliberately corrupting clean slides and training two cooperating agents to denoise the corruption via reinforcement learning. The authors give a proof that structural denoising is a consistent surrogate for the original inverse-planning problem and, separately, that the two-agent factorization strictly reduces policy-gradient variance versus a monolithic policy. Experiments claim a clear win over prior AI-driven slide-personalization methods.
   - Theme: (2) multi-agent frameworks & orchestration (two-agent RL with a formal variance-reduction argument).
   - Relevance note: One of the rare multi-agent-RL papers this window that pairs its cooperation pattern with an explicit theoretical justification — a nice datapoint for "why decompose at all."
   - Link: https://arxiv.org/abs/2607.00407

7. **Multimodal Continuous Reasoning via Asymmetric Mutual Variational Learning (AMVL)**
   - Authors: Shijie Li, Yilin Gao, Siyuan Yang, et al.
   - Summary: The paper targets the "language-space bottleneck" in MLLMs that forces continuous visual evidence through discrete tokens, and proposes to reason instead in a continuous latent — but flags a specific failure mode of the naïve variational recipe, where the ground-truth-conditioned posterior takes an "answer-dependent shortcut" and the inference-time prior is forced to mimic something it can't actually reproduce (answer leakage). AMVL fixes this with a bidirectional KL objective — a forward KL that aligns the target-agnostic prior to the posterior, and a novel reverse KL that keeps the posterior itself from collapsing into inference-incompatible regions. On the BLINK reasoning benchmark they report +10.83 average and up to +32.00 on individual tasks, with stability analyses of the latent space.
   - Theme: (3) core LLM advances (latent reasoning for MLLMs).
   - Relevance note: Latent/continuous reasoning is having a moment; the answer-leakage formalization is the useful concept to steal even if you don't adopt the full recipe.
   - Link: https://arxiv.org/abs/2607.00461

8. **Graph-PRefLexOR: Graph-Native Reinforcement Learning for Traceable Scientific Hypothesis Generation**
   - Authors: Subhadeep Pal, Shashwat Sourav, Tirthankar Ghosal, Markus J. Buehler
   - Summary: A GRPO-fine-tuned family of models that explicitly structures reasoning into phased sub-steps — mechanism exploration, graph construction, pattern extraction, hypothesis synthesis — so the intermediate graph is a first-class object that can be inspected, extended, and reused across queries. On 100 open-ended materials-science and mechanics problems, the reported gain over base models is 40–65% with the biggest deltas showing up in *traceability* rather than raw accuracy, and embedding/backtracking analyses argue for 2–3× broader semantic exploration than baselines. Test-time graph expansion is analyzed as a way to trade compute for long-range conceptual recombination inside a bounded semantic space.
   - Theme: (3) core LLM advances (planning / interpretable reasoning with GRPO).
   - Relevance note: Neat instantiation of "reasoning as an explicit structured artifact" rather than free-text chain-of-thought — the traceability angle is what makes it worth surfacing over generic GRPO work.
   - Link: https://arxiv.org/abs/2607.00924

9. **Perceive-to-Reason: Decoupling Perception and Reasoning for Fine-Grained Visual Reasoning**
   - Authors: Hongxing Li, Xiufeng Huang, Dingming Li, et al.
   - Summary: For fine-grained visual reasoning on high-resolution images the paper argues you should stop conflating "where is the evidence" with "what does it imply", and instead run a two-stage Perceiver-then-Reasoner pipeline where stage one localizes and crops question-relevant regions and stage two answers over the annotated image plus crops. They train this end-to-end with only final-answer supervision using PRA-GRPO, a role-aware RL scheme that alternates between updating the Perceiver role and updating the Reasoner role. Instantiated on Qwen3-VL-Instruct at 2B, 4B, and 8B scales, the split-role setup reports consistent gains across model sizes.
   - Theme: (3) core LLM advances (multimodal reasoning + role-decomposed RL).
   - Relevance note: Complementary to AMVL (#7) — different failure mode (perceptual localization vs answer leakage), same overall message that MLLM reasoning benefits from explicit factorization.
   - Link: https://arxiv.org/abs/2607.01191

10. **CausalMix: Data Mixture as Causal Inference for Language Model Training**
    - Authors: Zinan Tang, Yukun Zhang, Shaomian Zheng, et al.
    - Summary: CausalMix reframes training-data-mixture selection as a causal-inference problem rather than a static optimization over proxy-model runs: they fit a causal model over 512 Qwen2.5-0.5B training runs to estimate the Conditional Average Treatment Effect of each domain, then extrapolate the optimal mixture for an 800K-example pool and use it to train a 7B model. The same recipe is shown to transfer to long chain-of-thought data on Qwen3-4B-Base. They report consistent wins over RegMix and other mixture baselines, and use the CATE surface itself to interpret *why* the chosen mixture works.
    - Theme: (3) core LLM advances (training recipe / data curation).
    - Relevance note: Least agent-shaped of the ten, but genuinely useful for anyone doing continued-pretraining or post-training on custom mixtures, and the causal framing gives you an interpretable knob rather than another black-box weight search.
    - Link: https://arxiv.org/abs/2607.01104

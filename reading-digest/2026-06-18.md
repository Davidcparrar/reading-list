# Reading digest — 2026-06-18

Found 10 paper(s) in the last 48h window.

## 1. OPD-Evolver: Cultivating Holistic Agent Evolver via On-Policy Distillation
- **Authors:** Guibin Zhang, Xun Xu, Yanwei Yue, et al.
- **Summary:** Proposes a self-evolving agent stack that jointly upgrades the agent's memory store and its acting policy through a slow/fast co-evolution loop driven by on-policy distillation. The "slow" track curates and rewrites long-term memory while the "fast" track distills behavior from the agent's own recent trajectories, giving a single recipe that improves both retrieval-conditioned acting and policy quality across multiple domains.
- **Theme:** Memory for LLMs and agents (+ multi-agent self-improvement).
- **Relevance note:** Treats memory management as a first-class learning target alongside policy — directly on-theme for our top priority.
- **Link:** https://arxiv.org/abs/2606.17628

## 2. Beyond the Current Observation: Evaluating Multimodal Large Language Models in Controllable Non-Markov Games (RNG-Bench)
- **Authors:** Shengyuan Ding, Xilin Wei, Xinyu Fang, et al.
- **Summary:** Introduces a controllable benchmark of two interactive games whose state is non-Markov: solving them requires the model to remember and reconstruct prior observations rather than respond only to the current frame. A "memory gap" metric quantifies how far a model degrades as the required recall depth grows, isolating working/episodic memory failure modes from raw perception or reasoning.
- **Theme:** Memory for LLMs and agents (evaluation).
- **Relevance note:** A clean, calibratable probe for the exact recall behaviors we care about in agentic memory work.
- **Link:** https://arxiv.org/abs/2606.19338

## 3. Native Active Perception as Reasoning for Omni-Modal Understanding (OmniAgent)
- **Authors:** Zhenghao Xing, Ruiyang Xu, Yuxuan Wang, et al.
- **Summary:** Builds an omni-modal agent that interleaves explicit observe → think → act steps, choosing which video/audio span to re-attend to next as part of its reasoning trace. The active-perception loop turns long-context understanding into a sequence of cheap targeted lookups and reportedly beats larger passive models on long-video tasks.
- **Theme:** Multi-agent frameworks & orchestration (single-agent loop with tool-style perception calls).
- **Relevance note:** Architectural pattern (decision-making over perception tools) generalizes naturally to multi-agent and retrieval-over-memory settings.
- **Link:** https://arxiv.org/abs/2606.19341

## 4. CEO-Bench: Can Agents Play the Long Game?
- **Authors:** Haozhe Chen, Karthik Narasimhan, Zhuang Liu
- **Summary:** Sets up a 500-simulated-day startup environment in which an LLM agent must juggle hiring, product, and finance threads, exposing whether models can plan and re-plan over long horizons rather than only handling short tool-use episodes. Results emphasize how quickly current agents drift, forget commitments, and fail to coordinate parallel goals.
- **Theme:** Multi-agent / agentic-system evaluation (long-horizon planning).
- **Relevance note:** Directly tests the planning + persistent-state qualities memory and orchestration work is supposed to deliver.
- **Link:** https://arxiv.org/abs/2606.18543

## 5. Externalizing Research Synthesis and Validation in AI Scientists through a Research Harness (Xcientist)
- **Authors:** Zijian Wang, Hanqi Li, Ziyue Yang, et al.
- **Summary:** Proposes a "research harness" wrapper around AI-scientist agents that externalizes intermediate artifacts (hypotheses, experiment plans, results, critiques) into a persistent structured store the agents must read from and write to. Coordination then happens via these shared artifacts instead of opaque chat history, making synthesis and validation steps inspectable and replayable.
- **Theme:** Multi-agent frameworks & orchestration (artifact-mediated coordination).
- **Relevance note:** Concrete protocol design for shared-state multi-agent systems, with explicit treatment of long-term scratch memory.
- **Link:** https://arxiv.org/abs/2606.18874

## 6. GameCraft-Bench: Can Agents Build Playable Games End-to-End in a Real Game Engine?
- **Authors:** Tongxu Luo, Rongsheng Wang, Jiaxi Bi, et al.
- **Summary:** Tasks coding agents with producing a fully playable game in a real engine from a natural-language spec, then scores them on engine grounding, artifact completeness, and interactive verification rather than just unit-test pass rates. The benchmark stresses tool use, multi-file project state, and the ability to iterate against a running runtime.
- **Theme:** Multi-agent / agentic-system evaluation (long, stateful coding workflows).
- **Relevance note:** A serious end-to-end testbed for agent orchestration where intermediate state is large and persistent.
- **Link:** https://arxiv.org/abs/2606.17861

## 7. Zone of Proximal Policy Optimization: Teacher in Prompts, Not Gradients (ZPPO)
- **Authors:** Byung-Kwan Lee, Ximing Lu, Shizhe Diao, et al.
- **Summary:** Reframes knowledge distillation as teacher-shaped prompt rewriting plugged into PPO: the teacher does not provide logits or gradients, but rewrites the prompt so the student is steered through both successful and failed rollouts in its zone of proximal capability. Gains are largest for small students, suggesting prompt-level supervision is a cheaper alternative to logit/feature distillation for agentic post-training.
- **Theme:** Core LLM advances (fine-tuning / PEFT-style distillation).
- **Relevance note:** Cheap recipe for sharpening small models that often serve as sub-agents or memory routers.
- **Link:** https://arxiv.org/abs/2606.18216

## 8. LoopCoder-v2: Only Loop Once for Efficient Test-Time Computation Scaling
- **Authors:** Jian Yang, Shawn Guo, Wei Zhang, et al.
- **Summary:** Investigates "looped" parallel-transformer decoding for code generation and finds the sweet spot is a single additional loop: two passes refine representations enough to beat one-pass models, while further loops hurt due to positional-mismatch costs. Result is a concrete recipe for cheap test-time compute scaling on code without retraining.
- **Theme:** Core LLM advances (reasoning / test-time compute).
- **Relevance note:** Practical, agent-friendly inference trick relevant whenever sub-agents call a code model.
- **Link:** https://arxiv.org/abs/2606.18023

## 9. Learning from the Self-future: On-policy Self-distillation for dLLMs (d-OPSD)
- **Authors:** Yifu Luo, Zeyu Chen, Haoyu Wang, et al.
- **Summary:** Adapts on-policy self-distillation, which had been built for autoregressive models, to diffusion LLMs by reshaping how a "self-teacher" is sampled and how supervision is applied across the non-autoregressive denoising trajectory. The technique stabilizes training of diffusion-style language models and improves their downstream performance.
- **Theme:** Core LLM advances (training / PEFT for alternative LM families).
- **Relevance note:** Diffusion LMs are an increasingly relevant substrate for fast or parallel agents; this addresses a real training-stability gap.
- **Link:** https://arxiv.org/abs/2606.18195

## 10. Learning User Simulators with Turing Rewards
- **Authors:** Yingshan Susan Wang, Cedegao E. Zhang, Linlu Qiu, et al.
- **Summary:** Trains user-simulator models with RL against a Turing-test reward: a discriminator tries to tell simulator turns from real human turns, and the simulator is optimized to be indistinguishable. The result is a more realistic interlocutor for evaluating dialog agents and for generating training data for assistant policies.
- **Theme:** Core LLM advances (agentic-system evaluation, simulators).
- **Relevance note:** Better user simulators are the bottleneck for honest offline eval of agent and memory systems.
- **Link:** https://arxiv.org/abs/2606.19336

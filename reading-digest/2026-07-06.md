1. **AutoMem: Automated Learning of Memory as a Cognitive Skill**
   - Authors: Shengguang Wu, Hao Zhu, Yuhui Zhang, Xiaohan Wang, Serena Yeung-Levy
   - Summary: AutoMem promotes filesystem operations to first-class memory actions the agent invokes alongside its task actions, and then optimizes memory management along two axes with two separate loops: an outer loop where a strong critic LLM rewrites the memory *structure* (prompts, file schemas, action vocabulary) after reviewing full trajectories, and an inner loop that mines the agent's own good memory decisions to fine-tune its memory *proficiency*. On three procedurally generated long-horizon games (Crafter, MiniHack, NetHack) memory optimization alone — task-action policy unchanged — yields ~2× to ~4× gains, pushing a 32B open-weight backbone into range of Claude Opus 4.5 and Gemini 3.1 Pro Thinking.
   - Theme: (1) memory for LLMs and agents (long-term episodic/working memory as a trainable skill).
   - Relevance note: Cleanest new framing of memory-as-a-learnable-skill in the window, with a concrete 32B recipe that closes most of the gap to frontier agents on long-horizon tasks — top-priority theme with real numbers.
   - Link: https://arxiv.org/abs/2607.01224

2. **AgenticSTS: A Bounded-Memory Testbed for Long-Horizon LLM Agents**
   - Authors: Xiangchen Cheng, Yunwei Jiang, Jianwen Sun, et al.
   - Summary: The authors argue that the standard "append everything to the prompt" contract makes it impossible to isolate the effect of any single memory component, and propose an alternative bounded contract in which every decision is made from a fresh user message assembled by typed retrieval — no raw cross-decision transcript. They instantiate it in Slay the Spire 2 (a stochastic deck-builder where frontier LLMs currently score zero wins at the lowest difficulty against a 16% human baseline) and release 298 tagged trajectories, prompt records, and analysis scripts. A fixed-A0 ablation with triggered strategic skills enabled shows the largest observed lift (3/10 → 6/10 wins), which they carefully report as directional (Fisher p≈0.37) rather than decisive.
   - Theme: (1) memory for LLMs and agents (memory-architecture methodology).
   - Relevance note: Rare paper that treats "what should the prompt-assembly contract be" as the primary experimental variable and releases the harness — useful for anyone designing an agent-memory ablation study.
   - Link: https://arxiv.org/abs/2607.02255

3. **MemSyco-Bench: Benchmarking Sycophancy in Agent Memory**
   - Authors: Zhishang Xiang, Zerui Chen, Yunbo Tang, et al.
   - Summary: MemSyco-Bench targets a failure mode existing memory benchmarks miss: even when memories are stored, retrieved, and updated correctly, an agent will often *over-trust* them at the expense of factual accuracy — memory-induced sycophancy. The benchmark covers five task types that probe whether an agent can reject invalid memories as factual evidence, respect the scope in which a memory applies, resolve conflicts between memory and objective evidence, track memory updates over time, and still use valid memory for legitimate personalization. It is positioned as a downstream-reasoning complement to storage/retrieval-correctness suites.
   - Theme: (1) memory for LLMs and agents (evaluation of memory-conditioned reasoning).
   - Relevance note: New evaluation axis that's directly actionable — any team shipping agent memory into production needs a sycophancy probe, and there was none before.
   - Link: https://arxiv.org/abs/2607.01071

4. **ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning**
   - Authors: Yanjun Zhao, Ruizhong Qiu, Tianxin Wei, et al.
   - Summary: ReContext is a training-free inference harness that closes the "context access ≠ context utilization" gap in long-context LLMs by using the model's own internal relevance signals to build a query-conditioned evidence pool and *replay* it before the final generation, while keeping the full original context intact — no external memory, no pruning, no fine-tune. The authors give an associative-memory reading of the mechanism (context = memory store, query = retrieval cue, attention = cue-trace association, replay = trace reactivation) and evaluate on eight 128K-context datasets across Qwen3-4B, Qwen3-8B, and Llama3-8B, getting the best average rank on all three backbones.
   - Theme: (1) memory for LLMs and agents (long-context / working memory at inference time).
   - Relevance note: Practical drop-in for anyone stuck at "the answer is somewhere in the 100K context and the model won't use it" — training-free and backbone-agnostic is the key part.
   - Link: https://arxiv.org/abs/2607.02509

5. **Logit-Contribution Scoring Identifies Non-Literal Retrieval Heads**
   - Authors: Aryo Pradipta Gema, Beatrice Alex, Pasquale Minervini
   - Summary: Prior "retrieval head" detectors score attention heads by *what they attend to*, which misses heads that synthesize an answer non-literally from the meaning of a context span rather than copy-pasting it. LOCOS scores heads instead by *what they write to the output* — the OV-circuit contribution to the answer tokens' logits — and, on the NoLiMa non-literal retrieval benchmark, ablating the top-scoring 50 heads on Qwen3-8B collapses ROUGE-L much more sharply than ablating heads chosen by baseline detectors, without hurting unrelated reasoning tasks. Verified across Qwen3, Gemma-3, and OLMo-3.1.
   - Theme: (1) memory for LLMs and agents (mechanistic view of retrieval from context).
   - Relevance note: Sharpens the retrieval-head literature in exactly the direction that matters for RAG and long-context — non-literal synthesis is where retrieval quietly succeeds or fails.
   - Link: https://arxiv.org/abs/2607.01002

6. **PACE: A Proxy for Agentic Capability Evaluation**
   - Authors: Yueqi Song, Lintang Sutawika, Jiarui Liu, et al.
   - Summary: Full agentic benchmarks (SWE-Bench, GAIA) cost thousands of dollars and days per run; PACE builds a cheap proxy by fitting a regression from a model's scores on a compact, carefully curated set of *non-agentic* instances to its score on a target agentic benchmark. The instance set is selected by combining a target-relevance local strategy with a globally informative one. Across 14 models, 4 agentic benchmarks, and 19 non-agentic sources, PACE-Bench predicts agentic scores with LOOCV MAE under 4%, Spearman > 0.80, and ~85% pairwise ranking accuracy at under 1% of the full evaluation cost.
   - Theme: (2) multi-agent frameworks & orchestration (agent evaluation).
   - Relevance note: Immediately useful for model selection and routing in agent stacks — the "give me a rank without running GAIA" problem is real and this hits it head-on.
   - Link: https://arxiv.org/abs/2607.02032

7. **SkillCoach: Self-Evolving Rubrics for Evaluating and Enhancing Agentic Skill-Use**
   - Authors: Jiayin Zhu, Kelong Mao, Yudong Guo, et al.
   - Summary: In a realistic skill library, overlapping skills mean the outcome-only "did the verifier pass" signal is too coarse: an agent can pass by selecting distractor skills, skipping required steps, or composing workflows wrong. SkillCoach derives skill-grounded process rubrics from real rollouts and grades trajectories along four axes — skill selection, skill following, skill composition, and skill-grounded reflection — while keeping the external verifier as an independent outcome signal. The evolved rubrics not only expose failures that outcome accuracy hides but also serve as process supervision that beats outcome-only filtering for downstream training.
   - Theme: (2) multi-agent frameworks & orchestration (process supervision for skill orchestration).
   - Relevance note: The right shape of process reward for tool/skill agents — pairs well with TACO and MRPO from prior weeks and is a plausible default supervision scheme.
   - Link: https://arxiv.org/abs/2607.01874

8. **EvoPolicyGym: Evaluating Autonomous Policy Evolution in Interactive Environments**
   - Authors: Zhilin Wang, Han Song, Runzhe Zhan, et al.
   - Summary: Existing agent evaluations tend to collapse policy iteration into a single final score, or entangle it with open-ended software-engineering progress. EvoPolicyGym isolates *autonomous policy evolution* — the agent repeatedly refines an executable policy under a fixed feedback budget in a compact interactive RL environment — across 16 environments, with trajectory-level diagnostics showing how each budget unit is spent and how feedback gets converted into parameter adjustments. GPT-5.5 takes the top aggregate score; the finer diagnostic is that successful evolution requires discovering task-specific mechanisms rather than generic hill-climbing.
   - Theme: (2) multi-agent frameworks & orchestration (agent-loop evaluation).
   - Relevance note: Complements PACE by measuring a different agent capability (self-improvement loop quality) that outcome benchmarks don't isolate.
   - Link: https://arxiv.org/abs/2607.02440

9. **AgenticDataBench: A Comprehensive Benchmark for Data Agents**
   - Authors: Zhaoyan Sun, Shan Zhong, Daizhou Wen, et al.
   - Summary: A benchmark for LLM-based data-science agents with 15 vertical domains — including five real cases contributed by a fintech company — and, notably, a task-generation recipe that pulls diverse data-science *skills* from Stack Overflow solutions to ensure the tasks cover the actual skill surface. For domains without real tasks, workflows are LLM-generated in a structured way rather than free-form. The result is a fine-grained skill-level evaluation of frontier data agents in a segment where prior benchmarks are thin.
   - Theme: (2) multi-agent frameworks & orchestration (agent evaluation, data-science vertical).
   - Relevance note: Data agents are one of the near-term production agent segments — a proper skill-level benchmark here is overdue.
   - Link: https://arxiv.org/abs/2607.01647

10. **The State-Prediction Separation Hypothesis**
    - Authors: Giovanni Monea, Nathan Godey, Kianté Brantley, Yoav Artzi
    - Summary: Standard Transformers overload one forward stream with two jobs: predict the next token *and* stash state that later tokens will need. The paper introduces a Transformer variant that splits these into two distinct computation streams and shows it consistently improves data/compute efficiency, hits lower validation loss, and outperforms a matched vanilla Transformer by 2–3 points on downstream tasks. The empirical analysis identifies fundamentally different gradient dynamics between the two streams as the mechanism.
    - Theme: (3) core LLM advances (architecture, with obvious implications for how models carry information across steps — i.e., working memory).
    - Relevance note: Architectural take on the same "what should the model be storing vs. producing" question the memory-agent papers hit at the systems level — worth watching for scaling results.
    - Link: https://arxiv.org/abs/2607.01218

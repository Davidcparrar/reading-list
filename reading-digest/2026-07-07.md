1. **MRMS: A Multi-Resolution Memory Substrate for Long-Lived AI Agents**
   - Authors: Jizhizi Li, Amy Shi-Nash
   - Summary: Proposes an agent memory architecture organized across two axes at once — representational (structured records, vectors, graphs) and temporal (short / medium / long term) — so retrieval has a principled ordering rather than pure semantic similarity. Structured records govern eligibility while the graph layer resolves support and contradiction before content is projected into context. The claim is that keeping the three representations synchronized is what lets a long-lived agent avoid the drift and conflicts that pure-vector memory hits.
   - Theme: (1) memory for LLMs and agents.
   - Relevance note: Clean architectural take on multi-representation, multi-timescale agent memory — the exact top-priority track.
   - Link: https://arxiv.org/abs/2607.04617

2. **Memory-Orchestrated Semantic System (MOSS): An Auditable Agentic Memory Architecture**
   - Authors: Serge Lacasse, Jérémie Hatier, Alex Baker
   - Summary: MOSS drops embedding-similarity retrieval in favor of agents querying a relational memory store directly, and derives its indexing vocabulary inductively from the corpus itself. The architecture is deliberately model-, storage-, and API-agnostic, so every retrieval decision is reproducible and can be replayed for auditing. Aimed at deployments where you need to explain, not just accept, why the agent brought a particular fact into context.
   - Theme: (1) memory for LLMs and agents.
   - Relevance note: Auditability is under-served in the memory literature; a first-class relational alternative to vector RAG is worth reading even if you disagree with it.
   - Link: https://arxiv.org/abs/2607.04391

3. **PLACEMEM: Toward a Compute-Aware Memory Plane for Lifelong Agents**
   - Authors: Sukanta Ganguly
   - Summary: Treats agent memory as versioned "capsules" that bundle semantics, provenance, validity constraints, and reusable runtime state under one compute-aware plane. The prototype handles concurrency-safe invalidation and cascades corrections across streamed backends, targeting agents whose memory has to survive across sessions and compute contexts. Framing memory as a system-level object with versioning is a real departure from the read/write RAG mental model.
   - Theme: (1) memory for LLMs and agents.
   - Relevance note: Systems-flavored counterweight to the two architecture papers above; good triangulation on what "memory" means at the substrate layer.
   - Link: https://arxiv.org/abs/2607.04089

4. **When Claws Remember but Do Not Tell: Stealthy Memory Injection in Persistent Personal Agents**
   - Authors: Yechao Zhang, Shiqian Zhao, Jiawen Zhang, et al.
   - Summary: Studies stealth memory-injection attacks where untrusted external content silently poisons a personal agent's persistent memory across sessions. Introduces WhisperBench (108 cases) and a concrete attack, MemGhost, that reaches 87.5% end-to-end success on real IMAP/SMTP workflows while evading input-, model-, and system-level defenses. The value here is a benchmark and a strong attack that operational memory designs will have to answer to.
   - Theme: (1) memory for LLMs and agents (adversarial robustness).
   - Relevance note: If you build persistent memory, this is the threat model to design against; pairs naturally with the three architecture papers above.
   - Link: https://arxiv.org/abs/2607.05189

5. **GaP: A Graph-as-Policy Multi-Agent Self-Learning Harness for Variational Automation Tasks**
   - Authors: Kaiyuan Chen, Shuangyu Xie, Letian Fu, et al.
   - Summary: A multi-agent coding harness that emits a directed computation graph over modular perception/planning/control skills for industrial automation. The system spins up an internal simulator to rehearse task instances in parallel and iteratively refines both graph structure and node parameters — the graph itself is the policy. Reports deployments on Crazyflie and L1-class robot platforms without downstream fine-tuning.
   - Theme: (2) multi-agent frameworks & orchestration.
   - Relevance note: One of the more interesting "policy = graph of agents" formulations of the week — orchestration where the topology is the learned object, not the prompt.
   - Link: https://arxiv.org/abs/2607.05369

6. **OptiAgent: End-to-End Optimization Modeling via Multi-Agent Iterative Refinement**
   - Authors: Adriana Laurindo Monteiro, Nayse Fagundes, Gabriel Mattos Langeloh, et al.
   - Summary: Multi-agent framework that turns Operations Research problem descriptions into solver-ready mathematical formulations plus executable code end-to-end. Uses a multi-loop validation architecture with four specialized feedback mechanisms, each targeting a distinct failure mode, and iterates through self-correction cycles until validation passes. Interesting as a template for verifier-loop orchestration in a domain with hard correctness checks.
   - Theme: (2) multi-agent frameworks & orchestration.
   - Relevance note: The "multiple specialized validators, each for a specific failure mode" pattern generalizes beyond OR — worth reading with that in mind.
   - Link: https://arxiv.org/abs/2607.05346

7. **TACTIC-KG: Toward Small Agent Teams for Cyber Threat Intelligence Knowledge Graph Construction**
   - Authors: Mouhamed Amine Bouchiha, Gregory Blanc
   - Summary: Decomposes cyber threat intelligence KG construction into a team of specialized 3B–8B LLM agents handling extraction, typing, verification, and curation as separate roles. The team of smaller specialized agents consistently beats larger monolithic in-context-learning baselines on extraction F1, typing accuracy, and structural graph similarity. Neat empirical support for "specialization > scale" when the pipeline is decomposable.
   - Theme: (2) multi-agent frameworks & orchestration.
   - Relevance note: Concrete evidence for the small-model-specialization thesis on a domain that punishes hallucination — good counterpoint to monolith-agent baselines.
   - Link: https://arxiv.org/abs/2607.05001

8. **LLM-as-a-Verifier: A General-Purpose Verification Framework**
   - Authors: Jacky Kwok, Shulu Li, Pranav Atreya, et al.
   - Summary: Reframes LLM verification as a probabilistic score derived from the raw scoring-token logit distribution instead of a binary judgment. Scaling along three orthogonal axes — score granularity, repeated evaluation, and criteria decomposition — pushes the same framework to SOTA on Terminal-Bench V2 (86.5%), SWE-Bench Verified (78.2%), and RoboRewardBench (87.4%). Practically useful as a drop-in verifier for agent loops.
   - Theme: (3) core LLM advances (agentic-system evaluation / verifier design).
   - Relevance note: Strong empirical results on three heterogeneous benchmarks make this the most transportable verifier recipe from the week.
   - Link: https://arxiv.org/abs/2607.05391

9. **EvoAgentBench: Benchmarking Agent Self-Evolution via Ability Transfer**
   - Authors: Xingze Gao, Chuanrui Hu, Hongda Chen, et al.
   - Summary: Benchmark for agent self-evolution across web research, reasoning, software engineering, and knowledge work. Extracts trace-grounded "abilities" from execution logs and organizes them into per-domain ability graphs so gains can be attributed and transferred. Finds curated ability content transfers reliably across model families, but no current automatic self-evolution method sustains positive gain universally — a useful negative result.
   - Theme: (3) core LLM advances (agent evaluation).
   - Relevance note: One of the few benchmarks that measures transfer of learned agent behavior, not single-turn task success — worth knowing about even if you don't run it.
   - Link: https://arxiv.org/abs/2607.05202

10. **STAPO: Selective Trajectory-Aware Policy Optimization for LLM Agent Training**
    - Authors: Qiuyi Qi, Tian Liang, Mutian Bao, et al.
    - Summary: Addresses "trajectory neglect" in RL-trained LLM agents using a normalized-entropy signal that measures confidence deviation relative to the agent's own baseline behavior. Adds a trajectory-aware reward and a trajectory-independent penalty so long-horizon trajectories get credit assignment without overwhelming shorter ones. Reports SOTA on ALFWorld, WebShop, and Search-Augmented QA.
    - Theme: (3) core LLM advances (agent training / RL).
    - Relevance note: Clean, focused agent-RL contribution with results across three standard agent benchmarks — the most portable piece of the core-LLM slice this run.
    - Link: https://arxiv.org/abs/2607.04963

## 1. Do Agents Dream of False Memories? Black-box Visual Attacks on Long-term Memory in Multimodal AI Agents
- **Authors:** Halima Bouzidi, Mboutidem Ekemini Mkpong, Mohammad Abdullah Al Faruque
- **Summary:** Lucid is a black-box adversarial framework that poisons the persistent visual memory of multimodal agents through imperceptible image perturbations, assuming no access to the MLLM, the retrieval encoder, or the text channel. It exercises two failure modes — memory poisoning (adversarial image replaces a benign one whose meaning is reinforced by prior text) and memory injection (adversarial image seeds a turn with no textual anchor) — against five architectures including graph-structured, LLM-summarized, and commercially deployed memory stores. Attack-success rates of 61.6% and 58.4% reveal that current multimodal memory pipelines treat visual data with unearned trust, exposing a structural weakness of long-term memory as an agent primitive.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Rare adversarial study of long-term multimodal memory itself (not RAG), with concrete threat model and cross-architecture results — directly stresses episodic/long-term memory design assumptions.
- **Link:** https://arxiv.org/abs/2607.15657

## 2. MemoGuard: An Adaptive Runtime for Guarding Against Memory Traps in Communication-Limited Robot Navigation
- **Authors:** Rajat Bhattacharjya, Hyeonjong Ju, Sing-Yao Wu, Eli Bozorgzadeh, Nikil Dutt
- **Summary:** The paper introduces "memory traps": episodic retrievals whose similarity to the current observation is high but whose action is no longer valid because topology, resources, or prior outcomes have shifted. MemoGuard is a lightweight runtime layer that validates candidate memories against three explicit contracts (topology, resource, outcome) before reuse and only falls back to local LLM reasoning when validation fails. On a graph-based corridor-inspection simulator it cuts safety violations by 76.6% versus similarity-only reuse and 21.4% fewer fallback reasoning calls versus always-reason, saving seconds and joules per trial on a Jetson AGX Xavier.
- **Theme:** Memory for LLMs and agents
- **Relevance:** Moves episodic-memory literature beyond "retrieve and reuse" to explicit contract-based validation before use — a concrete idea for making agentic memory safe under distribution shift.
- **Link:** https://arxiv.org/abs/2607.15589

## 3. A JoLT for the KV Cache: Near-Lossless KV Cache Compression via Joint Tucker and JL-Residual Allocation for LLMs
- **Authors:** Rahul Krishnan, Volker Schulz
- **Summary:** JoLT treats each layer's KV cache as a third-order tensor over heads / tokens / features and applies a partial Tucker decomposition that compresses only the token and feature axes, then restores the discarded energy with a random-orthogonal-rotated low-bit residual. A single Lagrangian-dual allocator decides Tucker ranks and residual bit-widths jointly per layer-group and separately for K and V, all under one byte budget. The result is near-lossless 2–3× compression that preserves perplexity, GSM8K accuracy, and needle-in-a-haystack retrieval on Mistral-7B (GQA) and LLaMA-2-13B (MHA).
- **Theme:** Memory for LLMs and agents
- **Relevance:** Rethinks the cache as a tensor rather than a stack of matrices; the joint rank-and-bit allocation under one budget is the kind of substrate-level idea that carries beyond a single model family.
- **Link:** https://arxiv.org/abs/2607.12550

## 4. Long-Context Fine-Tuning with Limited VRAM
- **Authors:** Vladimir Fedosov, Aleksandr Sazhin, Artemiy Grinenko, Frank Woernle
- **Summary:** The system fuses Hierarchical Global Attention with segment-wise backprop and a tiered KV store: only the active segment is differentiable on the GPU, older KV pages spill to RAM and NVMe, and HGA loads a bounded set of exact historical tokens per query block. On a 16 GB Quadro RTX 5000 with 4-bit QLoRA on Qwen3-8B, dense training tops out at 2K tokens while HGA fits 16K training and runs inference through 131K tokens, with matched perplexity to dense-attention adapters at the shared 2K length. The savings come from turning VRAM into a rolling window while RAM and NVMe hold the rest of the working memory.
- **Theme:** Memory for LLMs and agents (with PEFT overlap)
- **Relevance:** Sits at the memory / PEFT intersection — one of the more practical routes to long-context adaptation on commodity hardware, with an explicit tiered memory story.
- **Link:** https://arxiv.org/abs/2607.15105

## 5. When Do Multi-Agent Systems Help? An Information Bottleneck Perspective
- **Authors:** Wendi Yu, Lianhao Zhou, Xiangjue Dong, Sai Sudarshan Barath, Declan Staunton, et al.
- **Summary:** The paper reframes single- vs multi-agent LLM systems through an information-bottleneck lens: a single agent accumulates its full reasoning trace in one shared context, while a multi-agent system splits reasoning into isolated local contexts joined by bounded relay messages. Under a formal parameter β capturing the compression–loss trade-off, they prove that with infinite relay bandwidth any single-agent system can be simulated by a multi-agent one, so genuine MAS gains only arise under bounded relays. Eighteen controlled experiments across five benchmarks and three model scales confirm that MAS helps when relays are near-sufficient (especially for weaker models) and hurts when relays lose task-relevant information (especially for stronger models).
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** Gives the multi-agent debate a principled decision rule instead of a beauty contest — the β-controlled compression view is directly useful for framework design.
- **Link:** https://arxiv.org/abs/2607.16133

## 6. SearchOS-V1: Towards Robust Open-Domain Information-Seeking Agent Collaboration
- **Authors:** Yuyao Zhang, Junjie Gao, Zhengxian Wu, Jiaming Fan, Jin Zhang, et al.
- **Summary:** SearchOS attacks the "agents loop on failed searches" pathology by turning implicit progress into explicit shared state: open-domain information seeking is cast as relational-schema completion with grounded citations, and a Search-Oriented Context Management layer exposes a Frontier Task, Evidence Graph, Coverage Map, and Failure Memory that any sub-agent can read and write. On top of this state, a pipeline-parallel scheduler overlaps sub-agent execution and refills freed slots against unresolved coverage gaps, while a Search Tool Middleware Harness intercepts model/tool calls to record evidence and reactively handle stalls or budget exhaustion. The result is a concrete substrate for stateful multi-agent collaboration with a reusable hierarchical skill layer on top.
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** Rare example of orchestration where the "state between agents" is designed as first-class typed structures (evidence graph, coverage map) rather than freeform relay text — a good complement to the theoretical MAS paper above.
- **Link:** https://arxiv.org/abs/2607.15257

## 7. BrainPilot: Automating Brain Discovery with Agentic Research
- **Authors:** Haoxuan Li, Tianci Gao, Jianhe Li, Yang Fan, Runze Shi, et al.
- **Summary:** BrainPilot is an open-source multi-agent system for brain-science research in which a principal-investigator agent coordinates specialist agents grounded in a curated 7,233-item knowledge base and a 72-unit skill library across seven brain-science domains. Every planning step, tool call, and evidence chain is logged in a "Graph of Trace" that lets human experts audit and intervene, and an Auditor agent runs fabrication checks over intermediate claims. It is evaluated on three brain-science tasks from Agents' Last Exam plus a new domain-specific benchmark.
- **Theme:** Multi-agent frameworks & orchestration
- **Relevance:** Concrete PI-coordinator/specialist/auditor architecture with traceable audit graph and human intervention points — a datapoint for how role assignment and orchestration handle safety-critical scientific workflows.
- **Link:** https://arxiv.org/abs/2607.15079

## 8. Understanding Reasoning from Pretraining to Post-Training
- **Authors:** Jingyan Shen, Ang Li, Salman Rahman, Yifan Sun, Micah Goldblum, et al.
- **Summary:** Using chess as a fully controlled testbed, the authors pretrain LMs from 5M to 1B parameters on human games, SFT on synthetic reasoning traces, and run RL on puzzles with verifiable rewards, then sweep both stages. They find post-RL performance at a given RL-compute budget is well-predicted by pretraining loss and that RL-reward slope grows roughly linearly with pretraining tokens. Mechanistically, RL amplifies already-preferred correct moves on easy puzzles but on hard puzzles surfaces correct moves that were nearly absent under SFT — quantifying the "what does RL actually do" question at compute scales impractical in general-domain LLM work.
- **Theme:** Core LLM advances (reasoning)
- **Relevance:** Rare controlled compute sweep across both pretraining and RL — gives predictive scaling relations for reasoning gains that current LLM-scale ablations can only approximate.
- **Link:** https://arxiv.org/abs/2607.16097

## 9. SEED: Self-Evolving On-Policy Distillation for Agentic Reinforcement Learning
- **Authors:** Jinyang Wu, Shuo Yang, Zhengxi Lu, Fan Zhang, Yuhao Shen, et al.
- **Summary:** SEED targets the supervision gap in outcome-based agentic RL by having the policy itself analyze its completed trajectories and distill natural-language "hindsight skills" (reusable workflows, decisive observations, failure-avoidance rules) back into training. During RL the same model both collects rollouts and extracts skills, so hindsight supervision co-evolves with the policy; skill-augmented and vanilla action scoring are then converted into a dense token-level on-policy distillation signal that runs alongside the outcome RL objective. This yields consistent gains on text- and vision-based long-horizon benchmarks over sparse-reward RL baselines.
- **Theme:** Core LLM advances (agentic RL / fine-tuning)
- **Relevance:** A concrete answer to the sparse-reward long-horizon problem that doesn't require an external teacher — turns the agent's own trajectories into dense supervision, which is directly on-thread for agentic RL infrastructure.
- **Link:** https://arxiv.org/abs/2607.14777

## 10. Recursive Harness Self-Improvement
- **Authors:** Hyunin Lee, Jinglue Xu, Jeffrey Seely, Donghyun Lee, Matei Zaharia, et al.
- **Summary:** RHI treats the agent harness as a prompt-level specification of the agent loop and iteratively refines it from pairwise feedback over its own revision history, so the scaffold itself is co-optimized with the model. Across 30 synthetic ML-research tasks spanning quant finance, robotics, and pharmacy, a few RHI iterations lift low-reasoning-effort agents past the maximum-reasoning-effort configuration while cutting inference cost by up to 60%. An information-theoretic analysis argues the gains come from improved task-specific context management and better inter-agent information flow, not from longer reasoning traces.
- **Theme:** Core LLM advances (agentic-system evaluation and self-improvement)
- **Relevance:** Optimizes the harness rather than the model — an underexplored axis with meaningful headroom, and it slots naturally next to the multi-agent-orchestration entries above.
- **Link:** https://arxiv.org/abs/2607.15524

## Appendix — recent late-May 2026 candidates OUTSIDE the strict 48h window (leads only, NOT counted)

These surfaced via WebSearch this run and are relevant to the three themes. Their IDs/search-derived
dates place them earlier than the 2026-05-30 → 2026-06-01 window — most are around May 27–28, i.e.
adjacent to but not inside the window. They are also not in the previous digest. Listed so the
curator has leads; deliberately excluded from the count above.

- **MemTrace — Tracing and Attributing Errors in Large Language Model Memory Systems** (~May 27,
  per a public summary) — exposes memory-system execution as a unified operation–variable graph so
  that, when an agent fails much later during retrieval/response, the faulty earlier
  construction/update/deletion operation can be located by following information dependencies
  across turns and sessions. Theme 1 (memory). https://arxiv.org/abs/2605.28732

- **Parallax — Parameterized Local Linear Attention for Language Modeling** (~May 27, per a public
  launch post) — keeps softmax attention but adds a learned covariance-correction branch that
  approximates a local-linear (rather than local-constant) regression estimate, removing LLA's
  numerical solver. Authors report pretraining at 0.6B and 1.7B with consistent perplexity gains
  transferring to downstream benchmarks, and a decode kernel competitive with FlashAttention 2/3.
  Theme 3 (core LLM advance — directly relevant as an associative-memory upgrade to attention).
  https://arxiv.org/abs/2605.29157

- **What Do Agents Communicate? Characterizing Information Exchange in Multi-Agent Systems** (~late
  May) — empirically characterizes the actual content of inter-agent messages in LLM multi-agent
  systems, rather than just protocol form. Theme 2 (multi-agent / communication protocols).
  Concrete empirical numbers were not retrievable through the search-only channel available in this
  run. https://arxiv.org/abs/2605.20548

- **Is Agent Memory a Database? Rethinking Data Foundations for Long-Term AI Agent Memory** (~late
  May) — argues that persistent agent memory needs storage primitives closer to database systems
  (versioning, transactions, query planning) rather than ad-hoc vector stores, and outlines what a
  memory-native DB would look like. Theme 1 (memory). https://arxiv.org/abs/2605.26252

- **Rethinking Memory as Continuously Evolving Connectivity (FluxMem)** (~May 27–28) — appears in
  the prior digest (`2026-05-29`); flagged here only because if WebSearch's coverage of the freshest
  window were better, more papers in this vicinity would have surfaced. Do not re-curate.
  https://arxiv.org/abs/2605.28773

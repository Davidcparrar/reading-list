## 1. Danus: Orchestrating Mathematical Reasoning Agents
- **Authors:** Jihao Liu, Guoxiong Gao, Zeming Sun
- **Summary:** Danus is a multi-agent orchestration layer for research-level math proofs that uses a shared "fact graph" as global memory across parallel proof-search workers. Instead of each agent replaying its own chain-of-thought, intermediate lemmas and claims are written into a common structured store that other agents read from and extend, turning proof search into a coordinated graph-expansion procedure. The design lets specialized reasoners contribute in parallel without redundant re-derivation while keeping the search auditable.
- **Theme:** Multi-agent frameworks & orchestration (with memory)
- **Relevance:** Directly hits two of the three themes at once — a shared fact graph functions as global working memory for a coordinated multi-agent system, exactly the axis this digest tracks.
- **Link:** https://arxiv.org/abs/2607.06447

## 2. Light-Omni: Reflex over Reasoning in Agentic Video Understanding with Long-Term Memory
- **Authors:** Chang Nie, Jiaju Wei, Junlan Feng, et al.
- **Summary:** Light-Omni is a multimodal agent that watches long video streams by maintaining two contextual memory tracks — a fast "reflex" track and a slower semantic track — so it can respond without invoking iterative reasoning on every frame. The dual-context split acts as a compact long-term memory that preserves cross-modal alignment while dropping most of the reasoning latency of prior agentic video systems. The claim is that reflexive readout from structured memory can substitute for repeated deliberation.
- **Theme:** Memory for LLMs & agents (long-term / dual-context)
- **Relevance:** Novel take on episodic vs. reflex memory in an agent — directly targets the long-term / working-memory question without leaning on RAG or a vector store.
- **Link:** https://arxiv.org/abs/2607.05511

## 3. KVpop — Key-Value Cache Compression with Predictive Online Pruning
- **Authors:** Lukas Hauzenberger, Niklas Schmidinger, Anamaria-Roberta Hartl, et al.
- **Summary:** KVpop replaces heuristic KV-cache eviction with a learned scorer that is trained against a "future-attention" target — an efficient supervisory signal that estimates whether a token will actually be attended to downstream, without materializing full attention maps. A delayed-scoring variant lets the model peek a fixed number of steps ahead before deciding what to drop, which turns out to matter for reasoning traces. On AIME/HMMT with Qwen3, they retain ~97–98% of full-attention performance at 75–88% cache compression.
- **Theme:** Memory for LLMs (context/state management, KV cache)
- **Relevance:** Directly attacks the working-memory bottleneck for long-context reasoning; the learned future-attention target is a cleaner supervision signal than typical proxy scores.
- **Link:** https://arxiv.org/abs/2607.05061

## 4. DepthWeave-KV: Token-Adaptive Cross-Layer Residual Factorization
- **Authors:** Anna Cordoba, Adam Puente Tercero, Nerea Angulo Hijo
- **Summary:** DepthWeave-KV compresses the KV cache by tying keys and values across neighboring transformer layers to a shared low-rank channel basis, and only stores a small per-token residual on top. Because the residual is token-adaptive rather than a fixed low-rank projection, tokens that carry unusual state don't get flattened. The paper reports roughly 8.3× KV memory reduction while holding long-context inference quality.
- **Theme:** Memory for LLMs (KV cache / long-context state)
- **Relevance:** Complements KVpop with an orthogonal angle — structural factorization across layers rather than eviction — making it a useful pairing when comparing memory-compression regimes.
- **Link:** https://arxiv.org/abs/2607.06523

## 5. DynaKRAG: Learnable Evidence Control in Multi-Hop RAG
- **Authors:** Yaqi Wu, Xiaolei Guo, Chenyu Zhou
- **Summary:** DynaKRAG frames multi-hop retrieval as state-conditioned control over a small set of atomic evidence operations (retrieve, reformulate, critique, filter) and trains a learned controller to sequence them. Validity layers between operations reject malformed intermediate states so the agent doesn't compound errors across hops. This gives a compact orchestration policy for evidence gathering rather than a hand-tuned RAG pipeline.
- **Theme:** Multi-agent frameworks & orchestration (evidence-gathering agent)
- **Relevance:** Reframes RAG as a controllable multi-step agent — same coordination question our multi-agent theme cares about, applied to retrieval workflows.
- **Link:** https://arxiv.org/abs/2607.06507

## 6. From Voting to Agent Collaboration: Answer-Type-Aware LLM Pipelines
- **Authors:** Taeyun Roh, Eunha Lee, Wonjune Jang
- **Summary:** This work argues that generic multi-agent voting under-serves biomedical QA because yes/no, factoid, and list questions each demand a different evidence-handling pattern. They route each question type through a specialized sub-pipeline of extractor and verifier agents rather than aggregating votes over a homogeneous ensemble. The role-assignment layer is the contribution — the agents themselves are standard LLM calls.
- **Theme:** Multi-agent frameworks & orchestration (role/task routing)
- **Relevance:** A clean small-scale example of type-aware role assignment beating majority voting — the kind of orchestration primitive worth watching as multi-agent systems mature.
- **Link:** https://arxiv.org/abs/2607.06452

## 7. Doomed from the Start: Early Abort of LLM Agent Episodes
- **Authors:** Kai Ruan, Zihe Huang, Ziqi Zhou
- **Summary:** The paper trains lightweight per-round probes on the agent's hidden activations that predict whether the current episode is heading toward failure, then aborts doomed rollouts before wasting more tool calls. The probes are cheap (a few numbers per step) and trigger a cascade of abort thresholds, saving roughly 37–47% of inference compute at comparable task success. It's a concrete argument that internal representations already encode failure signals earlier than external reward.
- **Theme:** Core LLM advances (agentic-system evaluation & efficiency)
- **Relevance:** Cleanly formulated, has empirical teeth, and points at a general primitive — probe-based rollout gating — that would compose with any of the multi-agent frameworks above.
- **Link:** https://arxiv.org/abs/2607.06503

## 8. TurnOPD: Making On-Policy Distillation Turn-Aware for Efficient Long-Horizon Agent Training
- **Authors:** Yuhang Zhou, Kai Zheng, Haoling Li, et al.
- **Summary:** Full-trajectory on-policy distillation for long-horizon agents burns most compute on late turns whose gradients contribute little. TurnOPD adds a turn-level budget: distillation targets and rollout length are scheduled per turn rather than per episode, so the training signal is spent where it actually shapes behavior. The result is more agent-task learning per GPU-hour without a distinct algorithmic change to the objective.
- **Theme:** Core LLM advances (fine-tuning / agentic training)
- **Relevance:** Practical fine-tuning trick specifically for agentic post-training — a class of methods that will keep getting attention as long-horizon RL becomes mainstream.
- **Link:** https://arxiv.org/abs/2607.05804

## 9. TREK: Distill to Explore, Reinforce to Refine
- **Authors:** Yuanda Xu, Zhengze Zhou, Kayhan Behdin, et al.
- **Summary:** TREK is a two-stage recipe: first widen the policy's exploration support by distilling from a stronger teacher, then run policy-optimization RL on top of that broadened distribution. The distillation stage matters because RL from a narrow initial policy collapses on hard math and agentic problems; the distillation stage gives RL something to refine. The paper reports gains on math-reasoning and agent-task benchmarks over pure RL or pure SFT baselines.
- **Theme:** Core LLM advances (reasoning + fine-tuning)
- **Relevance:** A clean, reusable staging pattern for reasoning-heavy post-training; the exploration/refinement split is the kind of recipe worth borrowing.
- **Link:** https://arxiv.org/abs/2607.05339

## 10. An Experimental Design Approach to Evaluating Agentic AI
- **Authors:** Hao He, Xueying Liu, Chris J. Kuhlman
- **Summary:** The authors treat coding agents as stochastic operators over specification → code and apply factorial experimental design — varying reasoning effort, model, and task complexity — to estimate how each factor moves quality, cost, and output complexity. The point is methodological: single-run agent benchmarks systematically mislead, and this gives a principled way to report agentic capabilities with confidence intervals. Useful as a template for how future agent evaluations should be reported.
- **Theme:** Core LLM advances (agentic-system evaluation)
- **Relevance:** Fills a gap most benchmark papers ignore — variance-aware evaluation of stochastic agents — and is stat-methods rigorous rather than another leaderboard.
- **Link:** https://arxiv.org/abs/2607.06413

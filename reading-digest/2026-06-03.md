## 1. Unified Context Evolution for LLM Agents
**Authors:** Zixuan Zhu, Yitong Hu, Yong Dai, et al.

**Summary:** This paper proposes UCE, a gradient-free framework that turns an agent's accumulated experience into an external, versioned library of typed "Evolvable Context Units" rather than baking it into weights or letting it die with each episode. Experience is explicitly decomposed into four kinds ‚Äî Memory, Strategy, Workflow, and Skill ‚Äî each generated from past trajectories under type-specific extraction conditions, retrieved at decision time, scored by repeated downstream-use outcomes, and pruned when they stop paying off. The reported results push ALFWorld success from 75.4% ‚Üí 96.3% and WebShop task score from 45.1% ‚Üí 61.3%, which makes UCE a serious candidate framework for episodic/procedural agent memory without retraining.

**Theme:** 1 ‚Äî Memory (typed long-term/episodic memory, retrieval-augmented context state).

**Relevance:** Clearest top-of-priority Theme 1 contribution in the window: it directly attacks the "useful strategies vanish at episode end" failure mode, the typed taxonomy (Memory/Strategy/Workflow/Skill) is unusually principled compared to flat experience buffers, and the ALFWorld/WebShop deltas are large enough to be more than noise. Method-and-architecture paper, not a survey.

**Link:** https://arxiv.org/abs/2606.02304

## 2. ExpWeaver: LLM Agents Learn from Experience via Latent RAG
**Authors:** Tao Feng, Tianyang Luo, Jingjun Xu, et al.

**Summary:** ExpWeaver attacks the same "agents should learn from past episodes" problem from a structurally different angle than UCE: instead of building a text-space library and re-injecting selected entries into the context window, it encodes experiences using the LLM's *own hidden states* and retrieves them in latent space at each decoding step, then folds them into generation via cross-attention aggregation and gated residual connections. This collapses the usual retrieval-then-prompt-concatenation pipeline into a single in-model mechanism, with the explicit goal of cutting the token overhead that grows linearly in surfaced experiences. The contribution is methods-level: a tighter coupling between an agent's experience store and its decoder than text-space RAG allows.

**Theme:** 1 ‚Äî Memory (retrieval-augmented memory; representation of experience in latent rather than token space).

**Relevance:** Pairs naturally with #1 as a methodological contrast ‚Äî both target experience reuse for agents but split on the representational substrate (typed text units vs. latent hidden states). Worth reading next to UCE to see which side of that trade-off you find more convincing.

**Link:** https://arxiv.org/abs/2606.01041

## 3. COMAP: Co-Evolving World Models and Agent Policies for LLM Agents
**Authors:** Youwei Liu, Jian Wang, Hanlin Wang, et al.

**Summary:** COMAP frames the agent-improvement loop as joint evolution of a *textual world model* and a *policy* through closed-loop interaction, rather than treating either as fixed. At each decision step the world model predicts feedback for candidate actions, the agent does "future-aware reflection" by estimating the reliability of that predicted feedback, and then refines its action ‚Äî and over time both the world model and the policy are updated against each other. The contribution is the dual-update architecture: a way to keep the agent's internal predictive model and its action policy from drifting apart as experience accumulates.

**Theme:** 1 ‚Äî Memory (context/state evolution via a learned, evolving textual world model that functions as a long-horizon memory of dynamics).

**Relevance:** Sits at the boundary of memory and planning ‚Äî earns Theme 1 placement because the textual world model is effectively a structured, updateable belief store. Notable because it explicitly addresses the reliability-of-its-own-memory problem (the agent reasons about how much to trust its world model), which most Theme 1 papers in this window don't.

**Link:** https://arxiv.org/abs/2606.02372

## 4. AMP / memorywire: A Vendor-Neutral Wire Format for Agent Memory Operations
**Authors:** Thamilvendhan Munirathinam

**Summary:** This is an infrastructure/standards paper, not a learning paper: it defines `memorywire` (originally drafted as "AMP"), a JSON-Schema 2020-12 wire format specifying five memory operations ‚Äî `remember`, `recall`, `forget`, `merge`, `expire` ‚Äî over four declared memory types (semantic, episodic, procedural, emotional), together with a `MemoryStore` interface, a fan-out router, and an optional human-in-the-loop governance channel. The point is portability: any agent framework can implement the wire format and any memory backend can plug in behind it, so memory becomes a swappable layer rather than a per-framework bespoke implementation. The contribution is the explicit operation/type taxonomy and the governance hook, not new algorithms.

**Theme:** 1 ‚Äî Memory (cross-framework memory protocol; explicit operation set for long-term/episodic memory management).

**Relevance:** Theme 1 in a different sense than #1‚Äì#3: it doesn't propose a new memory mechanism, it proposes the *interface* that multiple memory mechanisms would have to share to interoperate. Useful if you care about the systems/engineering side of agent memory or about whether "agent memory" is heading toward standardization. Small-author paper; treat as a position-and-spec piece, not a results paper.

**Link:** https://arxiv.org/abs/2606.01138

## 5. Characterization of Multi-Model Agentic AI Systems on General Tasks via Trace-Driven Simulation
**Authors:** (multiple; see arXiv) ‚Äî paper introduces GAIATrace + Vidur-Agent.

**Summary:** This work argues that we evaluate agentic systems mostly as black boxes and proposes the missing instrumentation: GAIATrace, a token-level trace dataset captured from two state-of-the-art agentic systems (MiroThinker and OWL) running the heterogeneous GAIA benchmark, recording full reasoning tokens, task-level structure, and the activity of every participating LLM. Alongside the dataset they release Vidur-Agent, a trace-driven simulator that replays GAIATrace to enable reproducible, low-cost system evaluation across different simulated environments. The contribution is a measurement substrate for multi-agent / multi-model agentic AI, intended to make systems-level behavior reproducible despite the usual non-determinism and proprietary-model opacity.

**Theme:** 2 ‚Äî Multi-agent frameworks & orchestration (systems-level characterization and reproducible evaluation of multi-model agentic AI).

**Relevance:** The most rigorous in-window Theme 2 paper from a systems-research angle: trace-level visibility into a real multi-model agentic stack plus a public simulator is the kind of artefact that the multi-agent eval literature has been missing. Strong complement to any framework paper since it gives you a way to actually replay and dissect runs.

**Link:** https://arxiv.org/abs/2606.01725

## 6. LLM Consortium for Software Design Refinement: A Controlled Experiment on Multi-Agent Collaboration Topologies
**Authors:** Nagarjuna Kanamarlapudi, Praveen K

**Summary:** This paper runs a controlled empirical comparison of different multi-agent collaboration *topologies* for an LLM consortium tasked with software design refinement, rather than just proposing one framework and reporting wins. The setup treats the choice of topology (how agents are wired, which roles communicate, how decisions are aggregated) as the independent variable. The contribution is empirical-methodological: a structured experiment over multi-agent topologies for a concrete engineering task, which is the kind of comparison that's usually missing from "we built a multi-agent system" papers.

**Theme:** 2 ‚Äî Multi-agent frameworks & orchestration (topology, role/communication structure, controlled comparison).

**Relevance:** Different in flavor from #5: this is a topology/design-space study rather than a measurement-and-simulation infrastructure paper. Useful as the "what wiring should we pick?" companion to #5's "how do we measure what wiring did?".

**Link:** https://arxiv.org/abs/2606.01490

## 7. An Agentic Approach Towards Replication Package Quality Evaluation
**Authors:** Maximilian Alexander Amougou Mbida, Florian Angermeir

**Summary:** This paper translates open-science reproducibility guidelines into machine-verifiable criteria ‚Äî consolidating 380 requirements from 34 sources down to 51 criteria, 31 of which they operationalize for automated artifact-based evaluation ‚Äî and then implements a multi-agent prototype that inspects replication packages and emits evidence-grounded improvement reports. A preliminary five-package evaluation reports 91.4% inter-run consistency and 75.4% micro-averaged correctness against a manual baseline. The contribution is a concrete, criterion-grounded multi-agent application: the framework strongly relies on role-specialized inspector agents whose outputs have to be aggregated into a single defensible verdict.

**Theme:** 2 ‚Äî Multi-agent frameworks & orchestration (role-specialized agents, evidence aggregation, criterion-grounded coordination).

**Relevance:** A worked example of "multi-agent system with verifiable acceptance criteria", which is interesting because the *criteria* substitute for the usual fuzzy LLM-as-judge step. The qualitative-criterion failure mode the authors flag is itself a useful methodological signal.

**Link:** https://arxiv.org/abs/2606.02006

## 8. Discovering Agents for Discovery: The Case for DNS
**Authors:** Ramachandra Rao Seethiraju, Sameer Thakar, Karthik Shyamsunder, et al.

**Summary:** This is a position-and-proposal paper from Verisign arguing that as AI agents proliferate across the Internet, the unanswered question of how agents *find* each other ‚Äî and authenticate, establish integrity, authorize, and scale across organizational boundaries ‚Äî should be handled by the DNS rather than by reinventing a parallel directory. They sketch the basic semantics an agent-discovery layer on top of DNS would need, and use that to ground a discussion of what discovery actually requires in practice. The contribution is an infrastructure proposal for multi-agent interoperability, anchored in an existing widely-deployed substrate rather than a greenfield protocol.

**Theme:** 2 ‚Äî Multi-agent frameworks & orchestration (cross-organization agent discovery; coordination infrastructure).

**Relevance:** Theme 2 from the infrastructure angle: it's not a coordination algorithm but a claim about what *layer* coordination should live at. Worth reading specifically as a counterpoint to in-framework agent registries / vendor-specific agent gateways ‚Äî orthogonal to #5 (measurement) and #6 (topology), and pairs interestingly with #4's "memory should be a portable wire format" stance.

**Link:** https://arxiv.org/abs/2606.02314

## 9. TimeSage-MT: A Multi-Turn Benchmark for Evaluating Agentic Time Series Reasoning
**Authors:** Yaxuan Kong, Qingren Yao, Yuqi Nie, et al.

**Summary:** TimeSage-MT is a 240-task / 2,680-dialogue-turn evaluation suite for agentic time-series reasoning across 8 real-world domains, deliberately structured as *multi-turn* dialogues that range from basic exploration to decision-oriented analysis ‚Äî not just one-shot QA over a series. The benchmark ships with a unified evaluation protocol and a public leaderboard so different agentic time-series systems can be compared on the same footing. The contribution is a benchmark with a coherent evaluation protocol for an under-served regime (multi-turn agentic reasoning on temporal data), targeting the gap between single-turn time-series QA benchmarks and how time-series agents are actually used.

**Theme:** 3 ‚Äî Core LLM advances (agentic-system evaluation; multi-turn reasoning on a specific data modality).

**Relevance:** The most directly Theme-3-on-evaluation paper in window: agentic-system evaluation is explicitly called out in the spec, and "multi-turn over time series" is one of the more practically relevant agent settings to benchmark. Narrower scope than a general agentic benchmark, but with a real protocol and leaderboard.

**Link:** https://arxiv.org/abs/2606.01498

## 10. Geometric Latent Reasoning Induces Shorter Generations in LLMs
**Authors:** Shashi Kumar, Yacouba Kaloga, Petr Motlicek, et al.

**Summary:** This paper adds a lightweight *latent transition head* to an LLM that predicts continuous direction updates in embedding space, approximating the trajectory that token-level decoding would normally trace out. The point is to do part of the chain-of-thought work in continuous latent space, so that the surface generation can be shorter while the underlying reasoning is preserved ‚Äî a different cut on the "spend less tokens, reason as well or better" axis than chain-of-thought compression or speculative decoding. The contribution is a method, with the empirical claim that geometric latent reasoning measurably shortens generations.

**Theme:** 3 ‚Äî Core LLM advances (reasoning, latent reasoning, efficient generation).

**Relevance:** Theme 3 from the reasoning side rather than the evaluation side, balancing #9. Worth including because latent-reasoning methods are a fast-moving sub-area and this is a concrete proposal rather than a survey or a position piece; relevance to the digest's core themes is "core LLM advance directly useful for agents that pay per token" rather than agentic per se.

**Link:** https://arxiv.org/abs/2606.02248

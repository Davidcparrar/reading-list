# Reading digest — 2026-06-30

Found 10 paper(s) in the last 48h window.

## 1. Self-Evolving World Models for LLM Agent Planning
- Authors: Xuan Zhang, Wenxuan Zhang, See-Kiong Ng, Yang Deng
- Summary: WorldEvolver wraps a frozen LLM agent with a self-evolving world model that revises only its deployment-time context. It combines an episodic memory of real action transitions used for retrieval-based simulation, a semantic memory that distills persistent heuristics from prediction–observation mismatches, and a selective-foresight filter that drops low-confidence predictions before they enter the agent's reasoning, yielding the top prediction accuracy on Word2World and the strongest downstream success on AgentBoard/ALFWorld/ScienceWorld.
- Theme: Memory for LLMs and agents
- Relevance: Clean episodic + semantic memory framework that pairs retrieval and rule extraction with no parameter updates — directly on-theme for memory-augmented planning.
- Link: https://arxiv.org/abs/2606.30639

## 2. Mandol: An Agglomerative Agent Memory System for Long-Term Conversations
- Authors: Yuhan Zhang, Zhiyuan Guo, Ziheng Zeng, et al.
- Summary: Mandol replaces the typical patchwork of vector + graph stores with a single memory-native architecture: a hierarchical model splits raw and abstract memories into structured semantic graphs, and an agglomerative SemanticMap/SemanticGraph fuses key-value, vector, and graph access behind unified hybrid retrieval operators. A query-adaptive router with token-constrained context generation removes the need to call an LLM during retrieval, beating prior agent-memory systems on LoCoMo and LongMemEval while delivering 5.4× retrieval and 4.8× insertion speedups under load.
- Theme: Memory for LLMs and agents
- Relevance: A systems-level rethink of long-term conversational memory — exactly the kind of memory architecture work the digest tracks.
- Link: https://arxiv.org/abs/2606.29778

## 3. Neural Procedural Memory: Empowering LLM Agents with Implicit Activation Steering
- Authors: Chengfeng Zhao, Yuqiao Tan, Shizhu He, Yequan Wang, Jun Zhao, Kang Liu
- Summary: NPM is a training-free agent-memory framework that stores procedural skills as activation-space steering vectors distilled from contrastive past experiences, sidestepping the text-action disconnect of RAG-style instruction injection. Across four agent benchmarks it matches explicit-instruction baselines, and stacking implicit steering with explicit workflows is complementary; representational analysis shows the vectors form organized, task-consistent structures.
- Theme: Memory for LLMs and agents
- Relevance: A novel non-RAG approach to agent memory that lives inside the model's activations — a fresh angle on procedural/working memory.
- Link: https://arxiv.org/abs/2606.29824

## 4. Selective Memory Retention for Long-Horizon LLM Agents
- Authors: Pranath Reddy
- Summary: TraceRetain is a bounded external memory for frozen LLM agents that scores entries on interpretable features (success, age, access, redundancy, specificity, similarity, downstream utility) and evicts the lowest-scoring ones. On clean ALFWorld retention policies are statistically indistinguishable, but under a 75% noisy-write stress unbounded memory and FIFO-K50 collapse on Precision@5 while TraceRetain-CEM stays essentially flat and preserves 97/100 task success, isolating when retention actually pays.
- Theme: Memory for LLMs and agents
- Relevance: Rare empirical study isolating the conditions under which memory retention strategies matter — useful for anyone building bounded agent memory.
- Link: https://arxiv.org/abs/2606.29178

## 5. MemDelta: Controlled Baselines and Hidden Confounds in Agent Memory Evaluation
- Authors: Kuan Wang
- Summary: MemDelta runs controlled ablations across agent-memory pipelines and shows that swapping a single component — for example just the embedding model — can shift accuracy by +6.2 pp, large enough to flip the ranking between published methods. The paper surfaces hidden confounds in how agent memory is benchmarked and proposes baselines that hold all but one factor fixed so reported gains can be attributed to memory design rather than to incidental pipeline choices.
- Theme: Memory for LLMs and agents
- Relevance: Methodology paper that should change how memory results are read and reported — high signal for anyone tracking the subfield.
- Link: https://arxiv.org/abs/2606.29914

## 6. MAS-Lab: A Specification-Driven Validation Framework for Reliable Multi-Agent Systems
- Authors: Jordan Augé, Giovanna Carofiglio, Giulio Grassi, Jacques Samain
- Summary: MAS-Lab pushes MAS away from ad-hoc imperative scripts toward an engineered distributed system: a framework-agnostic declarative Spec layer captures intent, a stateful MAS Operating System provides pluggable execution and control primitives, and Labs overlays add observability and evaluation. Together they support intent-based validation, reproducible experiments, and a seamless path from prototype to governed production deployment.
- Theme: Multi-agent frameworks & orchestration
- Relevance: Concrete architecture for production-grade MAS — directly on the orchestration/framework axis of the digest.
- Link: https://arxiv.org/abs/2606.30546

## 7. Minority Sentinel: When to Overturn Majority Voting in Multi-Agent LLM Debates
- Authors: Chuan He, Zebin Chen, Zhengyi Yang, et al.
- Summary: The paper shows that because LLM agents share pretraining corpora their errors are correlated, so majority voting in multi-agent debate systematically suppresses correct minority answers — about one in four divergent cases. Minority Sentinel trains a lightweight LightGBM meta-classifier on a multi-dimensional debate fingerprint to decide when to flip the majority, achieving 81.2% flip precision with positive net gain across six benchmarks and beating an LLM-as-Judge baseline that recovers more but flips unsafely.
- Theme: Multi-agent frameworks & orchestration
- Relevance: Surgical fix for a well-known failure mode of voting-based MAS, with a non-LLM classifier that's cheap to deploy.
- Link: https://arxiv.org/abs/2606.29270

## 8. PolicyGuard: A Dialogue-Grounded Sub-Agent Verifier for Policy Adherence in LLM Agents
- Authors: Seongjae Kang, Taehyung Yu, Sung Ju Hwang (KAIST / DeepAuto.ai)
- Summary: PolicyGuard reframes agent policy compliance as a dialogue-level problem rather than an argument-level safeguard: a sub-agent verifier shares the main agent's full conversation view, reasons over the system-prompt policy in context, and feeds back actionable corrections for the next turn. On τ²-bench airline with GPT-5.4, Claude Sonnet 4.6, and Gemini 2.5 Pro it lifts Pass⁴ by +6 to +12 pp while blocking about half as often as argument-level guards, with higher policy-violation recall.
- Theme: Multi-agent frameworks & orchestration
- Relevance: Verifier-as-sub-agent pattern is becoming a core MAS primitive — strong empirical wins on a live tool-use benchmark.
- Link: https://arxiv.org/abs/2606.29225

## 9. Scaling the Horizon, Not the Parameters: Reaching Trillion-Parameter Performance with a 35B Agent
- Authors: Wenlong Zhang, Fenghua Ling, Xin Li, et al. (Intern Science)
- Summary: Agents-A1 is a 35B mixture-of-experts model trained to scale agent horizon rather than parameter count: a knowledge-action infrastructure threads tool calls and external knowledge across up to 45K tokens, and a three-stage pipeline (full-domain SFT → domain-specific teacher models → multi-teacher on-policy distillation with salient-vocabulary alignment) unifies six domains. It matches or beats much larger models on SEAL-0, IFBench, FrontierScience-Olympiad, and MolBench-Bind while staying cheap to serve.
- Theme: Core LLM advances directly relevant to agents (training & long-horizon agentic capability)
- Relevance: Concrete recipe for compact agentic models with long horizons — useful blueprint for downstream agent training.
- Link: https://arxiv.org/abs/2606.30616

## 10. TACO: Tool-Augmented Credit Optimization for Agentic Tool Use
- Authors: Mingkuan Feng, Jinyang Wu, Hao Gu, Fangrui Lv, Ruihan Jin, et al.
- Summary: TACO is a GRPO variant that gives code-tool agents per-call credit without any external judge. A Differential Answer-Probe Reward inserts probe tokens to compare the model's predicted answer with and without each tool call — positive credit if the call helped, negative if it misled — and Outcome-Gated Advantage Routing then distributes the final-answer advantage only to the segments responsible for it. Trained with a two-stage SFT+RL pipeline, TACO yields consistent gains across perception, reasoning, and general multimodal benchmarks and learns to invoke tools only when they actually help.
- Theme: Core LLM advances directly relevant to agents (RL credit assignment for tool use)
- Relevance: A judge-free, drop-in RL recipe for tool-using agents — exactly the kind of agentic-system advance the digest tracks.
- Link: https://arxiv.org/abs/2606.30251

1. **MemLearner: Learning to Query Context Memory for Video World Models**
   - Authors: Jiwen Yu, Jianxiong Gao, Jianhong Bai, et al.
   - Summary: The paper attacks the long-standing "which past frames should I attend to" problem in video world models by making the retrieval itself a learned policy — a set of query tokens conditioned on the current state adaptively selects context frames from an episodic buffer, rather than relying on hand-designed sliding windows or fixed re-encoders. The retrieval module is bootstrapped on top of frozen visual priors so no extra encoder training is needed, and is trained on a mix of annotated rendered and unannotated real-world videos. Reported wins over prior context-selection baselines on scene consistency under occlusion and dynamic scenes.
   - Theme: (1) memory for LLMs and agents (episodic/context memory).
   - Relevance note: The clearest new "learned memory retrieval" contribution in the window — exactly the top-priority track, and the recipe is transportable beyond video world models.
   - Link: https://arxiv.org/abs/2606.31734

2. **ReFreeKV: Towards Threshold-Free KV Cache Compression**
   - Authors: Xuanfan Ni, Liyan Xu, Chenyang Lyu, et al.
   - Summary: ReFreeKV replaces the hand-tuned attention/importance thresholds used by most KV-cache eviction schemes with an adaptive per-input budget allocator, so compression stops requiring dataset- or model-specific configuration to stay accurate. The method reallocates the budget across layers/heads based on cache statistics at runtime and evaluates across diverse tasks and model scales. The take-away is a plug-in compression path for long-context inference that survives changes in workload without re-tuning.
   - Theme: (1) memory for LLMs and agents (working memory / KV context management).
   - Relevance note: Cache compression IS agent working memory in disguise; a threshold-free recipe matters for long-running agent loops.
   - Link: https://arxiv.org/abs/2502.16886

3. **Little Brains, Big Feats: Exploring Compact Language Models**
   - Authors: Dari Baturova, Elena Bruches, Ivan Chernov, et al.
   - Summary: This is a benchmarking study of small (sub-billion / few-billion-parameter) LMs used as the generator inside retrieval-augmented pipelines, across mixed-domain datasets and question types. The authors ask whether these models can run RAG entirely on-device (no GPU) within reasonable latency, and report where quality holds up versus where it degrades. Result: compact LMs are practically deployable for on-device RAG across a wider slice of subject areas than one might expect from raw benchmark scores.
   - Theme: (1) memory for LLMs and agents (retrieval-augmented memory).
   - Relevance note: Useful sanity check for anyone designing on-device agent-memory stacks with small backbones.
   - Link: https://arxiv.org/abs/2606.30062

4. **Scaling the Horizon, Not the Parameters: Reaching Trillion-Parameter Performance with a 35B Agent (Agents-A1)**
   - Authors: Lei Bai, Zongsheng Cao, Yang Chen, et al.
   - Summary: Agents-A1 argues that agentic capability is bottlenecked by *horizon* (long trajectory training) rather than raw parameter count, and lays out a three-stage training methodology that mixes heterogeneous agent abilities and long-horizon trajectory supervision on top of a 35B MoE backbone. The claim is that this configuration reaches trillion-parameter performance on agent benchmarks. It is framed as a full recipe (data curation + staged training) rather than an architectural tweak.
   - Theme: (2) multi-agent frameworks & orchestration (long-horizon agent training).
   - Relevance note: One of the more ambitious end-to-end agent recipes in the window; directly relevant to teams sizing their own agent backbones.
   - Link: https://arxiv.org/abs/2606.30616

5. **DataEvolver: Self-Evolving Multi-Agent Data Construction for Text-Rich Image Generation**
   - Authors: Siyu Yan, Yizhen Gao, Yilin Wang, Dongxing Mao, Alex Jinpeng Wang
   - Summary: DataEvolver treats data construction as a policy that a team of specialised agents — retrieval, verification, criticism, generation — evolves round-by-round, with a shared updated memory guiding the next round's decisions. The framework is applied to text-rich image generation and produces its own training corpus. At the 0.75M-example scale the authors report +85.3% OCR-F1 on TextScenesHQ and +35.3% on LongTextBench over the strongest single-agent baseline.
   - Theme: (2) multi-agent frameworks & orchestration (with a shared-memory component).
   - Relevance note: A clean "role-decomposed team with shared memory" pattern — exactly the multi-agent orchestration shape we're tracking, and one of the few reporting concrete downstream deltas.
   - Link: https://arxiv.org/abs/2606.31537

6. **OSWorld2.0: Benchmarking Computer Use Agents on Long-Horizon Real-World Tasks**
   - Authors: Mengqi Yuan, Zilong Zhou, Xinzhuang Xiong, et al.
   - Summary: OSWorld2.0 is a benchmark of 108 long-horizon computer-use workflows that require on the order of ~318 tool calls each, exposing hidden-state recovery, constraint tracking, and multi-app coordination in a way the original OSWorld did not. Frontier agents complete only ~20.6% of tasks in the reported evaluation, and the authors break down the dominant failure modes. The dataset is positioned as the new bar for evaluating computer-use / GUI agent stacks.
   - Theme: (2) multi-agent frameworks & orchestration (evaluation).
   - Relevance note: If you're building an agent framework, this is the benchmark to point at going forward — the failure taxonomy alone is useful.
   - Link: https://arxiv.org/abs/2606.29537

7. **TACO: Tool-Augmented Credit Optimization for Agentic Tool Use**
   - Authors: Mingkuan Feng, Jinyang Wu, Hao Gu, et al.
   - Summary: TACO tackles the credit-assignment problem in multi-step tool-using agents by explicitly separating "useful" code/tool operations from "misleading" ones via a differential answer-probe reward, then routing the outcome-gated advantage back to the tool-invocation steps that actually moved the probe. It is applied to multimodal agents and pitched as an alternative to uniform per-step rewards or coarse outcome-only RL. The framing is: reward the tool step in proportion to how much it shifted the model's answer confidence.
   - Theme: (2) multi-agent frameworks & orchestration (tool orchestration + RL).
   - Relevance note: Concrete new signal for training tool-using agents — complements the "reward-hacking" thread from last week's Verification Horizon paper.
   - Link: https://arxiv.org/abs/2606.30251

8. **Xiaomi-GUI-0 Technical Report**
   - Authors: Wanxia Cao, Chengzhen Duan, Pei Fu, et al.
   - Summary: A native multimodal GUI agent trained in a hybrid infrastructure that mixes real physical Android devices with lightweight sandboxes, with an "error-driven flywheel" that turns rolled-out failure traces into corrected actions plus reflection data for the next training round. Reported at 72.0% on RealMobile and 78.9% on AndroidWorld with better stability than prior GUI-agent stacks. The report is dense on the data and infrastructure recipe.
   - Theme: (2) multi-agent frameworks & orchestration (GUI agent architecture).
   - Relevance note: Rare open-ish look at what an end-to-end production GUI-agent training loop actually looks like — infrastructure notes are the interesting bit.
   - Link: https://arxiv.org/abs/2606.31410

9. **Reinforcement Learning with Metacognitive Feedback Elicits Faithful Uncertainty Expression in LLMs**
   - Authors: Gabrielle Kaili-May Liu, Avi Caciularu, Gal Yona, Idan Szpektor, Arman Cohan
   - Summary: RLMF re-ranks preference pairs during RL not by outcome but by the *quality of the model's own self-judgment* about its answer, then uses a two-stage pipeline that first calibrates confidence and then maps that calibrated confidence to linguistic hedges ("I'm 60% sure that…"). The authors report a 63% relative improvement in faithful-uncertainty expression over standard RL while holding accuracy fixed. The result is a route to LLMs whose stated confidence tracks their real confidence.
   - Theme: (3) core LLM advances (RL + reasoning + honesty).
   - Relevance note: Directly relevant to agents that need to abstain or escalate — calibrated verbal confidence is the missing signal in most orchestration stacks today.
   - Link: https://arxiv.org/abs/2606.32032

10. **Multi-Block Diffusion Language Models**
    - Authors: Yijie Jin, Jiajun Xu, Yuxuan Liu, et al.
    - Summary: This paper extends block-diffusion LMs from single-block to concurrent multi-block decoding, introducing a Multi-block Teacher Forcing objective with randomised noise schedules that match the actual inference-time state, plus a decoding scheme that preserves prefix-cache reuse under static input shapes. The reported effect is Tokens-Per-Forward-pass climbing from 3.47 to 6.19 with an *accuracy improvement*, not the usual quality regression that plagues parallel decoding. A useful data point on non-autoregressive scaling.
    - Theme: (3) core LLM advances (non-AR / diffusion LM decoding).
    - Relevance note: Follow-up to last week's iLLaDA (also 8B masked diffusion) — the diffusion-LM track is getting real enough to matter for backbone choice.
    - Link: https://arxiv.org/abs/2606.29215
